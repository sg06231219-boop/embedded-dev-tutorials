1. 串口基础概念

1.1 什么是串口通信？

串口通信是一种串行通信方式，数据按位（bit）逐个传输，只需要少量的信号线。与之相对的是并行通信，一次传输多个位。

想象一条单车道公路，车辆（数据位）必须一辆接一辆地通过，这就是串行通信。而并行通信就像八车道高速公路，八辆车可以并排同时通过。

1.2 UART协议详解

UART（Universal Asynchronous Receiver Transmitter，通用异步收发器）是串口通信的核心协议。

异步 vs 同步
  • 异步通信：不需要时钟线同步，双方约定好波特率即可。UART就是异步的。
  • 同步通信：需要一根时钟线（如SCL）同步数据，如SPI、I2C。

UART数据帧结构

一个完整的UART数据帧包含以下部分：

空闲状态(高电平) → 起始位(低) → 数据位(5-9位) → 校验位(可选) → 停止位(高)
部分 说明 典型值
起始位 拉低电平，表示数据开始 1位
数据位 实际传输的数据 8位（最常用）
校验位 用于检测传输错误 无校验/奇校验/偶校验
停止位 拉高电平，表示数据结束 1位或2位

1.3 波特率详解

波特率（Baud Rate）表示每秒传输的码元数量，单位是bps（bits per second）。在UART中，波特率等于比特率。

常见波特率

9600 19200 38400 57600 115200

115200是最常用的波特率，每秒可传输约 115200 ÷ 10 = 11520字节（假设8N1配置）。

波特率必须双方一致！如果发送方用115200，接收方用9600，数据会完全乱掉。

1.4 数据位、校验位、停止位

数据位

数据位表示每个数据帧携带的有效数据量，常用8位（正好一个字节）。

校验位

校验位用于检测传输过程中的错误：

  • 无校验（None）：不使用校验位，最常用
  • 奇校验（Odd）：数据位+校验位共有奇数个1
  • 偶校验（Even）：数据位+校验位共有偶数个1
示例：数据 0x55 (01010101)

有4个1，偶校验时校验位=0，奇校验时校验位=1

停止位

停止位表示数据帧的结束，可以是1位或2位。1位停止位是最常用的配置。

1.5 常见串口配置

串口配置通常用 "数据位-校验位-停止位" 格式表示：

配置 含义 应用场景
8-N-1 8数据位，无校验，1停止位 最常用，适合大多数场景
8-E-1 8数据位，偶校验，1停止位 工业设备，需要错误检测
7-E-1 7数据位，偶校验，1停止位 老式调制解调器

2. 硬件连接

2.1 USB转TTL模块介绍

现代电脑很少有真正的COM口，我们使用USB转TTL模块来模拟串口。

常用芯片对比
芯片型号 特点 价格 推荐度
CH340 国产，便宜，稳定 ￥2-5 ⭐⭐⭐⭐⭐
CP2102 Silicon Labs，性能好 ￥8-15 ⭐⭐⭐⭐
PL2303 Prolific，老牌 ￥5-10 ⭐⭐⭐
FT232 FTDI原厂，最稳定 ￥20-40 ⭐⭐⭐⭐⭐
新手推荐买CH340模块，便宜好用，坏了也不心疼。

2.2 模块引脚说明

一个典型的USB转TTL模块有以下引脚：

引脚 名称 说明 连接方式
VCC 电源输出 给目标板供电（可选） 接目标板电源输入
GND 地线 必须连接！ 接目标板GND
TXD 发送端 模块发送数据 接目标板RX
RXD 接收端 模块接收数据 接目标板TX

2.3 正确的接线方法

核心原则：TX接RX，RX接TX，GND必须共地！
接线步骤
  1. 共地：模块GND → 目标板GND（必须先接！）
  2. 交叉连接：模块TXD → 目标板RX
  3. 交叉连接：模块RXD → 目标板TX
  4. 电源（可选）：模块VCC → 目标板电源输入
接线示意图
USB转TTL模块              STM32开发板
    ┌─────────┐              ┌─────────┐
    │   VCC   │──────────────│   5V    │
    │   GND   │══════════════│   GND   │  ← 必须共地！
    │   TXD   │──────────────│   RX    │  ← 交叉
    │   RXD   │──────────────│   TX    │  ← 交叉
    └─────────┘              └─────────┘
想象打电话：你的嘴巴（TX）对着别人的耳朵（RX），别人的嘴巴（TX）对着你的耳朵（RX）。如果嘴巴对嘴巴，耳朵对耳朵，那谁也听不见谁！

2.4 电压问题：3.3V vs 5V

这是新手最容易踩的坑！

常见电平标准
电平类型 电压范围 典型设备
5V TTL 0V/5V Arduino Uno, 老式设备
3.3V TTL 0V/3.3V STM32, ESP32, ESP8266
RS232 ±12V 传统DB9串口
5V模块直接接3.3V芯片可能烧毁芯片！

解决方案：

  • 购买带3.3V/5V切换开关的模块
  • 使用电平转换模块
  • 串联1K电阻限流（不推荐，治标不治本）
大多数CH340模块已经兼容3.3V和5V，但买之前一定要确认！

    port='COM3',

5.5 完整收发示例

import serial
import threading
import time

class SerialComm:
    def __init__(self, port, baudrate=115200):
        self.ser = serial.Serial(
            port=port,
            baudrate=baudrate,
            timeout=0.1
        )
        self.running = False
        
    def start(self):
        self.running = True
        # 启动接收线程
        self.rx_thread = threading.Thread(target=self._receive_loop)
        self.rx_thread.daemon = True
        self.rx_thread.start()
        
    def stop(self):
        self.running = False
        self.ser.close()
        
    def _receive_loop(self):
        """接收线程"""
        while self.running:
            if self.ser.in_waiting > 0:
                try:
                    data = self.ser.readline()
                    if data:
                        print(f"[RX] {data.decode('utf-8', errors='replace').strip()}")
                except Exception as e:
                    print(f"[错误] {e}")
                    
    def send(self, msg):
        """发送数据"""
        if self.ser.is_open:
            self.ser.write((msg + '\n').encode('utf-8'))
            print(f"[TX] {msg}")

# 使用示例
if __name__ == '__main__':
    comm = SerialComm('COM3', 115200)
    comm.start()
    
    try:
        while True:
            msg = input("输入消息（q退出）: ")
            if msg.lower() == 'q':
                break
            comm.send(msg)
    except KeyboardInterrupt:
        pass
    finally:
        comm.stop()
        print("串口已关闭")

5.6 读取传感器数据示例

假设传感器输出格式：TEMP:25.6,HUM:45.2\r\n

import serial
import re
import time

def read_sensor(port='COM3', baudrate=9600):
    ser = serial.Serial(port, baudrate, timeout=2)
    time.sleep(0.5)  # 等待稳定
    
    try:
        while True:
            line = ser.readline()
            if not line:
                continue
                
            try:
                # 解析数据
                text = line.decode('utf-8').strip()
                
                # 正则匹配
                match = re.match(r'TEMP:(\d+\.?\d*),HUM:(\d+\.?\d*)', text)
                if match:
                    temp = float(match.group(1))
                    hum = float(match.group(2))
                    print(f"温度: {temp}°C | 湿度: {hum}%")
                else:
                    print(f"原始数据: {text}")
                    
            except UnicodeDecodeError:
                print(f"HEX: {line.hex()}")
                
    except KeyboardInterrupt:
        print("\n停止读取")
    finally:
        ser.close()

# 运行
read_sensor('COM3', 9600)

6.3 接收数据

方式一：阻塞接收

uint8_t rx_buf[64];
HAL_UART_Receive(&huart1, rx_buf, 10, 1000);  // 超时1000ms
阻塞接收会卡住程序！仅用于调试或简单场景。

方式二：中断接收（推荐）

// 全局变量
uint8_t rx_buf[64];
uint8_t rx_index = 0;
volatile uint8_t data_ready = 0;

// 启动中断接收（在初始化后调用）
HAL_UART_Receive_IT(&huart1, &rx_buf[rx_index], 1);

// 中断回调函数
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
{
    if (huart->Instance == USART1) {
        // 检查是否收到换行符
        if (rx_buf[rx_index] == '\n' || rx_index >= 63) {
            rx_buf[rx_index + 1] = '\0';  // 字符串结束
            data_ready = 1;               // 标记数据就绪
            rx_index = 0;
        } else {
            rx_index++;
        }
        
        // 重新启动中断接收
        HAL_UART_Receive_IT(&huart1, &rx_buf[rx_index], 1);
    }
}

// 主循环处理
while (1) {
    if (data_ready) {
        data_ready = 0;
        printf("Received: %s", rx_buf);
    }
    // 其他任务...
}

6.4 DMA接收（最佳方案）

DMA接收不占用CPU，适合高速、大量数据。

// 使用串口空闲中断 + DMA
#define RX_BUF_SIZE 256

uint8_t rx_buf[RX_BUF_SIZE];
volatile uint16_t rx_len = 0;

// 初始化时启动DMA接收
HAL_UART_Receive_DMA(&huart1, rx_buf, RX_BUF_SIZE);
__HAL_UART_ENABLE_IT(&huart1, UART_IT_IDLE);  // 使能空闲中断

// 在 stm32xxx_it.c 中添加空闲中断处理
void USART1_IRQHandler(void)
{
    if (__HAL_UART_GET_FLAG(&huart1, UART_FLAG_IDLE)) {
        __HAL_UART_CLEAR_IDLEFLAG(&huart1);
        
        // 计算接收长度
        rx_len = RX_BUF_SIZE - huart1.hdmarx->Instance->CNDTR;
        
        // 停止DMA，处理数据
        HAL_UART_DMAStop(&huart1);
        
        // 处理接收到的数据
        printf("DMA RX[%d]: %.*s\r\n", rx_len, rx_len, rx_buf);
        
        // 重新启动DMA接收
        HAL_UART_Receive_DMA(&huart1, rx_buf, RX_BUF_SIZE);
    }
    
    HAL_UART_IRQHandler(&huart1);
}
空闲中断原理

当接收端停止发送数据（总线空闲）时，硬件会触发空闲中断。此时DMA已经接收了所有数据，我们可以知道实际接收的长度。

6.5 完整工程框架

// main.c

#include "usart.h"

extern UART_HandleTypeDef huart1;

// 接收缓冲区

uint8_t rx_buf[RX_BUF_SIZE];
volatile uint8_t cmd_ready = 0;

void UART_ProcessCommand(uint8_t *buf, uint16_t len)
{
    // 解析命令
    if (strncmp((char*)buf, "LED ON", 6) == 0) {
        HAL_GPIO_WritePin(LED_GPIO_Port, LED_Pin, GPIO_PIN_SET);

        HAL_GPIO_WritePin(LED_GPIO_Port, LED_Pin, GPIO_PIN_RESET);
        printf("LED已关闭\r\n");
    }
    else {

    }
}

}

7. 串口调试技巧

7.1 HEX模式调试

当遇到乱码或二进制数据时，用十六进制查看最直观：

import serial

ser = serial.Serial('COM3', 115200)

while True:
    if ser.in_waiting:
        data = ser.read(ser.in_waiting)
        # 同时显示HEX和ASCII
        hex_str = ' '.join(f'{b:02X}' for b in data)
        ascii_str = ''.join(chr(b) if 32 <= b < 127 else '.' for b in data)
        print(f"HEX: {hex_str}")
        print(f"ASC: {ascii_str}")
常用调试软件：SSCOM、串口调试助手、RealTerm 都支持HEX显示。

    print(f"[{ts}] {msg}")

import serial
from datetime import datetime

log_file = open(f"serial_log_{datetime.now():%Y%m%d_%H%M%S}.txt", 'w', encoding='utf-8')

def log(msg):
    print(msg)
    log_file.write(msg + '\n')
    log_file.flush()

# 使用
log("开始记录...")

# 结束时关闭
log_file.close()

7.4 常见乱码原因

现象 可能原因 解决方案
全部乱码 波特率不匹配 确认双方波特率一致
部分乱码 编码问题 尝试UTF-8/GBK切换
每2字节错1个 TX/RX接反 交叉接线
数据丢失 波特率太高或缓冲溢出 降低波特率或增加缓冲
随机乱码 电平不匹配或干扰 检查电压、加下拉电阻

8. 常见问题FAQ

Q1: 设备管理器中找不到COM口？

排查步骤
  1. 检查物理连接：USB线是否插好？LED是否亮？
  2. 更换USB口：尝试不同的USB口
  3. 检查驱动：设备管理器是否有"未知设备"？
  4. 重装驱动：卸载后重新安装驱动
  5. 更换USB线：有些线只能充电不能传输数据！
  6. 更换电脑测试：排除模块损坏的可能

Q2: Python报"Permission denied"？

原因：串口被其他程序占用！

解决：

  • 关闭PuTTY、串口调试助手等软件
  • 检查是否有其他Python脚本在运行
  • Windows任务管理器查找占用COM口的进程

Q3: 波特率不匹配会怎样？

波特率不匹配的表现
正确波特率(115200):
Hello World!

错误波特率(9600):
àààààààààààà

完全无法辨认的乱码通常是波特率问题。

Q4: 数据丢失或粘包？

粘包：发送方快速发送"ABC"和"DEF"，接收方可能一次性收到"ABCDEF"，无法区分两条消息。

解决方案：

  • 协议设计：使用帧头+长度+数据+校验的格式
  • 分隔符：每条消息以\r\n结尾
  • 超时机制：空闲中断检测帧结束
  • 定长发送：每条消息固定长度
// 推荐的帧格式
// [0xAA][0x55][LEN][DATA...][CHECKSUM]
// 帧头(2B) + 长度(1B) + 数据(NB) + 校验(1B)

typedef struct {
    uint8_t header[2];   // 0xAA, 0x55
    uint8_t length;      // 数据长度
    uint8_t data[32];    // 数据区
    uint8_t checksum;    // 校验和
} SerialFrame;
