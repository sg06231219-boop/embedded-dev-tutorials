🌟 STM32 LED跑马灯 + 按键控制

📋 学习目标：掌握GPIO输出控制LED、GPIO输入读取按键、外部中断的使用，以及软件/硬件两种消抖方法。

一、硬件接线图

1.1 元器件清单

元器件数量备注
STM32F103C8T6最小系统板1块蓝色小板
LED（红色/绿色）4个3mm或5mm均可
轻触按键1个4脚带弹簧
限流电阻4个330Ω~1kΩ，建议470Ω
上拉电阻1个10kΩ（外接）
面包板+跳线若干方便接线

1.2 接线示意

        STM32F103C8T6 最小系统板
        ┌─────────────────────┐
        │                     │
        │   ┌─────────────┐   │
        │   │   MCU芯片   │   │
        │   └─────────────┘   │
        │                     │
        │  PA0 ──────► LED1  │  (通过470Ω限流电阻)
        │  PA1 ──────► LED2  │  (通过470Ω限流电阻)
        │  PA2 ──────► LED3  │  (通过470Ω限流电阻)
        │  PA3 ──────► LED4  │  (通过470Ω限流电阻)
        │                     │
        │  PA8 ◄─── 按键 ─┤  │  (按键另一端接地GND)
        │           └──10kΩ─┘  │  (PA8内置上拉，或外接10kΩ到3.3V)
        │                     │
        │  3.3V ───► VCC      │  (所有LED和上拉电阻共正极)
        │  GND  ───► GND      │  (共地)
        └─────────────────────┘

【灌电流接法说明】
  MCU输出高电平"1"时 → LED熄灭
  MCU输出低电平"0"时 → LED点亮（电流从VCC→LED→MCU流入地）

【按键接线】
  按键未按下 → PA8检测到高电平(3.3V，经上拉)
  按键按下   → PA8检测到低电平(0V，对地短路)
💡 灌电流接法：LED正极接VCC，负极通过限流电阻接MCU引脚。MCU引脚输出低电平时电流从VCC流入MCU，这种方式比"拉电流"更稳定，是推荐接法。

二、CubeMX配置步骤

2.1 新建工程

  1. 打开STM32CubeMX，点击 File → New Project
  2. 在搜索框输入 STM32F103C8T6，选择对应芯片
  3. 点击Start Project开始配置

2.2 配置时钟（RCC）

  1. 在Pinout view左侧，点击 RCC
  2. High Speed Clock (HSE) 选择 Crystal/Ceramic Resonator
  3. Low Speed Clock (LSE) 选择 Crystal/Ceramic Resonator（可选）

2.3 配置GPIO输出（LED）

  1. 点击芯片上的 PA0 引脚
  2. 选择 GPIO_Output
  3. 同样方式配置 PA1、PA2、PA3
  4. 在左侧GPIO标签页中： 
    PA0 → Output Push Pull, High (初始高电平-灯灭)
PA1 → Output Push Pull, High
PA2 → Output Push Pull, High
PA3 → Output Push Pull, High

2.4 配置GPIO输入（按键）

  1. 点击芯片上的 PA8 引脚
  2. 选择 GPIO_Input
  3. 在左侧GPIO标签页中： 
    PA8 → GPIO Pull-up (上拉电阻启用)

2.5 配置外部中断

  1. 点击芯片上的 PA8 引脚
  2. 选择 GPIO_EXTI8
  3. 在左侧GPIO标签页中配置： 
    PA8 → GPIO_EXTI8
       → Trigger: Both Edges (上升沿和下降沿都触发)
               或: Falling Edge (仅下降沿)

2.6 配置定时器TIM2（用于硬件消抖）

  1. 在左侧Timers → TIM2
  2. 勾选 Activated
  3. Configuration标签： 
    Prescaler = 72-1    (72MHz / 72 = 1MHz，计数一次=1us)
Counter Period = 10000-1  (10ms定时)
Trigger Output: None
  4. 点击NVIC Settings，勾选TIM2 interrupt

2.7 配置时钟树

  1. 点击左侧Clock Configuration
  2. HSE = 8MHz（根据你的晶振）
  3. PLL锁相环配置：HSE × 9 = 72MHz
  4. APB1 = 36MHz，APB2 = 72MHz

2.8 生成代码

  1. 点击Project Manager
  2. 填写项目名称：led_button_demo
  3. 选择保存路径
  4. Toolchain/IDE 选择 MDK-ARM（Keil）
  5. 点击 GENERATE CODE生成代码

/**
  * @brief  主函数
  * @param  None
  * @retval None
  */
int main(void)
{
  /* 初始化HAL库 */
  HAL_Init();

  /* 配置系统时钟 */
  SystemClock_Config();

  /* 初始化所有外设 */
  MX_GPIO_Init();
  MX_TIM2_Init();

  /* 启动TIM2用于硬件消抖定时 */
  HAL_TIM_Base_Start_IT(&htim2);

  /* 初始化所有LED为熄灭状态 */
  LED_All_Off();

  /* 无限循环主程序 */
  while (1)
  {
    /* 方法1：软件延时跑马灯（CPU空转方式） */
    static uint32_t last_led_time = 0;
    static uint8_t current_led = 0;

    if (HAL_GetTick() - last_led_time >= 200)  // 200ms切换一次
    {
      last_led_time = HAL_GetTick();

      /* 熄灭所有LED */
      LED_All_Off();

      /* 点亮当前LED */
      switch (current_led)
      {
        case 0: HAL_GPIO_WritePin(LED1_GPIO_Port, LED1_Pin, GPIO_PIN_RESET); break;  // 低电平点亮
        case 1: HAL_GPIO_WritePin(LED2_GPIO_Port, LED2_Pin, GPIO_PIN_RESET); break;
        case 2: HAL_GPIO_WritePin(LED3_GPIO_Port, LED3_Pin, GPIO_PIN_RESET); break;
        case 3: HAL_GPIO_WritePin(LED4_GPIO_Port, LED4_Pin, GPIO_PIN_RESET); break;
        default: break;
      }

      /* 移动到下一个LED */
      current_led++;
      if (current_led >= 4)
      {
        current_led = 0;
      }
    }

    /* 方法2：如果按键被按下，执行相应操作 */
    if (key_pressed_flag == 1)
    {
      key_pressed_flag = 0;  // 清除标志

      /* 按键按下后，所有灯闪烁3次 */
      for (int i = 0; i < 3; i++)
      {
        LED_All_On();
        HAL_Delay(200);
        LED_All_Off();
        HAL_Delay(200);
      }
    }

    /* 方法3：查询方式读取按键（带软件消抖） */
    static uint8_t key_state = 1;      // 1=未按下, 0=按下
    static uint8_t last_key_state = 1;
    static uint32_t key_last_change_time = 0;

    uint8_t current_key = HAL_GPIO_ReadPin(KEY_GPIO_Port, KEY_Pin);

    if (current_key != last_key_state)
    {
      /* 按键状态发生变化，记录时间 */
      key_last_change_time = HAL_GetTick();
      last_key_state = current_key;
    }

    /* 等待消抖时间后，确认按键状态 */
    if ((HAL_GetTick() - key_last_change_time) > DEBOUNCE_TIME_MS)
    {
      if (current_key != key_state)
      {
        key_state = current_key;

        /* 按键释放时（从按下变为未按下）执行操作 */
        if (key_state == 1)  // 释放
        {
          /* 反向流水灯方向 */
          led_pattern = !led_pattern;
        }
      }
    }

    /* 正常情况下不做任何阻塞操作，让CPU处理其他事务 */
  }
}

/**
  * @brief  系统时钟配置
  * @param  None
  * @retval None
  */
void SystemClock_Config(void)
{
  RCC_OscInitTypeDef RCC_OscInitStruct = {0};
  RCC_ClkInitTypeDef RCC_ClkInitStruct = {0};

  /* 初始化RCC振荡器 */
  RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSE;
  RCC_OscInitStruct.HSEState = RCC_HSE_ON;
  RCC_OscInitStruct.HSEPredivValue = RCC_HSE_PREDIV_DIV1;
  RCC_OscInitStruct.PLL.PLLState = RCC_PLL_ON;
  RCC_OscInitStruct.PLL.PLLSource = RCC_PLLSOURCE_HSE;
  RCC_OscInitStruct.PLL.PLLMUL = RCC_PLL_MUL9;
  if (HAL_RCC_OscConfig(&RCC_OscInitStruct) != HAL_OK)
  {
    Error_Handler();
  }

  /* 初始化CPU、AHB和APB时钟 */
  RCC_ClkInitStruct.ClockType = RCC_CLOCKTYPE_HCLK | RCC_CLOCKTYPE_SYSCLK
                              | RCC_CLOCKTYPE_PCLK1 | RCC_CLOCKTYPE_PCLK2;
  RCC_ClkInitStruct.SYSCLKSource = RCC_SYSCLKSOURCE_PLLCLK;
  RCC_ClkInitStruct.AHBCLKDivider = RCC_SYSCLK_DIV1;
  RCC_ClkInitStruct.APB1CLKDivider = RCC_HCLK_DIV2;
  RCC_ClkInitStruct.APB2CLKDivider = RCC_HCLK_DIV1;

  if (HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_2) != HAL_OK)
  {
    Error_Handler();
  }
}

/**
  * @brief  TIM2中断回调函数（硬件消抖用）
  * @param  htim: 定时器句柄
  * @retval None
  */
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
  if (htim->Instance == TIM2)
  {
    /* 硬件定时器消抖处理 - 可以在这里检查按键状态 */
    /* 注意：由于TIM2中断频率较高，这里只设置一个标志 */
    /* 实际的消抖逻辑可以在主循环中处理 */

    /* 示例：读取按键并更新状态（带硬件定时器消抖） */
    static uint8_t debounce_counter = 0;
    static uint8_t stable_key_state = 1;

    uint8_t raw_key = HAL_GPIO_ReadPin(KEY_GPIO_Port, KEY_Pin);

    if (raw_key != stable_key_state)
    {
      debounce_counter++;
      if (debounce_counter >= (HARDWARE_DEBOUNCE_MS))  // 约50ms消抖
      {
        stable_key_state = raw_key;
        debounce_counter = 0;

        /* 按键按下（下降沿） */
        if (stable_key_state == 0)
        {
          key_pressed_flag = 1;
        }
      }
    }
    else
    {
      debounce_counter = 0;
    }
  }
}

/**
  * @brief  软件消抖函数（延时法）
  * @param  None
  * @retval None
  */
void Key_Software_Debounce(void)
{
  /* 读取按键状态 */
  uint8_t key_value = HAL_GPIO_ReadPin(KEY_GPIO_Port, KEY_Pin);

  if (key_value == 0)  // 按键按下
  {
    HAL_Delay(DEBOUNCE_TIME_MS);  // 延时50ms消抖

    /* 再次读取确认 */
    if (HAL_GPIO_ReadPin(KEY_GPIO_Port, KEY_Pin) == 0)
    {
      /* 确认按键确实按下 */
      key_pressed_flag = 1;
    }
  }
}

/**
  * @brief  TIM2微秒延时（使用定时器实现精确延时）
  * @param  us: 延时微秒数
  * @retval None
  */
void TIM2_Delay_us(uint32_t us)
{
  /* TIM2配置：72分频，计数频率1MHz */
  /* 每次计数 = 1us */
  htim2.Instance->CNT = 0;
  htim2.Instance->ARR = us;
  htim2.Instance->EGR = 0x0001;  // 产生更新事件，重置计数器

  /* 启动定时器 */
  __HAL_TIM_ENABLE(&htim2);

  /* 等待计数完成 */
  while (__HAL_TIM_GET_FLAG(&htim2, TIM_FLAG_UPDATE) == RESET);

  /* 清除标志 */
  __HAL_TIM_CLEAR_FLAG(&htim2, TIM_FLAG_UPDATE);

  /* 停止定时器 */
  __HAL_TIM_DISABLE(&htim2);
}

4.4 软件消抖 vs 硬件消抖

消抖方法原理优点缺点
软件延时消抖检测到变化后延时50ms再确认简单直接阻塞CPU，响应慢
时间戳消抖记录变化时间，主循环判断时间差不阻塞需要在主循环处理
定时器消抖定时器周期性检查，稳定后才确认精确，非阻塞占用定时器资源

五、常见问题排查

5.1 LED不亮

可能原因1：GPIO模式配置错误 
❌ 错误配置（GPIO_MODE_INPUT 输入模式）
GPIO_InitStruct.Mode = GPIO_MODE_INPUT;

✅ 正确配置（推挽输出）
GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;
可能原因2：使用了开漏模式 
❌ 开漏模式无法输出高电平点亮LED
GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_OD;  // 开漏

✅ 推挽模式可以输出高低电平
GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;   // 推挽
可能原因3：限流电阻太小导致引脚损坏 
正确阻值选择（LED压降约2V，电流5-10mA）：
  I = (3.3V - 2V) / R
  R = 1.3V / 0.01A = 130Ω  （最小值）

推荐：
  330Ω → 4mA（较暗但安全）
  470Ω → 2.8mA（推荐亮度）
  1kΩ  → 1.3mA（较暗）

5.2 按键抖动波形分析

示波器观察到的按键抖动示意图：

按下去时:     释放时:
      ___           ___
     /   \         /   \
____/     \_______/     \____
   10-20ms抖动   10-20ms抖动

消抖策略：
1. 检测到下降沿后，等待 T = 50ms
2. 如果仍然是低电平 → 确认按下
3. 如果变成高电平   → 认为是抖动，忽略

5.3 中断回调函数名拼写错误

常见错误：函数名写错导致中断不响应 
❌ 错误写法（中断不会触发）
void EXTI9_5_IRQHandler(void)
{
    My_Own_Callback();  // 这个函数名不会被自动调用
}

❌ 错误写法（函数名拼写错误）
void HAL_GPIO_EXTI8_Callback(uint16_t GPIO_Pin)  // 多了数字8

❌ 错误写法（漏掉了HAL_前缀）
void GPIO_EXTI_Callback(uint16_t GPIO_Pin)

✅ 正确写法
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin)
{
    if (GPIO_Pin == GPIO_PIN_8)
    {
        // 处理中断
    }
}

5.4 中断服务函数与回调函数的区别

┌─────────────────────────────────────────────────────────┐
│                    中断处理流程                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  硬件触发中断 ──► ISR函数 ──► 调用回调函数              │
│                                                         │
│  EXTI9_5_IRQHandler() ──► HAL_GPIO_EXTI_Callback()     │
│  (汇编入口，固定名称)      (HAL库标准回调)              │
│                                                         │
│  用户只修改回调函数！ISR由HAL库处理！                   │
│                                                         │
└─────────────────────────────────────────────────────────┘

5.5 按键按下/松开判断

/* 场景：检测按键按下（下降沿） */
if (HAL_GPIO_ReadPin(KEY_GPIO_Port, KEY_Pin) == GPIO_PIN_RESET)
{
    // 按键按下
}

/* 场景：检测按键松开（上升沿） */
if (HAL_GPIO_ReadPin(KEY_GPIO_Port, KEY_Pin) == GPIO_PIN_SET)
{
    // 按键松开
}

/* 场景：在中断中判断是按下还是松开 */
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin)
{
    uint8_t level = HAL_GPIO_ReadPin(KEY_GPIO_Port, GPIO_Pin);

    if (level == GPIO_PIN_RESET)
    {
        /* 下降沿 - 按下 */
    }
    else
    {
        /* 上升沿 - 松开 */
    }
}

❓ 常见问题 FAQ

Q: LED亮度不一样？

A: 不同颜色LED正向压降不同（红2V/蓝3V），需要不同限流电阻。简单做法：统一用1kΩ电阻。

Q: 按键抖动怎么办？

A: 硬件：0.1μF电容滤波。软件：延时10~20ms消抖。推荐：定时器扫描+状态机消抖（不阻塞）。
