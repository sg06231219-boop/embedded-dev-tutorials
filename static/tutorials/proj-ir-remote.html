🔬 NEC 红外协议详解

NEC协议帧结构

NEC协议一次传输32位数据（4字节），结构如下：

NEC 红外协议 — 单帧结构

┌──────────┬────────┬────────┬────────┬────────┐
│ 引导码   │ 地址码  │ 地址反码│ 数据码  │ 数据反码│
│ (9ms+4.5ms)│ 8bit   │ 8bit   │ 8bit   │ 8bit   │
└──────────┴────────┴────────┴────────┴────────┘
  引导码部分     数据部分（16bit地址 + 16bit数据）

引导码详解：
  9ms低电平 → 4.5ms高电平（起始位）
  9ms低电平 → 2.5ms高电平 = 重复码（按住不放时发送）

数据位编码（逻辑0和逻辑1）：
  逻辑0:  560μs低 + 560μs高   = 1.125ms
  逻辑1:  560μs低 + 1690μs高  = 2.25ms

完整一帧 = 9ms + 4.5ms + 32×1.125ms(平均) ≈ 67.5ms

引导码与数据位时序图

引导码（Init）：
    ─────┐          ┌──────────
    9ms  │ 4.5ms   │ 4.5ms   │
         └──────────┘          ← 高电平

逻辑0：
         ┌──┐     ┌──┐
    560μs└──┘560μs└──┘

逻辑1：
         ┌──┐           ┌──────┐
    560μs└──┘  1690μs  └──────┘

接线注意：
- VS1838B的OUT引脚输出的是解调后的信号（38kHz载波被移除）
- 模块输出低电平表示收到了红外信号（空闲时为高电平）
- 确保STM32的EXTI中断已使能

📝 分步操作指引

Step 1：CubeMX 配置
1. 配置PA0为GPIO_EXTI0（下降沿/上升沿触发）
2. 配置TIM2为输入捕获（用于测脉宽）或仅用作计数器
3. 配置PC13/PC14/PC15为GPIO_Output（LED）
4. 配置USART1用于打印解码结果
5. 使能EXTI0中断
Step 2：理解测距原理
利用外部中断检测引脚电平变化，配合定时器计数器测量两次中断间的时间差。
Step 3：编写状态机解码
按照"引导码→地址码→数据码"的状态顺序，逐位解析红外信号。
Step 4：打印键值
先不绑定功能，只打印解码出的键值，确认能正常工作。
Step 5：绑定功能
根据打印出的键值，绑定LED、电机等控制逻辑。

/* USER CODE BEGIN Includes */
#include <stdio.h>

/* USER CODE BEGIN PV */

// NEC 解码状态
typedef enum {
    IR_STATE_IDLE = 0,      // 空闲，等待引导码
    IR_STATE_LEADER_LOW,     // 引导码低电平
    IR_STATE_LEADER_HIGH,    // 引导码高电平
    IR_STATE_DATA_LOW,       // 数据位低电平
    IR_STATE_DATA_HIGH,      // 数据位高电平
    IR_STATE_DONE            // 解码完成
} IR_State_t;

// 解码结果
typedef struct {
    uint8_t address;    // 地址码
    uint8_t addr_inv;   // 地址反码
    uint8_t command;   // 命令码
    uint8_t cmd_inv;    // 命令反码
    uint8_t valid;      // 校验标志
} IR_Data_t;

// 全局变量
volatile IR_State_t g_ir_state = IR_STATE_IDLE;
volatile IR_Data_t g_ir_data;
volatile uint32_t g_ir_raw_data = 0;   // 32bit原始数据
volatile uint8_t g_ir_bit_index = 0;   // 当前解析的bit位
volatile uint16_t g_last_tick = 0;     // 上次中断的计数器值

// 定时器计数器（需要配置TIM2或TIM3）
extern TIM_HandleTypeDef htim2;

// NEC 协议时间参数（单位：μs）
#define NEC_LEADER_LOW_MIN  8500   // 引导码低电平最小 9ms - 10%
#define NEC_LEADER_LOW_MAX  9900   // 引导码低电平最大 9ms + 10%
#define NEC_LEADER_HIGH_MIN 4000   // 引导码高电平最小 4.5ms - 10%
#define NEC_LEADER_HIGH_MAX 5400   // 引导码高电平最大 4.5ms + 10%
#define NEC_REPEAT_HIGH_MIN 2000   // 重复码高电平最小
#define NEC_REPEAT_HIGH_MAX 2500   // 重复码高电平最大
#define NEC_BIT_LOW_MIN     400    // 数据低电平最小
#define NEC_BIT_LOW_MAX     800    // 数据低电平最大
#define NEC_BIT0_HIGH_MIN   400    // 逻辑0高电平最小
#define NEC_BIT0_HIGH_MAX   800    // 逻辑0高电平最大
#define NEC_BIT1_HIGH_MIN   1400   // 逻辑1高电平最小
#define NEC_BIT1_HIGH_MAX   2000   // 逻辑1高电平最大

// 遥控器键值映射（根据你的遥控器修改）
// 先用串口打印command值，然后在此表中添加对应键名

    {0x0C, "1"},
    {0x18, "2"},

    {0x1C, "5"},

// 获取按键名称
const char* IR_GetKeyName(uint8_t code)
{
    for (uint8_t i = 0; i < sizeof(g_key_map)/sizeof(IR_KeyMap_t); i++) {
        if (g_key_map[i].key_code == code) {
            return g_key_map[i].key_name;
        }
    }
    return "UNKNOWN";
}

// 获取当前定时器计数（微秒）
static inline uint16_t Get_Timer_US(void)
{
    return (uint16_t)__HAL_TIM_GET_COUNTER(&htim2);
}

// 重置定时器
static inline void Reset_Timer(void)
{
    __HAL_TIM_SET_COUNTER(&htim2, 0);
}

// 外部中断回调 — 核心解码逻辑
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin)
{
    if (GPIO_Pin != GPIO_PIN_0) return;  // 只处理PA0
    
    uint16_t now = Get_Timer_US();
    uint16_t duration = now - g_last_tick;
    g_last_tick = now;
    
    GPIO_PinState pin_state = HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_0);
    
    switch (g_ir_state) {
        
        case IR_STATE_IDLE:
            if (pin_state == GPIO_PIN_LOW) {
                // 引导码低电平开始
                Reset_Timer();
                g_ir_state = IR_STATE_LEADER_LOW;
            }
            break;
            
        case IR_STATE_LEADER_LOW:
            if (pin_state == GPIO_PIN_HIGH) {
                // 引导码低电平结束，检查时间
                if (duration >= NEC_LEADER_LOW_MIN && duration <= NEC_LEADER_LOW_MAX) {
                    Reset_Timer();
                    g_ir_state = IR_STATE_LEADER_HIGH;
                } else {
                    g_ir_state = IR_STATE_IDLE; // 超时，重新检测
                }
            }
            break;
            
        case IR_STATE_LEADER_HIGH:
            if (pin_state == GPIO_PIN_LOW) {
                // 引导码高电平结束
                if (duration >= NEC_LEADER_HIGH_MIN && duration <= NEC_LEADER_HIGH_MAX) {
                    // 有效引导码，开始接收数据
                    Reset_Timer();
                    g_ir_bit_index = 0;
                    g_ir_raw_data = 0;
                    g_ir_state = IR_STATE_DATA_LOW;
                } else if (duration >= NEC_REPEAT_HIGH_MIN && duration <= NEC_REPEAT_HIGH_MAX) {
                    // 重复码（按住不放）
                    g_ir_state = IR_STATE_IDLE;
                    // 可以在此处理重复码
                } else {
                    g_ir_state = IR_STATE_IDLE;
                }
            }
            break;
            
        case IR_STATE_DATA_LOW:
            if (pin_state == GPIO_PIN_HIGH) {
                // 数据位低电平结束
                if (duration < NEC_BIT_LOW_MIN || duration > NEC_BIT_LOW_MAX) {
                    g_ir_state = IR_STATE_IDLE; // 无效低电平
                } else {
                    Reset_Timer();
                    g_ir_state = IR_STATE_DATA_HIGH;
                }
            }
            break;
            
        case IR_STATE_DATA_HIGH:
            if (pin_state == GPIO_PIN_LOW) {
                // 数据位高电平结束，判断是0还是1
                if (duration >= NEC_BIT0_HIGH_MIN && duration <= NEC_BIT0_HIGH_MAX) {
                    // 逻辑0，什么都不用做（g_ir_raw_data默认0）
                } else if (duration >= NEC_BIT1_HIGH_MIN && duration <= NEC_BIT1_HIGH_MAX) {
                    // 逻辑1，设置对应位
                    g_ir_raw_data |= (1UL << g_ir_bit_index);
                } else {
                    g_ir_state = IR_STATE_IDLE;
                    break;
                }
                
                g_ir_bit_index++;
                Reset_Timer();
                
                if (g_ir_bit_index >= 32) {
                    // 32位接收完毕
                    g_ir_state = IR_STATE_DONE;
                } else {
                    g_ir_state = IR_STATE_DATA_LOW;
                }
            }
            break;
            
        case IR_STATE_DONE:
            // 解码完成（主循环会处理）
            g_ir_state = IR_STATE_IDLE;
            break;
            
        default:
            g_ir_state = IR_STATE_IDLE;
            break;
    }
}

// 处理解码结果
void IR_ProcessResult(void)
{
    if (g_ir_state == IR_STATE_DONE) {
        g_ir_state = IR_STATE_IDLE;
        
        // 解析32bit数据
        g_ir_data.address   = (g_ir_raw_data >> 0) & 0xFF;
        g_ir_data.addr_inv  = (g_ir_raw_data >> 8) & 0xFF;
        g_ir_data.command   = (g_ir_raw_data >> 16) & 0xFF;
        g_ir_data.cmd_inv   = (g_ir_raw_data >> 24) & 0xFF;
        
        // 校验地址码
        g_ir_data.valid = (g_ir_data.address == (uint8_t)~g_ir_data.addr_inv)
                       && (g_ir_data.command == (uint8_t)~g_ir_data.cmd_inv);
        
        // 串口打印
        printf("IR: Addr=0x%02X, Cmd=0x%02X [%s] %s\r\n",
               g_ir_data.address,
               g_ir_data.command,
               IR_GetKeyName(g_ir_data.command),
               g_ir_data.valid ? "OK" : "ERR");
        
        // 执行功能
        IR_ExecCommand(g_ir_data.command);
    }
}

// 根据键值执行对应功能
void IR_ExecCommand(uint8_t cmd)
{
    switch(cmd) {
        case 0x0C: // 1 - 红色LED
            HAL_GPIO_TogglePin(GPIOC, GPIO_PIN_13);
            break;
        case 0x18: // 2 - 绿色LED
            HAL_GPIO_TogglePin(GPIOC, GPIO_PIN_14);
            break;
        case 0x5E: // 3 - 蓝色LED
            HAL_GPIO_TogglePin(GPIOC, GPIO_PIN_15);
            break;
        case 0x08: // 4 - 全灭
            HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13|GPIO_PIN_14|GPIO_PIN_15, GPIO_PIN_SET);
            break;
        case 0x52: // 8 - 全亮
            HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13|GPIO_PIN_14|GPIO_PIN_15, GPIO_PIN_RESET);
            break;
        case 0x46: // CH - 彩虹灯效果
            // 简单的RGB循环效果
            static uint8_t rgb_step = 0;
            rgb_step = (rgb_step + 1) % 6;
            HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13|GPIO_PIN_14|GPIO_PIN_15, GPIO_PIN_SET);
            switch(rgb_step) {
                case 0: HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13, GPIO_PIN_RESET); break;
                case 1: HAL_GPIO_WritePin(GPIOC, GPIO_PIN_14, GPIO_PIN_RESET); break;
                case 2: HAL_GPIO_WritePin(GPIOC, GPIO_PIN_15, GPIO_PIN_RESET); break;
                case 3: HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13|GPIO_PIN_14, GPIO_PIN_RESET); break;
                case 4: HAL_GPIO_WritePin(GPIOC, GPIO_PIN_14|GPIO_PIN_15, GPIO_PIN_RESET); break;
                case 5: HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13|GPIO_PIN_15, GPIO_PIN_RESET); break;
            }
            break;
    }
}

{
    HAL_Init();
    SystemClock_Config();

    HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13|GPIO_PIN_14|GPIO_PIN_15, GPIO_PIN_SET);
    
    printf("IR Remote Decoder Ready!\r\n");
    printf("Press any button on the remote...\r\n");
    printf("Keys: 1=R LED, 2=G LED, 3=B LED, 4=All OFF, 8=All ON, CH=Rainbow\r\n");
    /* USER CODE END 2 */

    /* USER CODE BEGIN WHILE */
    while (1)
    {
        // 处理解码结果（在主循环中处理，避免在中断中做耗时操作）
        IR_ProcessResult();
        
        HAL_Delay(10);
    }
    /* USER CODE END WHILE */
}

CubeMX 配置说明

TIM2 配置（用于测脉宽）：
  Prescaler: 72-1    → 1MHz (1μs/计数)
  Counter Period: 65535 (最大, 用uint16_t)
  用途: __HAL_TIM_GET_COUNTER(&htim2) 获取当前计数

GPIO PA0 配置：
  Mode: External Interrupt Mode with Falling/Rising edge trigger
  (CubeMX会自动配置AFIO和NVIC)

NVIC 设置：
  Enable EXTI line[0] interrupt
  Preemption Priority: 0 (最高优先级，确保及时响应)

❓ 常见问题排查

问题1：解码失败，串口一直打印乱码

可能原因：
1. 脉宽容差设置太严格 — NEC协议允许±10%的误差，如果MCU主频不对或定时器频率不准，判断会出错。
2. 定时器PSC设置错误 — PSC=72时，TIM2计数频率=72MHz/72=1MHz，即每微秒计数+1。如果PSC不是72，脉宽计算会完全错误。
3. 中断优先级不够 — 确保EXTI0中断优先级较高。
4. 使用了错误的定时器 — 确认在回调中使用的htim2句柄与CubeMX配置的一致。

问题2：按一下遥控器，串口打印多次相同键值

这是正常现象！遥控器按一次会发送多帧数据（引导码+若干重复码）。
解决方案：
// 方法1：按键去抖（推荐）
static uint32_t last_key_time = 0;
static uint8_t last_key = 0;
if (cmd != last_key || (now - last_key_time) > 300) {
    last_key = cmd;
    last_key_time = now;
    IR_ExecCommand(cmd);  // 执行功能
}

// 方法2：忽略重复码
// 状态机中添加对重复码的处理，重复码时只打印不执行

问题3：遥控器键值和预设不匹配

每个遥控器的键值不同！需要先打印学习：
1. 先注释掉 IR_ExecCommand() 的调用
2. 上传程序，按遥控器每个按键
3. 串口会打印每个键的十六进制码（如 0x45）
4. 记录下来后，填入 g_key_map 表
5. 再放开 IR_ExecCommand() 的功能调用

问题4：中断回调里不能做耗时操作

绝对禁止在 HAL_GPIO_EXTI_Callback 中：
- 调用 printf / HAL_UART_Transmit
- 使用 HAL_Delay
- 进行复杂的字符串处理
- 访问文件系统

正确做法：回调中只设置标志位或更新全局变量，主循环中处理业务逻辑。

问题5：VS1838B没有输出

1. 检查VCC和GND是否接反（有些模块有防反接二极管，接反不工作）
2. 用手机摄像头对准遥控器红外LED，按键时能在屏幕上看到红外光（发光）
3. 示波器测量VS1838B OUT脚，按遥控器应有脉冲输出
4. 确认遥控器是NEC协议（常见电视遥控器通常都是）
5. 遥控器和接收头距离不要太远（1~3米最佳）

NEC协议细节（最常用）

NEC帧结构:
┌──────┐┌───┐  ┌─┐┌─┐┌─┐          ┌─┐┌─┐┌─┐┌─┐
│9ms   ││4.5│  │ ││ ││ │   ...    │ ││ ││ ││ │
│引导码││ms │  │0││1││1│          │0││0││1││0│
└──────┘└───┘  └─┘└─┘└─┘          └─┘└─┘└─┘└─┘
  Leader     地址(8bit) 地址反(8bit) 命令(8bit) 命令反(8bit)
             总32bit数据 + 1bit Stop

逻辑0: 562.5μs载波 + 562.5μs空闲 = 1.125ms
逻辑1: 562.5μs载波 + 1687.5μs空闲 = 2.25ms
重复码: 9ms载波 + 2.25ms空闲 + 562.5μs载波(按住不放时每110ms发送)

// NEC解码的容差范围(避免因晶振误差导致解码失败)
#define NEC_PULSE    560   // μs
#define NEC_TOLERANCE 0.30  // 30%容差
#define NEC_LEADER_H 9000

void IR_Replay(IR_Code* code) {
    for (int i = 0; i < code->len; i++) {
        IR_SendCarrier(code->pulses[i]);  // 38kHz载波开
        if (++i < code->len)
            IR_NoCarrier(code->pulses[i]);  // 载波关
    }

提升红外距离的实用技巧：
1. 用PWM+三极管驱动(不要GPIO直驱) → 距离翻3倍
2. 占空比用33%(1/3载波)而非50% → 峰值电流更大但RMS不过热
3. 接收端加红外滤光片 → 减少环境光干扰 → 有效距离提升
4. 38kHz载波用TIM硬件生成 → 比bit-bang准确、不占CPU

🔐 红外通信安全与加密

红外遥控信号是明文的，任何人都可以用接收头捕获。对于门禁、保险箱等安全应用：
- 滚动码(Rolling Code)：每次按发送递增计数器+加密，防止重放攻击
- 双向认证：设备收到命令后发送随机挑战码，遥控器计算响应
- 时间戳+签名：防止信号被录制后重放

❓ 常见问题 FAQ

Q: 解码不稳定？

A: 1)用输入捕获不要轮询 2)定时器时钟精度够高(1MHz以上) 3)加状态机容错 4)屏蔽重复码

Q: 不同遥控器协议不同？

A: 最常见NEC协议。还有Sony SIRC、Philips RC-5。一般只支持NEC就够了。
