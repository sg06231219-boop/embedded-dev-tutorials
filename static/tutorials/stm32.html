1. STM32F103C8T6 芯片概览

1.1 先认识这块芯片

STM32F103C8T6 是意法半导体（STMicroelectronics）推出的 32 位 ARM Cortex-M3 微控制器，"C8T6" 这几个字符包含了它的关键信息：

  • C = Cortex-M 内核
  • 8 = 64KB Flash 内存（对应 8×8KB，经典蓝色小板用这个）
  • T = LQFP 封装（48 引脚）
  • 6 = 工业级温度范围（-40~85°C）
想象一块芯片就是一个微型"电脑主板"，CPU 是大脑（72MHz），内存条是 Flash（64KB），内存是 RAM（20KB），各种外设就像 USB、网口、摄像头接口等——只不过这一切都集成在指甲盖大小的芯片里。

1.3 内存映射

ARM Cortex-M3 使用统一的 4GB 地址空间（32 位地址总线），地址分配如下：

区域地址范围大小说明
Code 区0x0000 0000 ~ 0x1FFF FFFF512MBFlash 起始于 0x0800 0000
SRAM 区0x2000 0000 ~ 0x3FFF FFFF512MBRAM 起始于 0x2000 0000（20KB）
外设区0x4000 0000 ~ 0x5FFF FFFF512MBGPIO、UART、TIM、ADC 等寄存器
内核外设0xE000 0000 ~ 0xE00F FFFF1MBNVIC、SCB、SysTick（仅内核使用）
// 常见寄存器地址（STM32F103）
#define FLASH_BASE     0x08000000UL  // Flash 起始地址
#define SRAM_BASE      0x20000000UL  // SRAM 起始地址
#define PERIPH_BASE    0x40000000UL  // 外设基地址

// GPIOA 基地址
#define GPIOA_BASE    (PERIPH_BASE + 0x00010800UL)  // = 0x40010800
在寄存器开发方式下，这些地址会频繁用到。HAL 库把这些地址封装成了结构体指针，间接访问即可。

2. HAL 库 vs LL 库 vs 寄存器

2.1 三种开发方式对比

对比维度寄存器HAL 库LL 库
代码层级直接写硬件寄存器封装层，跨平台轻量封装，近寄存器
学习难度⭐⭐⭐⭐⭐ 难⭐⭐ 简单⭐⭐⭐ 中等
代码体积最小较大较小
执行效率最高中等（有开销）接近寄存器
可读性低（数字魔数多）高（函数名语义清晰）中等
CubeMX 支持手动✅ 完全支持✅ 完全支持
典型场景研究、极致优化产品开发、快速原型中等性能需求

2.2 代码示例对比

方式一：直接操作寄存器（点灯）

// 开启 GPIOA 时钟
*(volatile uint32_t *)(0x40021000 + 0x18) |= (1 << 2);

// 配置 PA1 为输出（推挽，50MHz）
*(volatile uint32_t *)(0x40010800 + 0x00) &= ~(0xF << 4);   // CRL 清零
*(volatile uint32_t *)(0x40010800 + 0x00) |=  (0x3 << 4);   // CNF=00, MODE=11

// 输出高电平（点亮 LED）
*(volatile uint32_t *)(0x40010800 + 0x0C) |= (1 << 1);

方式二：HAL 库（点灯）

// 初始化
MX_GPIO_Init();  // CubeMX 自动生成

// 点灯/灭灯
HAL_GPIO_WritePin(GPIOA, GPIO_PIN_1, GPIO_PIN_SET);   // PA1 输出高
HAL_GPIO_WritePin(GPIOA, GPIO_PIN_1, GPIO_PIN_RESET); // PA1 输出低
HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_1);                 // 电平翻转

方式三：LL 库（点灯）

// 手动初始化
LL_AHB2_GRP1_EnableClock(LL_AHB2_GRP1_PERIPH_GPIOA);
LL_GPIO_SetPinMode(GPIOA, LL_GPIO_PIN_1, LL_GPIO_MODE_OUTPUT);
LL_GPIO_SetPinSpeed(GPIOA, LL_GPIO_PIN_1, LL_GPIO_SPEED_FREQ_HIGH);

// 读写
LL_GPIO_SetOutputPin(GPIOA, LL_GPIO_PIN_1);
LL_GPIO_ResetOutputPin(GPIOA, LL_GPIO_PIN_1);

2.3 何时用哪种？

推荐学习路径：寄存器原理 → HAL 库 → LL 库
实际工作：90% 的项目选 HAL 库，配合 CubeMX 效率最高。
需要极致性能：中断频繁、实时性极高的场景可考虑 LL 库或寄存器。
本教程后续所有示例均使用 HAL 库 + CubeMX，这是目前最主流的入门组合。

3.7 USER CODE 区域规则（最重要！）

CubeMX 重新生成代码时会保留标记区域内的内容，但会覆盖其他区域：

/* USER CODE BEGIN xxx */          ← 你写代码放这里 ✅
void SystemClock_Config(void) {
    // CubeMX 生成，勿手动修改
}
int main(void) {
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();           // CubeMX 生成
    MX_USART1_UART_Init();   // CubeMX 生成
    /* USER CODE BEGIN 2 */    // ← ✅ 安全区
    // 你的代码写这里
    /* USER CODE END 2 */
    while (1) {
        /* USER CODE BEGIN 3 */
        // 你的循环代码
        /* USER CODE END 3 */
    }
}
/* USER CODE END xxx */          ← 你写代码放这里 ✅
红线警告：不要在 BEGIN/END 注释之外写自己的代码，否则下次用 CubeMX 重新生成代码时会被清空！
安全区：所有用户代码只能写在 /* USER CODE BEGIN */ 和 /* USER CODE END */ 之间。
USER CODE 区域就像租房子——CubeMX 是房东，他负责装修框架（系统初始化），但给你留了几个房间（USER CODE 区）可以自由装修。下次房东重新装修（重新生成代码）时，只动公共区域，你租的房间完好无损。

4. GPIO 操作

4.1 推挽输出 — 点灯

GPIO 最基本的用法是输出模式，用来驱动 LED。电路通常是 LED 正极接 IO 口，负极通过电阻接地，IO 输出高时灯灭，输出低时灯亮（低电平点亮）：

CubeMX 配置

PA1 → GPIO_Output
User Label: LED_Pin  （会生成 LED_Pin 常量，代码更易读）

其他参数保持默认：
  GPIO output level:  Low（默认灭灯）
  GPIO mode:           Output Push-Pull
  GPIO Pull-up/Pull-down: No pull-up
  Maximum output speed: Low（LED 不需要高速）

main.c 代码

/* USER CODE BEGIN 0 */
#include "stdio.h"  // 后面 printf 重定向用
/* USER CODE END 0 */

// --- main() 中 ---
/* USER CODE BEGIN 2 */
printf("LED Demo Started!\r\n");
/* USER CODE END 2 */

while (1) {
    /* USER CODE BEGIN 3 */
    HAL_GPIO_WritePin(GPIOA, LED_Pin, GPIO_PIN_SET);   // PA1 = 高 → 灯灭
    HAL_Delay(500);                                     // 延时 500ms
    HAL_GPIO_WritePin(GPIOA, LED_Pin, GPIO_PIN_RESET); // PA1 = 低 → 灯亮
    HAL_Delay(500);
    /* USER CODE END 3 */
}
HAL_Delay() 是基于 SysTick 的阻塞延时，精度不高但够用。它内部调用了 HAL_IncTick()，由 Systick_IRQHandler 每 1ms 中断驱动。

4.2 按键输入 + 软件消抖

机械按键在按下/松开瞬间会产生接触抖动（约 5~10ms），软件消抖的思路是：检测到低电平后，延时 10~20ms 再确认一次，避免误判。

CubeMX 配置

PA0 → GPIO_Input
User Label: KEY_Pin

GPIO mode: External Interrupt with Falling edge trigger
  （注意：这里选中断模式，不是普通输入模式！）

Pull-up:  按键一端接地，另一端接 PA0，开启上拉 → 按下=低，松开=高

电路接法：
   3.3V ──┤├──── PA0（配置上拉，内部弱上拉到高）
              │
            KEY
              │
             GND

按下 KEY → PA0 被短路到 GND → 低电平（按键按下）
松开 KEY → 内部上拉电阻将 PA0 拉回高 → 高电平（按键松开）

按键消抖代码

// --- 方式1：轮询方式（简单，但占用 CPU）---
#define KEY_DEBOUNCE_MS  20

if (HAL_GPIO_ReadPin(GPIOA, KEY_Pin) == GPIO_PIN_RESET) {
    HAL_Delay(KEY_DEBOUNCE_MS);  // 等待抖动消失
    if (HAL_GPIO_ReadPin(GPIOA, KEY_Pin) == GPIO_PIN_RESET) {
        // 确认按键确实被按下
        HAL_GPIO_TogglePin(GPIOA, LED_Pin);
        // 等待按键松开，避免连击
        while (HAL_GPIO_ReadPin(GPIOA, KEY_Pin) == GPIO_PIN_RESET);
    }
}

// --- 方式2：使用标志位（更好的方式）---
static uint8_t key_pressed = 0;

if (HAL_GPIO_ReadPin(GPIOA, KEY_Pin) == GPIO_PIN_RESET && !key_pressed) {
    HAL_Delay(20);
    if (HAL_GPIO_ReadPin(GPIOA, KEY_Pin) == GPIO_PIN_RESET) {
        key_pressed = 1;
        HAL_GPIO_TogglePin(GPIOA, LED_Pin);
    }
} else if (HAL_GPIO_ReadPin(GPIOA, KEY_Pin) == GPIO_PIN_SET) {
    key_pressed = 0;
}

4.3 外部中断

当 CubeMX 配置引脚为 External Interrupt 时，需要在中断处理函数中响应事件：

CubeMX 配置

PA0 → GPIO_Exti0（自动映射到 EXTI0 中断线）

NVIC Settings：
  ✅ EXTI0_IRQHandler  ← 勾选使能（否则不会进中断）

中断处理函数（stm32f1xx_it.c）

// CubeMX 自动生成回调函数
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin)
{
    if (GPIO_Pin == KEY_Pin) {
        // 这里写按键中断的处理逻辑
        HAL_GPIO_TogglePin(GPIOA, LED_Pin);
    }
}

// CubeMX 生成的底层中断服务程序（不要修改）
void EXTI0_IRQHandler(void)
{
    HAL_GPIO_EXTI_IRQHandler(KEY_Pin);  // 会调用上面的 Callback
}
中断回调函数中 不要使用 HAL_Delay()！因为 HAL_Delay 本身依赖 SysTick 中断，如果在中断中调用，而该中断优先级 ≤ SysTick，可能导致死锁。

4.4 复用功能（AFIO）

STM32 的 GPIO 引脚可以通过复用功能（Alternate Function）连接到片上外设：

GPIO 引脚的工作模式（共 8 种）：

输入模式：
  - 浮空输入        GPIO_Mode_IN_FLOATING
  - 上拉输入        GPIO_Mode_IPU
  - 下拉输入        GPIO_Mode_IPD
  - 模拟输入        GPIO_Mode_AIN    （ADC 用）

输出模式：
  - 推挽输出        GPIO_Mode_Out_PP
  - 开漏输出        GPIO_Mode_Out_OD

复用模式：
  - 复用推挽输出    GPIO_Mode_AF_PP   （外设控制，如 USART TX）
  - 复用开漏输出    GPIO_Mode_AF_OD
CubeMX 配置引脚功能后，会自动配置对应的复用功能，无需手动设置 AFIO 寄存器。

5. UART 通信

5.1 基础概念

UART（通用异步收发传输器）就像两个人隔空喊话，不需要约定说话节奏（异步），但需要说同一种方言（相同的波特率）。TX 发送，RX 接收，接线时交叉连接（TX 接对方的 RX）。
参数常见值说明
波特率9600 / 115200每秒传输的位数，最常用 115200
数据位8 / 9一帧中的有效数据位数
停止位1 / 2表示一帧结束
校验位None / Odd / Even奇偶校验

5.2 初始化

CubeMX 配置

USART1：
  Mode: Asynchronous
  Baud Rate: 115200
  Word Length: 8 Bits
  Parity: None
  Stop Bits: 1

引脚自动配置：
  PA9  → USART1_TX
  PA10 → USART1_RX

NVIC Settings：
  ✅ USART1_IRQHandler  ← 勾选（使用中断时需要）

CubeMX 生成的初始化（main.c）

void MX_USART1_UART_Init(void)
{
    huart1.Instance = USART1;
    huart1.Init.BaudRate = 115200;
    huart1.Init.WordLength = UART_WORDLENGTH_8B;
    huart1.Init.StopBits = UART_STOPBITS_1;
    huart1.Init.Parity = UART_PARITY_NONE;
    huart1.Init.Mode = UART_MODE_TX_RX;
    huart1.Init.HwFlowCtl = UART_HWCONTROL_NONE;
    huart1.Init.OverSampling = UART_OVERSAMPLING_16;
    if (HAL_UART_Init(&huart1) != HAL_OK) {
        Error_Handler();
    }
}

5.3 发送与接收（阻塞方式）

// 发送一个字符串（阻塞，直到发送完成）
char msg[] = "Hello STM32!\r\n";
HAL_UART_Transmit(&huart1, (uint8_t *)msg, strlen(msg), HAL_MAX_DELAY);

// 发送单个字符
HAL_UART_Transmit(&huart1, (uint8_t *)"A", 1, 100);

// 接收一个字符（阻塞，直到接收到数据）
uint8_t ch;
HAL_UART_Receive(&huart1, &ch, 1, HAL_MAX_DELAY);

5.4 中断接收（非阻塞）

使用中断接收可以避免 CPU 阻塞等待，当数据到来时自动触发中断：

/* USER CODE BEGIN 0 */
// 定义接收缓冲区
#define RX_BUF_SIZE 64
uint8_t rx_buffer[RX_BUF_SIZE];
volatile uint8_t rx_index = 0;
/* USER CODE END 0 */

// --- main() 初始化后开启第一次中断接收 ---
/* USER CODE BEGIN 2 */
HAL_UART_Receive_IT(&huart1, &rx_buffer[rx_index], 1);
/* USER CODE END 2 */

// --- 实现接收中断回调函数 ---
/* USER CODE BEGIN 4 */
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
{
    if (huart->Instance == USART1) {
        // 接收到换行符表示一帧数据结束
        if (rx_buffer[rx_index] == '\n') {
            rx_buffer[rx_index] = '\0';  // 字符串结束
            printf("Received: %s\r\n", rx_buffer);
            rx_index = 0;  // 清空缓冲区，准备下一帧
        } else {
            rx_index++;
            if (rx_index >= RX_BUF_SIZE) rx_index = 0;  // 防止溢出
        }
        // 继续接收下一个字节
        HAL_UART_Receive_IT(&huart1, &rx_buffer[rx_index], 1);
    }
}
/* USER CODE END 4 */
每次调用 HAL_UART_Receive_IT() 只接收 1 个字节，收到后需要再次调用才能继续接收下一字节。

5.5 DMA 接收（最高效）

DMA（直接内存访问）允许外设直接和内存交换数据，不占用 CPU：

// CubeMX 配置：
// DMA Settings → USART1_RX → Request: USART1_RX
//   Mode: Circular（循环模式）或 Normal（一次）
//   Increment Address: Memory ✓

// --- DMA 初始化（CubeMX 自动生成）---
/* USER CODE BEGIN 0 */
#define RX_DMA_BUF_SIZE  100
uint8_t dma_rx_buf[RX_DMA_BUF_SIZE];
/* USER CODE END 0 */

// --- 启动 DMA 接收（main 中）---
/* USER CODE BEGIN 2 */
memset(dma_rx_buf, 0, RX_DMA_BUF_SIZE);
HAL_UART_Receive_DMA(&huart1, dma_rx_buf, RX_DMA_BUF_SIZE);
__HAL_UART_ENABLE_IT(&huart1, UART_IT_IDLE);  // 开启空闲中断检测帧结束
/* USER CODE END 2 */

// --- USART1 中断处理函数（stm32f1xx_it.c）---
void USART1_IRQHandler(void)
{
    HAL_UART_IRQHandler(&huart1);
    // 空闲中断回调
    if (__HAL_UART_GET_FLAG(&huart1, UART_FLAG_IDLE)) {
        __HAL_UART_CLEAR_IDLEFLAG(&huart1);
        // 计算接收到的数据长度
        uint16_t len = RX_DMA_BUF_SIZE - __HAL_DMA_GET_COUNTER(huart1.hdmarx);
        printf("DMA received %d bytes: %s\r\n", len, dma_rx_buf);
        memset(dma_rx_buf, 0, RX_DMA_BUF_SIZE);
    }
}

5.6 printf 重定向（最常用）

有了 printf，就可以像在 PC 上编程一样输出调试信息了：

6. 定时器

6.1 基本概念

STM32 的定时器本质上是一个计数器，在时钟驱动下不断递增，当计数值达到预设的自动重装载值（ARR）时触发更新事件，可以产生中断或 DMA 请求：

定时周期计算公式：
  Tout = (ARR + 1) × (PSC + 1) / Tclk

其中：
  Tclk  = APB 总线时钟频率（STM32F103 = 72MHz）
  PSC   = 预分频系数（分频器，将 72MHz 降频）
  ARR   = 自动重装载值（计数器上限）
  Tout  = 溢出周期（秒）

例如：产生 1Hz 定时（每秒中断一次）
  Tclk = 72MHz, PSC = 7200-1, ARR = 10000-1
  计数频率 = 72MHz / 7200 = 10kHz
  周期 = 10000 / 10kHz = 1 秒 ✅

6.2 基本定时中断

CubeMX 配置（TIM2）

Timers → TIM2：
  Internal Clock（APB1 Timer Clocks = 72MHz）
  Prescaler:         7200-1
  Counter Period:     1000-1
  ✅ Global Interrupt → TIM2_IRQn 勾选

代码实现

// --- 开启定时器（在 main 初始化后）---
/* USER CODE BEGIN 2 */
HAL_TIM_Base_Start_IT(&htim2);  // 启动定时器，产生周期中断
/* USER CODE END 2 */

// --- 定时器中断回调 ---
/* USER CODE BEGIN 4 */
static uint32_t timer_count = 0;

void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM2) {
        timer_count++;
        if (timer_count >= 10) {  // 每秒（1Hz）
            timer_count = 0;
            HAL_GPIO_TogglePin(GPIOA, LED_Pin);  // LED 翻转
            printf("Timer fired! LED toggled.\r\n");
        }
    }
}
/* USER CODE END 4 */

6.3 PWM 输出

PWM（脉宽调制）通过调节高电平在一个周期中所占的比例来控制设备，如调节 LED 亮度或电机转速：

一个 PWM 周期：
  ┌──────────────┬────────────┐
  │ 高电平宽度   │ 低电平宽度 │
  └──────────────┴────────────┘
  |<-------- ARR + 1 -------->|
  |< CCR ---->|  （比较寄存器值）

PWM 占空比 = CCR / (ARR + 1) × 100%

例如：ARR=999, CCR=500 → 50% 占空比
      ARR=999, CCR=750 → 75% 占空比

CubeMX 配置（PWM 模式）

Timers → TIM3 → Channel 1: PWM Generation CH1

配置：
  Prescaler:        720-1    （计数频率 = 72MHz/720 = 100kHz）
  Counter Period:   999      （PWM 频率 = 100kHz/1000 = 100Hz）
  PWM Mode:         Mode 1
  Pulse (CCR):      0        （初始占空比 0%）
  Output compare preload: Enable

引脚自动分配：PA6 → TIM3_CH1

PWM 控制代码

/* USER CODE BEGIN 2 */
HAL_TIM_PWM_Start(&htim3, TIM_CHANNEL_1);  // 开启 PWM 输出
/* USER CODE END 2 */

while (1) {
    /* USER CODE BEGIN 3 */
    // 渐变呼吸灯效果
    for (uint16_t duty = 0; duty <= 999; duty += 5) {
        __HAL_TIM_SET_COMPARE(&htim3, TIM_CHANNEL_1, duty);
        HAL_Delay(10);
    }
    for (uint16_t duty = 999; duty > 0; duty -= 5) {
        __HAL_TIM_SET_COMPARE(&htim3, TIM_CHANNEL_1, duty);
        HAL_Delay(10);
    }
    /* USER CODE END 3 */
}
使用 __HAL_TIM_SET_COMPARE() 可以在运行时动态调整占空比，无需停止定时器。

6.4 输入捕获

输入捕获用于测量外部信号的频率或脉宽，例如测量编码器脉冲或 PWM 占空比：

应用场景：
  - 测量正交编码器的速度
  - 测量外部信号频率
  - 测量 PWM 信号的占空比和周期

CubeMX 配置（输入捕获）

Timers → TIM4 → Channel 1: Input Capture Direct Mode

Prescaler: 72-1  （计数频率 = 72MHz/72 = 1MHz，最小分辨率 1μs）
Counter Period: 0xFFFF（最大值 65535）
Trigger Output: Update Event（自由计数）

引脚：PB6 → TIM4_CH1（需要根据原理图连接待测信号）

频率测量代码

/* USER CODE BEGIN 0 */
volatile uint32_t capture_val = 0;
volatile uint8_t capture_done = 0;
/* USER CODE END 0 */

/* USER CODE BEGIN 2 */
HAL_TIM_IC_Start_IT(&htim4, TIM_CHANNEL_1);
/* USER CODE END 2 */

/* USER CODE BEGIN 4 */
void HAL_TIM_IC_CaptureCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM4 && htim->Channel == HAL_TIM_ACTIVE_CHANNEL_1) {
        capture_val = HAL_TIM_ReadCapturedValue(htim, TIM_CHANNEL_1);
        capture_done = 1;
    }
}
/* USER CODE END 4 */

// --- 在主循环计算频率 ---
/* USER CODE BEGIN 3 */
if (capture_done) {
    capture_done = 0;
    // 计数频率 1MHz, capture_val 为计数值

}
/* USER CODE END 3 */

6.5 输出比较（Output Compare）

输出比较可以在计数器达到某个值时翻转引脚电平，常用于生成固定频率的方波：

CubeMX 配置：TIM3 → Channel 1 → Output Compare No Output
Mode: Toggle on Match（匹配时翻转电平）
输出比较和 PWM 的区别：PWM 在一个周期内自动调节占空比，输出比较只在匹配点翻转一次，不自动重装，适合生成等占空比方波。

7. ADC（模数转换器）

7.1 基本原理

ADC 将连续的模拟电压信号（0~3.3V）转换为离散的数字量（0~4095，12位分辨率）：

数字值 = (Vin / Vref+) × 4095

例如：
  Vin = 1.65V, Vref+ = 3.3V → ADC值 = (1.65/3.3) × 4095 = 2047
  Vin = 0V   → ADC值 = 0
  Vin = 3.3V → ADC值 = 4095

实际电压计算：
  Vin = ADC_value × 3.3 / 4095

7.2 单次转换（最简单）

CubeMX 配置

Analog → ADC1 → IN1 (PA1)

配置：
  Scan Conversion Mode:    Disabled（单通道可关闭）
  Continuous Conversion:   Disabled（单次转换）
  DMA Continuous Requests: Disabled（不使用 DMA）

Rank（通道顺序）：
  Channel 1: ADC_CHANNEL_1 (PA1)

代码

/* USER CODE BEGIN 0 */
uint32_t adc_value;
/* USER CODE END 0 */

// 启动一次 ADC 转换
HAL_ADC_Start(&hadc1);
HAL_ADC_PollForConversion(&hadc1, 10);  // 等待转换完成，最多 10ms

if (HAL_IS_BIT_SET(HAL_ADC_GetState(&hadc1), HAL_ADC_STATE_REG_EOC)) {
    adc_value = HAL_ADC_GetValue(&hadc1);
    float voltage = adc_value * 3.3f / 4095.0f;
    printf("ADC=%lu, Voltage=%.3fV\r\n", adc_value, voltage);
}
HAL_ADC_Stop(&hadc1);

7.3 连续转换

// CubeMX 配置：Continuous Conversion: Enabled

// 启动后自动连续转换，无需每次手动触发
HAL_ADC_Start(&hadc1);
while (1) {
    if (HAL_ADC_PollForConversion(&hadc1, 10) == HAL_OK) {
        uint32_t val = HAL_ADC_GetValue(&hadc1);
        printf("ADC=%lu\r\n", val);
    }
    HAL_Delay(100);
}
HAL_ADC_Stop(&hadc1);

7.4 DMA + ADC（最实用）

使用 DMA 自动搬运 ADC 数据，CPU 只管读取，彻底解放：

CubeMX 配置

ADC1 配置：
  Scan Conversion Mode:    Enabled（多通道）
  Continuous Conversion:   Enabled
  DMA Continuous Requests: Enabled（触发 DMA）
  Number Of Conversion:    3（3 个通道）

Rank 1: Channel 1 (PA1) — 电位器
Rank 2: Channel 2 (PA2) — 热敏电阻
Rank 3: Channel 4 (PA4) — 光敏电阻

DMA Settings → ADC1：
  Mode: Circular（循环）
  Increment Address: Memory ✓, Peripheral ✗

DMA 多通道采集代码

/* USER CODE BEGIN 0 */
#define ADC_CHANNEL_NUM  3
uint16_t adc_buf[ADC_CHANNEL_NUM];  // DMA 目标缓冲区

// 对应 Rank 顺序：adc_buf[0]=IN1, [1]=IN2, [2]=IN4
/* USER CODE END 0 */

// --- main() 中启动 ---
/* USER CODE BEGIN 2 */
HAL_ADC_Start_DMA(&hadc1, (uint32_t *)adc_buf, ADC_CHANNEL_NUM);
/* USER CODE END 2 */

// --- 主循环中直接读取 ---
/* USER CODE BEGIN 3 */
float v1 = adc_buf[0] * 3.3f / 4095.0f;
float v2 = adc_buf[1] * 3.3f / 4095.0f;
float v3 = adc_buf[2] * 3.3f / 4095.0f;
printf("PA1=%.3fV PA2=%.3fV PA4=%.3fV\r\n", v1, v2, v3);
HAL_Delay(500);
/* USER CODE END 3 */

7.5 多通道扫描

多通道扫描时，ADC 会按 Rank 顺序依次转换所有通道，然后产生一个 EOC（转换结束）标志。如果启用 DMA，每转换完一个通道自动传输一个值，全部完成后 DMA 才触发一次中断。
注意注入通道：STM32F103 有两组 ADC 通道——规则通道（最多 16 个，通过 DMA 批量传输）和注入通道（最多 4 个，优先级更高，会打断规则通道）。初学者先用规则通道即可。

8. NVIC 中断

8.1 中断优先级分组

STM32 使用 4 位优先级字段（共 16 级），但 F103 只用了高 4 位，支持 0~15 共 16 个优先级，数字越小优先级越高：

优先级分组（通过 HAL_NVIC_SetPriorityGrouping 设置）：

  NVIC_PRIORITYGROUP_0: 0 位用于抢占，4 位用于子优先级（响应）
  NVIC_PRIORITYGROUP_1: 1 位用于抢占，3 位用于子优先级
  NVIC_PRIORITYGROUP_2: 2 位用于抢占，2 位用于子优先级  ← CubeMX 默认
  NVIC_PRIORITYGROUP_3: 3 位用于抢占，1 位用于子优先级
  NVIC_PRIORITYGROUP_4: 4 位用于抢占，0 位用于子优先级

CubeMX 默认使用 NVIC_PRIORITYGROUP_2（2位抢占 + 2位子优先级）：
  - 抢占优先级: 0~3（共 4 级）
  - 子优先级:   0~3（共 4 级）

8.2 中断优先级规则

优先级类型规则比喻
抢占优先级（Preempt）高抢占优先级可以打断低抢占优先级的正在执行的中断领导可以中断下属的工作
子优先级（Sub）抢占优先级相同时，高的子优先级先响应同一级别的两个人，抢答器快的赢
两者都相等自然顺位（硬件中断向量号小的优先）平局靠抽签
实用建议：普通外设中断（UART、TIM）用抢占=1，子优先级=0；实时性要求极高的（如 Systick）用抢占=0。日常开发记住：数字越小，优先级越高。

8.3 中断使能（CubeMX vs 代码）

// 方式1：CubeMX 可视化使能（推荐）
//   在 Pinout & Configuration → NVIC → 勾选对应中断的 Enabled 框
//   CubeMX 会自动在 SystemClock_Config 后调用：
HAL_NVIC_SetPriority(EXTI0_IRQn, 0, 0);   // 优先级：抢占=0，子=0（最高）
HAL_NVIC_EnableIRQ(EXTI0_IRQn);             // 使能中断

// 方式2：代码中手动使能
HAL_NVIC_SetPriority(TIM2_IRQn, 1, 0);     // 抢占=1，子=0
HAL_NVIC_EnableIRQ(TIM2_IRQn);

// 方式3：修改已有中断优先级
// （比如在某个场景下临时提高 TIM2 优先级）
__HAL_RCC_TIM2_CLK_ENABLE();  // 先开时钟
HAL_NVIC_SetPriority(TIM2_IRQn, 0, 0);     // 提高到最高
HAL_NVIC_EnableIRQ(TIM2_IRQn);

8.5 常用中断配置示例

// 设置中断优先级（main 中初始化后调用）
// 外部中断 0（PA0 按键）
HAL_NVIC_SetPriority(EXTI0_IRQn, 0, 0);    // 最高优先级
HAL_NVIC_EnableIRQ(EXTI0_IRQn);

// 串口 1 中断
HAL_NVIC_SetPriority(USART1_IRQn, 2, 0);  // 中等优先级

// 定时器 2 中断

中断中禁止的操作：
1. 不要在中断中调用 HAL_Delay()
2. 不要在中断中做大量计算或 IO 操作（影响实时性）
3. 共享资源需要使用 __HAL_UART_ENABLE_IT() 等原子操作
4. 标志位建议用 volatile 声明

9. 时钟配置

9.1 三种时钟源

时钟源频率精度适用场景
HSI8MHz（内部）±1%（受温度影响）默认、低成本、精度要求不高
HSE4~25MHz（外部晶振）高（晶振决定）串口通信、CAN、USB（需要精确频率）
PLL倍频后最大 72MHz继承输入源系统主频（SYSCLK）
HSI 就像手机的自动调频 CPU，便宜但会随温度漂移；HSE 就像外接了一个精准的节拍器，保证你唱歌不跑调；PLL 就是扩音器，把输入频率放大到需要的值。

9.2 SystemClock_Config 详解

这是 CubeMX 生成的时钟配置函数，手动修改容易出错，建议通过 CubeMX 图形界面配置后重新生成：

void SystemClock_Config(void)
{
    RCC_OscInitTypeDef RCC_OscInitStruct = {0};
    RCC_ClkInitTypeDef RCC_ClkInitStruct = {0};

    // ===== 第1步：配置振荡器 =====
    RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSE
                                     | RCC_OSCILLATORTYPE_HSI;
    RCC_OscInitStruct.HSEState = RCC_HSE_ON;            // 开启外部晶振
    RCC_OscInitStruct.HSEPredivValue = RCC_HSE_PREDIV_DIV1; // 预分频=1（不用额外分）
    RCC_OscInitStruct.HSIState = RCC_HSI_ON;            // 开启内部晶振（备用）
    RCC_OscInitStruct.PLL.PLLState = RCC_PLL_ON;        // 开启 PLL
    RCC_OscInitStruct.PLL.PLLSource = RCC_PLLSOURCE_HSE;// PLL 输入=HSE
    RCC_OscInitStruct.PLL.PLLMUL = RCC_PLL_MUL9;         // 8MHz × 9 = 72MHz
    if (HAL_RCC_OscConfig(&RCC_OscInitStruct) != HAL_OK) {
        Error_Handler();
    }

    // ===== 第2步：配置时钟分配 =====
    RCC_ClkInitStruct.ClockType = RCC_CLOCKTYPE_HCLK   // 总线时钟类型
                                | RCC_CLOCKTYPE_SYSCLK
                                | RCC_CLOCKTYPE_PCLK1
                                | RCC_CLOCKTYPE_PCLK2;
    RCC_ClkInitStruct.SYSCLKSource = RCC_SYSCLKSOURCE_PLLCLK; // SYSCLK=PLL输出
    RCC_ClkInitStruct.AHBCLKDivider = RCC_SYSCLK_DIV1;        // HCLK=72MHz
    RCC_ClkInitStruct.APB1CLKDivider = RCC_HCLK_DIV2;         // APB1=36MHz ⚠️
    RCC_ClkInitStruct.APB2CLKDivider = RCC_HCLK_DIV1;         // APB2=72MHz
    if (HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_2) != HAL_OK) {
        Error_Handler();
    }
}
APB1 总线频率不能超过 36MHz！
STM32F103 的 APB1 外设（UART2/3、SPI2/3、I2C1/2、TIM3~7）最高只支持 36MHz。
如果 SYSCLK=72MHz，APB1 分频系数必须 ≥2，即 RCC_HCLK_DIV2。

9.3 各总线时钟频率

典型配置（SYSCLK=72MHz）：

  SYSCLK (系统时钟)      = 72MHz  ← PLL 输出
       │
       ├── AHB Prescaler /1 ──▶ HCLK = 72MHz  (内核、Flash、DMA、总线矩阵)
       │                              │
       │                              ├── APB2 Prescaler /1 ──▶ APB2 = 72MHz
       │                              │    (GPIOA~E, ADC1/2, USART1, SPI1, TIM1)
       │                              │
       │                              └── APB1 Prescaler /2 ──▶ APB1 = 36MHz ⚠️
       │                                   (USART2/3, SPI2/3, I2C1/2, TIM2~7)
       │
       └── ADC Prescaler /6 ──▶ ADC Clock = 12MHz  (最大 14MHz，推荐 12MHz)

外设挂载规律：
  APB2（高速）: GPIO, USART1, SPI1, ADC, TIM1, TIM8, EXTI
  APB1（低速）: USART2/3, SPI2/3, I2C1/2, TIM2~7, CAN, USB
  AHB:         内核、DMA、Flash、SRAM

9.4 MCO 输出（验证时钟）

// PA8 可以输出内部时钟信号，用示波器/逻辑分析仪测量验证
// CubeMX: System Core → RCC → MCO → PLL/2

HAL_RCC_MCOConfig(RCC_MCO1, RCC_MCO1SOURCE_PLLCLK_DIV2, RCC_MCODIV_2);
// PA8 输出 = SYSCLK / 2 / 2 = 72MHz / 4 = 18MHz

10. 常见问题（FAQ）

❌ 问题1：HAL_Delay 死锁 / 程序卡死

现象：程序运行到某处后完全卡死，或者定时器/PWM 不工作。
根因：HAL_Delay() 依赖 SysTick 中断，如果 SysTick 中断被更高优先级的中断长期阻塞，或者在中断中调用 HAL_Delay，就会死锁。
死锁场景举例：
1. 在 USART1 中断中（优先级高于 SysTick）调用 HAL_Delay() → 卡死
2. 在某中断中关闭全局中断 __disable_irq() 后调用 HAL_Delay() → 卡死
3. 使用 FreeRTOS 时，在中断优先级高于 configMAX_SYSCALL_INTERRUPT_PRIORITY 的中断中调用 HAL API → 系统崩溃 
// ✅ 正确做法：在中断中绝对不要调用 HAL_Delay()
// ✅ 使用标志位替代延时逻辑

volatile uint32_t tick_flag = 0;

void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    tick_flag = 1;  // 只设置标志，不做耗时操作
}

while (1) {
    if (tick_flag) {
        tick_flag = 0;
        HAL_GPIO_TogglePin(GPIOA, LED_Pin);  // 主循环处理，不在中断中
    }
    // 可以在此做其他工作
}

// ✅ 或者使用 osDelay（FreeRTOS）替代 HAL_Delay

❌ 问题2：中断进不去

现象：配置了中断，但调试时发现中断处理函数从未被调用。
根因：NVIC 中断未使能 / 时钟未开启 / 引脚配置错误 
// 检查清单：
// 1. CubeMX NVIC Settings 中是否勾选了对应中断？
//    例：EXTI0_IRQn → ✅ Enabled

// 2. 外设时钟是否开启？
HAL_RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);  // GPIO 时钟
HAL_RCC_APB2PeriphClockCmd(RCC_APB2Periph_AFIO, ENABLE);    // AFIO 时钟（EXTI 必须）

// 3. USART 中断是否手动使能？（CubeMX 生成代码后可能需要检查）
__HAL_UART_ENABLE_IT(&huart1, UART_IT_RXNE);  // 接收中断使能

// 4. 检查优先级是否设置（优先级为 15 最低，可能被其他高优先级打断）
HAL_NVIC_SetPriority(USART1_IRQn, 3, 0);
HAL_NVIC_EnableIRQ(USART1_IRQn);

// 5. 检查中断引脚映射是否正确
//    例：PA0 对应 EXTI0，不是 EXTI1

❌ 问题3：时钟配置错误导致串口乱码

现象：串口输出全是乱码，波特率计算错误。
根因：USART 波特率由 APB 时钟决定，如果配置的和代码计算的不一致，就会乱码。 
// 波特率计算公式（USARTDIV）：
// USARTDIV = PCLK / (16 × 波特率)
//
// 例：APB1 = 36MHz, 波特率 = 115200
// USARTDIV = 36MHz / (16 × 115200) = 19.53
//
// 对应的 BRR 寄存器值 = 19.53 → 小数部分 × 16 = 0.53×16 = 8.5 ≈ 8
// BRR = 0x134 (0x13<<4 | 0x8) = 304
//
// 常见错误：
//  1. 串口挂在 APB1，但代码中按 APB2 时钟计算波特率 → 乱码
//  2. 使用 HSI（8MHz）时钟但 PLL 配置错误 → 时钟不对 → 乱码
//  3. 晶振频率写错（如外部晶振 8MHz 写成 12MHz）→ 乱码
//
// ✅ 解决：在 CubeMX 中正确配置时钟树，重新生成代码

// ✅ 验证方法：用示波器/MCO 测量 PA8 输出，确认 SYSCLK 实际值
HAL_RCC_MCOConfig(RCC_MCO1, RCC_MCO1SOURCE_PLLCLK_DIV2, RCC_MCODIV_1);
// PA8 输出 PLLCLK / 2 = 36MHz → 用示波器量一下

❌ 问题4：引脚冲突

现象：多个外设同时使用时报错，或者引脚状态不对。
根因：同一引脚被多次分配功能，或多个外设共享同一时钟未开启。 
// 常见冲突：

// 1. PA9/PA10 被占用
//    USART1_TX = PA9, USART1_RX = PA10
//    如果同时配置 SWD 调试口（PA13/14 默认 SWD），可能冲突
//    ✅ CubeMX: SYS → Debug → Serial Wire（只占用 PA13/14，不占 PA9/10）

// 2. TIM1 和 TIM3 都想用 PA6
//    CubeMX 中同时开启两个定时器的同一引脚会报错
//    ✅ 解决：选择不同的引脚或不同的定时器

// 3. GPIO 和 ADC 复用引脚
//    例：PA1 配置为 GPIO_Output 后又想作为 ADC 通道
//    ✅ GPIO 和 ADC 通道是复用的，在 CubeMX 中选择功能即可

// 4. ADC 时钟超过 14MHz
//    如果 PCLK2 = 72MHz, ADC_Prescaler = /4 → ADC=18MHz > 14MHz（超规格）
//    ✅ 改 ADC_Prescaler = /6 → ADC=12MHz

❌ 问题5：DMA 不工作

// 检查清单：

// 1. DMA 时钟是否开启？
__HAL_RCC_DMA1_CLK_ENABLE();  // DMA1 时钟，F103 只有 DMA1

// 4. 缓冲区地址是否对齐？（DMA 要求半字/字对齐）

11.3 时钟与定时计算速查

公式速记：
  PWM 频率 = Tclk / ((PSC+1) × (ARR+1))
  PWM 占空比 = CCR / (ARR+1) × 100%
  ADC 电压 = ADC_val × Vref / 4095
  UART 波特率误差 = |实际 - 期望| / 期望 × 100%
  定时周期 = (PSC+1) × (ARR+1) / Tclk

常用参数组合（72MHz SYSCLK）：
  APB1 = 36MHz (÷2), APB2 = 72MHz (÷1)
  ADC  = 12MHz (APB2 ÷6)
  TIM2 计数频率 = 72MHz / (PSC+1)
    PSC=7199 → 10kHz, ARR=9999 → 1Hz
    PSC=719  → 100kHz, ARR=999 → 100Hz PWM
    PSC=71   → 1MHz,   ARR=999 → 1kHz PWM

11.4 CubeMX USER CODE 安全区速查

  /* USER CODE BEGIN 0 */         ← 全局变量/头文件
  /* USER CODE END 0 */

  /* USER CODE BEGIN 1 */         ← System Clock 配置之前
  /* USER CODE END 1 */

  /* USER CODE BEGIN 2 */         ← main() 初始化后主循环前 ✅最常用
  /* USER CODE END 2 */

  /* USER CODE BEGIN 3 */         ← while(1) 循环内 ✅最常用
  /* USER CODE END 3 */

  /* USER CODE BEGIN 4 */         ← 回调函数（中断/UART/TIM等）
  /* USER CODE END 4 */

  /* USER CODE BEGIN PD */        ← 预处理器定义
  /* USER CODE END PD */
