🟢 ESP32 开发实战 细致教程

ESP32是乐鑫的双核WiFi+BLE芯片，比STM32多了无线通信能力，是IoT项目的首选。本教程覆盖Arduino和ESP-IDF两种开发框架。

🤔 ESP32 vs STM32 对比

特性ESP32STM32F103
核心Xtensa LX6 双核 240MHzCortex-M3 单核 72MHz
Flash4MB（内置，不可改）64KB~2MB（外挂）
RAM520KB20KB~1MB
WiFi✅ 802.11 b/g/n❌ 需外接模块
蓝牙✅ BLE 4.2+BR/EDR❌ 需外接模块
ADC18通道 12位16通道 12位
价格¥15~25（模块）¥5~10（芯片）
开发门槛低（Arduino）中高（HAL/寄存器）
实时性一般（WiFi占用CPU）强（裸机/RTOS）
生态Arduino社区庞大工业标准/ST支持
💡 类比：STM32像一辆精准的赛车——纯机械操控，响应快，适合控制；ESP32像一辆智能SUV——自带导航和通信，适合IoT，但操控感稍弱。

选型建议

场景推荐原因
电机控制/电源/工业STM32实时性强，定时器/PWM精确
智能家居/IoT/远程监控ESP32自带WiFi/BLE，省外设
可穿戴/电池供电ESP32-S2/S3（低功耗版）深度睡眠功耗10μA
学习嵌入式基础STM32寄存器级控制，理解底层
快速原型/创客ESP32Arduino生态，5分钟跑起来

🚀 Arduino 框架快速上手

Arduino是最简单的ESP32开发方式，适合新手和快速原型。

通过PlatformIO创建项目（推荐）

步骤2：PIO Home → New Project → Board选"Espressif ESP32 Dev Module" → Framework选"Arduino"
步骤3：自动生成项目结构

Blink — 点灯程序

// src/main.cpp
#include <Arduino.h>

#define LED_PIN 2  // ESP32开发板自带LED通常在GPIO2

void setup() {
    Serial.begin(115200);
    pinMode(LED_PIN, OUTPUT);
    Serial.println("Hello ESP32!");
}

void loop() {
    digitalWrite(LED_PIN, HIGH);
    delay(500);
    digitalWrite(LED_PIN, LOW);
    delay(500);
}
⚠️ Arduino框架的setup()/loop()：setup()只执行一次（初始化），loop()不断循环。这和STM32的main() while(1)本质一样，只是写法不同。

WiFi扫描

// src/main.cpp
#include <Arduino.h>
#include <WiFi.h>

void setup() {
    Serial.begin(115200);
    delay(1000);
    Serial.println("WiFi Scan Start");
}

void loop() {
    Serial.println("扫描中...");
    int n = WiFi.scanNetworks();
    if (n == 0) {
        Serial.println("没找到WiFi网络");
    } else {
        Serial.printf("找到 %d 个网络:\n", n);
        for (int i = 0; i < n; i++) {
            Serial.printf("  %d: %s (%d dBm) %s\n",
                i + 1,
                WiFi.SSID(i).c_str(),
                WiFi.RSSI(i),
                WiFi.encryptionType(i) == WIFI_AUTH_OPEN ? "开放" : "加密"
            );
        }
    }
    delay(10000);  // 每10秒扫描一次
}

最小ESP-IDF程序

// main/hello_world_main.c
#include <stdio.h>
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "esp_log.h"

static const char *TAG = "MAIN";

void app_main(void) {
    ESP_LOGI(TAG, "Hello ESP32!");

    while (1) {
        ESP_LOGI(TAG, "心跳: %d", xTaskGetTickCount() / configTICK_RATE_HZ);
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}
💡 ESP-IDF vs Arduino：ESP-IDF功能最全（完整WiFi/BLE/OTA API），但学习曲线陡。Arduino简单，大部分库都有。建议：初学用Arduino，项目复杂了切ESP-IDF。Arduino底层其实就是ESP-IDF的封装。

📶 WiFi 编程

STA模式（连路由器）

// ESP-IDF版
#include "esp_wifi.h"
#include "esp_event.h"
#include "nvs_flash.h"

#define WIFI_SSID "YourWiFi"
#define WIFI_PASS "YourPassword"

static void wifi_event_handler(void* arg, esp_event_base_t event_base,
                                int32_t event_id, void* event_data) {
    if (event_base == WIFI_EVENT && event_id == WIFI_EVENT_STA_START) {
        esp_wifi_connect();
    } else if (event_base == WIFI_EVENT && event_id == WIFI_EVENT_STA_DISCONNECTED) {
        ESP_LOGI("WiFi", "断开，重连...");
        esp_wifi_connect();
    } else if (event_base == IP_EVENT && event_id == IP_EVENT_STA_GOT_IP) {
        ip_event_got_ip_t* event = (ip_event_got_ip_t*) event_data;
        ESP_LOGI("WiFi", "获取IP:" IPSTR, IP2STR(&event->ip_info.ip));
    }
}

void wifi_init_sta(void) {
    nvs_flash_init();
    esp_netif_init();
    esp_event_loop_create_default();
    esp_netif_create_default_wifi_sta();

    wifi_init_config_t cfg = WIFI_INIT_CONFIG_DEFAULT();
    esp_wifi_init(&cfg);

    esp_event_handler_register(WIFI_EVENT, ESP_EVENT_ANY_ID, &wifi_event_handler, NULL);
    esp_event_handler_register(IP_EVENT, IP_EVENT_STA_GOT_IP, &wifi_event_handler, NULL);

    wifi_config_t wifi_config = {
        .sta = {
            .ssid = WIFI_SSID,
            .password = WIFI_PASS,
        },
    };
    esp_wifi_set_mode(WIFI_MODE_STA);
    esp_wifi_set_config(WIFI_IF_STA, &wifi_config);
    esp_wifi_start();
}

📡 BLE 蓝牙低功耗

BLE架构

概念类比说明
Server服务员提供数据的一方（通常是ESP32）
Client顾客读取数据的一方（通常是手机App）
Service菜单类别一组相关特征的集合
Characteristic具体菜品可读/可写/可通知的数据项
UUID菜品编号唯一标识Service/Characteristic

BLE广播+读写示例（Arduino）

#include <BLEDevice.h>
#include <BLEServer.h>
#include <BLEUtils.h>

#define SERVICE_UUID        "4fafc201-1fb5-459e-8fcc-c5c9c331914b"
#define CHARACTERISTIC_UUID "beb5483e-36e1-4688-b7f5-ea07361b26a8"

BLEServer* pServer = NULL;
BLECharacteristic* pCharacteristic = NULL;
bool deviceConnected = false;

class MyServerCallbacks: public BLEServerCallbacks {
    void onConnect(BLEServer* pServer) { deviceConnected = true; }
    void onDisconnect(BLEServer* pServer) { deviceConnected = false; }
};

void setup() {
    Serial.begin(115200);

    BLEDevice::init("ESP32-BLE");
    pServer = BLEDevice::createServer();
    pServer->setCallbacks(new MyServerCallbacks());

    BLEService* pService = pServer->createService(SERVICE_UUID);
    pCharacteristic = pService->createCharacteristic(
        CHARACTERISTIC_UUID,
        BLECharacteristic::PROPERTY_READ |
        BLECharacteristic::PROPERTY_WRITE |
        BLECharacteristic::PROPERTY_NOTIFY
    );

    pCharacteristic->setValue("Hello BLE");
    pService->start();

    BLEAdvertising* pAdvertising = BLEDevice::getAdvertising();
    pAdvertising->addServiceUUID(SERVICE_UUID);
    BLEDevice::startAdvertising();

    Serial.println("BLE已启动，用nRF Connect连接");
}

void loop() {
    if (deviceConnected) {
        // 通知客户端（推送数据）
        char buf[32];
        snprintf(buf, sizeof(buf), "ADC=%d", analogRead(34));
        pCharacteristic->setValue(buf);
        pCharacteristic->notify();
        delay(1000);
    }
    // 断开后重新广播
    if (!deviceConnected) {
        delay(500);
        BLEDevice::startAdvertising();
    }
}
💡 手机调试BLE：安装 nRF Connect（Nordic官方免费App），可以扫描/连接/读写BLE设备，是调试BLE必备工具。

🔄 FreeRTOS on ESP32

ESP32自带FreeRTOS，而且是双核版，可以指定任务跑在哪个核上。

双核任务分配

// Arduino框架下的FreeRTOS
TaskHandle_t Task1 = NULL;
TaskHandle_t Task2 = NULL;

void task1_func(void* param) {
    while (1) {
        Serial.printf("核心%d: 任务1\n", xPortGetCoreID());
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}

void task2_func(void* param) {
    while (1) {
        Serial.printf("核心%d: 任务2\n", xPortGetCoreID());
        vTaskDelay(pdMS_TO_TICKS(1500));
    }
}

void setup() {
    Serial.begin(115200);

    // xTaskCreatePinnedToCore：最后一个参数指定核心（0或1）
    xTaskCreatePinnedToCore(task1_func, "Task1", 4096, NULL, 1, &Task1, 0);  // 核心0
    xTaskCreatePinnedToCore(task2_func, "Task2", 4096, NULL, 1, &Task2, 1);  // 核心1
}

void loop() {
    // loop()默认运行在核心1，优先级1
    vTaskDelay(pdMS_TO_TICKS(5000));
}
⚠️ 双核注意事项：WiFi任务运行在核心0，不要在核心0放高实时性任务。核心1留给你的应用逻辑。共享变量必须用互斥量保护。

🔌 ESP32 外设操作

GPIO

// Arduino
#define BTN_PIN 0   // BOOT按钮（低有效）
#define LED_PIN 2

void setup() {
    pinMode(LED_PIN, OUTPUT);
    pinMode(BTN_PIN, INPUT_PULLUP);
}

void loop() {
    if (digitalRead(BTN_PIN) == LOW) {
        digitalWrite(LED_PIN, !digitalRead(LED_PIN));
        delay(200);  // 简单消抖
    }
}
⚠️ ESP32 GPIO限制：GPIO 6~11连Flash（禁用！），GPIO 34~39只能输入（无上拉），GPIO 0是BOOT按钮，GPIO 2上电必须为低。可用GPIO：1,3,4,5,12~19,21~23,25~27,32~39。

ADC读取

// Arduino
void setup() {
    Serial.begin(115200);
    analogReadResolution(12);  // 12位（0~4095）
    analogSetAttenuation(ADC_11db);  // 量程0~3.3V
}

void loop() {
    int val = analogRead(34);  // GPIO34
    float voltage = val * 3.3 / 4095;
    Serial.printf("ADC=%d, 电压=%.2fV\n", val, voltage);
    delay(500);
}

// Arduino - LED呼吸灯
#define LED_PIN 2
#define PWM_FREQ 5000     // 5kHz
#define PWM_RES 8         // 8位分辨率（0~255）

    // 呼吸灯
    for (int duty = 0; duty <= 255; duty++) {
        ledcWrite(PWM_CH, duty);
        delay(5);
    }
    for (int duty = 255; duty >= 0; duty--) {
        ledcWrite(PWM_CH, duty);
        delay(5);
    }
}

📤 OTA 空中升级

OTA = Over-The-Air，通过WiFi远程更新固件，不用插线。

Arduino OTA（最简单）

#include <Arduino.h>
#include <WiFi.h>
#include <ArduinoOTA.h>

const char* ssid = "YourWiFi";
const char* password = "YourPassword";

void setup() {
    Serial.begin(115200);
    WiFi.begin(ssid, password);
    while (WiFi.status() != WL_CONNECTED) delay(500);

    ArduinoOTA.setHostname("esp32-mydevice");
    ArduinoOTA.setPassword("admin123");  // OTA密码
    ArduinoOTA.begin();

    Serial.printf("OTA就绪！IP: %s\n", WiFi.localIP().toString().c_str());
}

void loop() {
    ArduinoOTA.handle();  // 必须在loop中调用！
    // 你的其他代码...
}
OTA使用方式：
方式1：Arduino IDE → 工具 → 端口 → 找到"esp32-mydevice at IP" → 点上传
方式2：PlatformIO → pio run -t upload --upload-port IP地址
方式3：curl发送.bin文件（HTTP OTA）

ESP-IDF OTA（更专业）

// ESP-IDF OTA核心流程
#include "esp_ota_ops.h"

void ota_update(void) {
    const esp_partition_t *update_partition = esp_ota_get_next_update_partition(NULL);
    esp_ota_handle_t update_handle = 0;

    esp_ota_begin(update_partition, OTA_SIZE_UNKNOWN, &update_handle);

    // 从HTTP下载固件（伪代码）
    while (/* 还有数据 */) {
        esp_ota_write(update_handle, data_buf, data_len);
    }

    esp_ota_end(update_handle);
    esp_ota_set_boot_partition(update_partition);  // 设置下次启动分区
    esp_restart();  // 重启进新固件
}
⚠️ OTA回滚：如果新固件启动后自检失败，可以自动回滚到旧版本。ESP-IDF的 esp_ota_mark_app_valid_cancel_rollback() 标记当前固件为有效。

🔋 深度睡眠与低功耗

#include <Arduino.h>
#include <esp_sleep.h>

#define WAKEUP_PIN GPIO_NUM_0  // BOOT按钮唤醒

void setup() {
    Serial.begin(115200);

    // 检查唤醒原因
    esp_sleep_wakeup_cause_t reason = esp_sleep_get_wakeup_cause();
    if (reason == ESP_SLEEP_WAKEUP_EXT0) {
        Serial.println("按钮唤醒！");
    } else if (reason == ESP_SLEEP_WAKEUP_TIMER) {
        Serial.println("定时器唤醒！");
    } else {
        Serial.println("正常上电");
    }

    // 做完工作后进入深度睡眠
    Serial.println("3秒后进入深度睡眠...");
    delay(3000);

    // 配置唤醒源（二选一或同时）
    // 方式1：定时器唤醒（每60秒）
    esp_sleep_enable_timer_wakeup(60 * 1000000);  // 微秒

    // 方式2：GPIO唤醒（按钮）
    esp_sleep_enable_ext0_wakeup(WAKEUP_PIN, 0);  // 低电平唤醒

    esp_deep_sleep_start();  // 进入深度睡眠（到这儿就停了）
    // 下面的代码不会执行！
}

void loop() {
    // 深度睡眠不用loop
}

功耗对比

模式功耗唤醒时间保持状态
活跃~160mA—全部
Light Sleep~0.8mA<1msRAM保持
Deep Sleep~10μA~10ms仅RTC内存
Hibernation~5μA~100ms无
💡 RTC内存：深度睡眠时普通RAM断电，但RTC内存保持。用 RTC_DATA_ATTR 声明的变量在深睡后还在：
RTC_DATA_ATTR int bootCount = 0;

🔧 esptool 使用

esptool是ESP32的官方烧录工具，已通过pip安装。

# 查看ESP32信息（芯片型号/Flash大小/MAC地址）
esptool.py --port COM3 chip_id

# 烧录固件
esptool.py --port COM3 --baud 921600 write_flash 0x0 app.bin

# 擦除Flash（恢复出厂）
esptool.py --port COM3 erase_flash

# 读取Flash内容（256KB）
esptool.py --port COM3 read_flash 0x0 0x40000 flash_dump.bin

# 合并多个bin（bootloader+partition+app）
esptool.py --chip esp32 merge_bin \
    -o merged.bin \
    0x0 bootloader.bin \
    0x8000 partitions.bin \
    0x10000 app.bin

# 设置Flash模式和大小
esptool.py --port COM3 --baud 921600 write_flash \
    --flash_mode dio --flash_size 4MB 0x0 app.bin

ESP32启动模式

GPIO0GPIO2模式说明
1（高）任意Flash启动正常运行
0（低）0（低）下载模式烧录固件
⚠️ 自动进入下载模式：现代ESP32开发板有自动复位电路（EN+GPIO0），esptool会自动控制，不用手动按按钮。老板子需要手动按住BOOT再按EN。

🔍 常见问题

问题1：不断重启（看Guru Meditation Error）

# ESP32崩溃时会打印backtrace
# Guru Meditation Error: Core  0 panic'ed (LoadProhibited)
# Backtrace: 0x400D12A7:0x3FFB1D70 0x400D12D1:0x3FFB1D90 ...

# 解码（需要addr2line或idf.py monitor自动解码）
# Arduino：Tools → Core Debug Level → Verbose
# ESP-IDF：idf.py monitor（自动解码）

# 常见原因：
# 1. 空指针（最常见）
# 2. 栈溢出（任务栈太小，加大4096→8192）
# 3. 看门狗超时（长时间关中断或死循环）
# 4. Flash读取失败（GPIO6~11被占用）

问题2：WiFi连不上

# 检查清单：
# 1. SSID/密码是否正确
# 2. 路由器是否2.4GHz（ESP32不支持5GHz！）
# 3. 路由器是否开了MAC过滤
# 4. WiFi事件回调是否注册
# 5. nvs_flash是否初始化

问题3：烧录失败

# A fatal error occurred: Failed to connect to ESP32: Timed out
# 解决：
# 1. 按住BOOT按钮 → 点烧录 → 等Connecting时松开
# 2. 检查COM口是否正确（设备管理器→端口）
# 3. 换USB线（有些线只能充电不能传数据）
# 4. 降低波特率：--baud 115200

问题4：内存不足

# 查看堆内存
ESP_LOGI("MEM", "空闲堆: %d", esp_get_free_heap_size());
ESP_LOGI("MEM", "最小空闲: %d", esp_get_minimum_free_heap_size());

# Arduino
Serial.printf("空闲堆: %d\n", ESP.getFreeHeap());
