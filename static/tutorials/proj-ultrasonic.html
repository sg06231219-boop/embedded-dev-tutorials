🔬 HC-SR04 工作原理

超声波测距原理

HC-SR04 使用超声波飞行时间（Time of Flight, ToF）原理：

                    被测物体
                       ▼
                       │
超声波脉冲 → ●━━━━━━━━━━━━━━━━━━━━━► ╱ ╲
                       │             ╱     ╲ 反射
                       │            ╱       ╲
                       │           ╱         ╲
                       │          ╱           ╲
                       │         ╱  超声波束   ╲
                       │        ╱               ╲
发射器 → ◄━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━► → 接收器
         │                                            │
         │ ←─────────── 往返距离 ─────────────→       │
         │                                            │
         └────────────────────────────────────────────┘

原理公式：
  距离 = 声速 × 飞行时间 ÷ 2

声速 = 340 m/s = 0.034 cm/μs

示例：
  如果 Echo 高电平持续 588μs
  距离 = 0.034 × 588 / 2 ≈ 10 cm

HC-SR04 时序图

MCU Trig ──┐                    ┌────────────
           │   至少10μs        │ Echo高电平宽度
           └────────────────────┘    │
                                    ▼
MCU Echo ───────────────────────────────────────────
           │         ↑ Echo上升沿  ↑ Echo下降沿
           │         │            │
           │    触发测量      测量此宽度
           │         │            │
           │    开始捕获      停止捕获
           │         │            │
           └─────────┴────────────┘
              ≈ 200μs    高电平持续时间
              (模块内部发射)

HC-SR04 参数：
  Trig 输入: 10μs 高电平脉冲
  工作电压: 5V（VCC和Echo都需要5V）
  测量范围: 2cm ~ 400cm
  测量角度: ±15°
  盲区:     < 2cm（无法测量）

距离计算公式

设 Echo 高电平持续时间为 T (微秒)

声速 = 340 m/s = 0.034 cm/μs = 34.3 cm/ms

单程距离 = 声速 × 时间 / 2
         = 0.034 × T / 2
         = 0.017 × T  cm

用定时器计数值计算（假设 PSC=72, 1MHz计数）:
  T(μs) = 定时器计数值
  
  distance_cm = T × 0.017
  distance_cm = T × 17 / 1000
  distance_cm = (T * 17) / 1000

常用值速查：
  T=58μs   → 1cm
  T=232μs  → 4cm
  T=580μs  → 10cm
  T=5800μs → 100cm
  T=11600μs→ 200cm
  T=23200μs→ 400cm (最大测量距离)

🔌 接线图

STM32F103C8T6          HC-SR04 模块
─────────────          ───────────
PA0 (GPIO Out) ──────── Trig (触发输入)
PA1 (TIM2_CH2 IC) ───── Echo (回响输出)⚠需要分压或直接5V接单片机

HC-SR04 模块           电源
─────────────          ────
VCC          ──────── 5V (必须5V!)
GND          ──────── GND (与STM32共地)

⚠ 关于Echo电压：
  HC-SR04的Echo输出是5V高电平，STM32是3.3V电平
  方案1（推荐）: 在Echo和PA1之间串联1kΩ+2kΩ电阻分压
  方案2: 直接接PA1（部分STM32有保护二极管，5V可能损坏）
  方案3: 用电压转换模块
  
分压电路:
  HC-SR04 Echo ───┬── 1kΩ ───┬── PA1 (STM32)
                  │          │
                  └── 2kΩ ───┴── GND
                  
  5V × (2kΩ / 3kΩ) ≈ 3.3V ✓

STM32F103C8T6          蜂鸣器 (可选)
─────────────          ───────
PA2 (GPIO)  ──────── 蜂鸣器信号
GND          ──────── 蜂鸣器GND
⚠ 重要警告：
- HC-SR04 必须使用5V供电，3.3V会导致模块工作不正常或无输出
- Echo输出是5V，必须做电平转换才能接到STM32的3.3V引脚！
- 如果没有分压电路，可能烧毁STM32的GPIO引脚！

📝 分步操作指引

Step 1：CubeMX 配置
1. PA0 → GPIO_Output（Trig触发）
2. PA1 → TIM2_CH2 Input Capture（测量Echo）
3. TIM2 Prescaler=72 (1MHz), Period=65535
4. TIM2_CH2 → Input Capture direct mode
5. 配置NVIC使能TIM2中断
6. 配置PA2为GPIO_Output（蜂鸣器）
Step 2：Trig 触发信号
每60ms以上，发送一个10μs的高电平脉冲给Trig。
Step 3：配置输入捕获
使用上升沿触发开始捕获，下降沿触发停止捕获。
Step 4：计算距离
distance = high_time × 17 / 1000（单位：cm）
Step 5：添加报警功能
当距离小于阈值时，蜂鸣器报警。

💻 完整代码

main.c — 超声波测距完整实现

/* USER CODE BEGIN Includes */
#include <stdio.h>
#include <stdint.h>
/* USER CODE END Includes */

/* USER CODE BEGIN PV */

// 超声波测量状态
typedef enum {
    US_STATE_IDLE = 0,
    US_STATE_WAITING_ECHO,  // 等待Echo上升沿
    US_STATE_MEASURING,     // 测量中
    US_STATE_DONE           // 测量完成
} US_State_t;

// 测量数据
volatile US_State_t g_us_state = US_STATE_IDLE;
volatile uint32_t g_us_start_tick = 0;  // Echo上升沿时刻
volatile uint32_t g_us_end_tick = 0;    // Echo下降沿时刻
volatile uint16_t g_distance_cm = 0;    // 测量距离(cm)
volatile uint8_t g_us_ready = 0;        // 测量完成标志

// 配置参数
#define US_TRIG_PORT GPIOA
#define US_TRIG_PIN  GPIO_PIN_0
#define US_TIMEOUT    25000  // 超时阈值（约4.3m/最大距离）
#define US_MIN_DIST   2      // 最小测量距离（盲区）
#define US_MAX_DIST   400     // 最大测量距离
#define BUZZER_PORT   GPIOA
#define BUZZER_PIN    GPIO_PIN_2
#define ALERT_THRESHOLD 10   // 报警阈值（cm）

// 定时器句柄（TIM2用于输入捕获）
extern TIM_HandleTypeDef htim2;

// 上次Trig发送时间
volatile uint32_t g_last_trig_tick = 0;
/* USER CODE END PV */

/* USER CODE BEGIN 0 */

// 发送超声波触发脉冲 (至少10μs高电平)
void US_Trigger(void)
{
    // 确保Trig为低
    HAL_GPIO_WritePin(US_TRIG_PORT, US_TRIG_PIN, GPIO_PIN_RESET);
    HAL_Delay(1);  // 等待稳定
    
    // 发送10μs高电平脉冲
    HAL_GPIO_WritePin(US_TRIG_PORT, US_TRIG_PIN, GPIO_PIN_SET);
    // 使用定时器延时（更精确）
    uint32_t start = HAL_GetTick();
    while (HAL_GetTick() - start < 1) {
        __NOP(); __NOP(); __NOP(); __NOP();
        __NOP(); __NOP(); __NOP(); __NOP();
        __NOP(); __NOP(); __NOP(); __NOP();
        __NOP(); __NOP(); __NOP(); __NOP();
    }
    HAL_GPIO_WritePin(US_TRIG_PORT, US_TRIG_PIN, GPIO_PIN_RESET);
    
    g_us_state = US_STATE_WAITING_ECHO;
}

// TIM2 输入捕获中断回调
void HAL_TIM_IC_CaptureCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance != TIM2) return;
    
    uint32_t current_tick = __HAL_TIM_GET_COUNTER(htim);
    
    if (htim->Channel == HAL_TIM_ACTIVE_CHANNEL_2) {
        
        if (g_us_state == US_STATE_WAITING_ECHO) {
            // 检测到上升沿（Echo变高）
            if (HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_1) == GPIO_PIN_SET) {
                __HAL_TIM_SET_COUNTER(htim, 0);  // 清零定时器
                g_us_start_tick = 0;
                g_us_state = US_STATE_MEASURING;
            }
        }
        else if (g_us_state == US_STATE_MEASURING) {
            // 检测到下降沿（Echo变低）
            if (HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_1) == GPIO_PIN_RESET) {
                g_us_end_tick = current_tick;
                
                // 计算距离
                uint32_t high_time = g_us_end_tick - g_us_start_tick; // μs
                
                // distance = high_time × 0.017 (cm)
                // 精确公式: distance_cm = (high_time * 34) / 200
                //         = (high_time * 17) / 100
                if (high_time > US_TIMEOUT || high_time == 0) {
                    g_distance_cm = 0; // 超时/无效
                } else {
                    g_distance_cm = (high_time * 17) / 100; // cm
                }
                
                g_us_ready = 1;
                g_us_state = US_STATE_IDLE;
            }
        }
    }
}

// TIM2 更新中断（用于超时检测）
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM2) {
        if (g_us_state == US_STATE_MEASURING) {
            // 测量超时（约25000μs × 1μs = 25ms）
            g_distance_cm = 0;
            g_us_ready = 1;
            g_us_state = US_STATE_IDLE;
        }
    }
}

// 报警控制
void Alert_Check(uint16_t dist)
{
    static uint8_t alert_on = 0;
    
    if (dist > 0 && dist < ALERT_THRESHOLD) {
        if (!alert_on) {
            alert_on = 1;
            HAL_GPIO_WritePin(BUZZER_PORT, BUZZER_PIN, GPIO_PIN_SET);
        }
    } else {
        if (alert_on) {
            alert_on = 0;
            HAL_GPIO_WritePin(BUZZER_PORT, BUZZER_PIN, GPIO_PIN_RESET);
        }
    }
}

// printf 重定向
#ifdef __GNUC__
int __io_putchar(int ch)
#else
int fputc(int ch, FILE *f)
#endif
{
    HAL_UART_Transmit(&huart1, (uint8_t *)&ch, 1, HAL_MAX_DELAY);
    return ch;
}

/* USER CODE END 0 */

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_TIM2_Init();
    MX_USART1_UART_Init();

    /* USER CODE BEGIN 2 */
    // 启动TIM2输入捕获 (Channel 2)
    HAL_TIM_IC_Start_IT(&htim2, TIM_CHANNEL_2);
    
    // 蜂鸣器初始关闭
    HAL_GPIO_WritePin(BUZZER_PORT, BUZZER_PIN, GPIO_PIN_RESET);
    
    // 初始化Trig为低
    HAL_GPIO_WritePin(US_TRIG_PORT, US_TRIG_PIN, GPIO_PIN_RESET);
    
    printf("Ultrasonic Distance Meter Ready!\r\n");
    printf("Measuring every 100ms...\r\n");
    /* USER CODE END 2 */

    /* USER CODE BEGIN WHILE */
    while (1)
    {
        // 每100ms触发一次测量
        static uint32_t last_measure_tick = 0;
        if (HAL_GetTick() - last_measure_tick >= 100) {
            last_measure_tick = HAL_GetTick();
            US_Trigger();
        }
        
        // 处理测量结果
        if (g_us_ready) {
            g_us_ready = 0;
            
            if (g_distance_cm > 0) {
                printf("Distance: %3d cm", g_distance_cm);
                
                // 简易条形图显示
                uint8_t bar_len = (g_distance_cm > 50) ? 50 : g_distance_cm;
                printf(" |");
                for (uint8_t i = 0; i < bar_len; i++) printf("#");
                printf("\r\n");
                
                // 报警检查
                Alert_Check(g_distance_cm);
            } else {
                printf("Distance: OUT OF RANGE\r\n");
            }
        }
        
        HAL_Delay(10);
    }
    /* USER CODE END WHILE */
}

htim2.Instance = TIM2;
htim2.Init.Prescaler = 72 - 1;      // 72MHz/72 = 1MHz → 1μs/计数
htim2.Init.CounterMode = TIM_COUNTERMODE_UP;
htim2.Init.Period = 0xFFFF;          // 最大值65535
htim2.Init.ClockDivision = TIM_CLOCKDIVISION_DIV1;
htim2.Init.AutoReloadPreload = TIM_AUTORELOAD_PRELOAD_DISABLE;

// 输入捕获配置 Channel 2
sConfigIC.ICPolarity = TIM_INPUTCHANNELPOLARITY_BOTH;
  // BOTH = 上升沿和下降沿都捕获，在中断中判断当前电平

sConfigIC.ICFilter = 0x0;

❓ 常见问题排查

问题1：距离始终为0

最可能的原因：
1. Echo接错了 — 最常见！确认PA1（TIM2_CH2）接的是Echo，不是Trig。
2. 电平转换没做好 — 5V Echo直接接STM32可能损坏引脚。
3. Trig没有真正发出10μs脉冲 — 用示波器或逻辑分析仪检查PA0。
4. HC-SR04没有5V供电 — 用万用表测量VCC引脚电压是否接近5V。
5. 输入捕获中断没有使能 — 检查NVIC配置。

问题2：测量值不准确

误差来源与解决方案：
1. 温度影响声速 — 声速随温度变化：
// 温度补偿公式
// 声速(m/s) = 331.4 + 0.59 × 温度(℃)

#define TEMP_CORRECTION  // 启用温度补偿

#ifdef TEMP_CORRECTION
    #define ROOM_TEMP 25  // 当前温度
    #define SOUND_SPEED (3314 + 59 * ROOM_TEMP) / 10  // 单位: 0.1cm/ms
    // 距离 = high_time × SOUND_SPEED / 2000
    g_distance_cm = (high_time * SOUND_SPEED) / 2000;
#else
    g_distance_cm = (high_time * 17) / 100;  // 默认20℃下的近似
#endif
2. 被测物体表面不是垂直的（斜面反射信号弱）
3. 测量距离超出规格（>400cm）
4. 目标物体太小或吸声材料
5. 多次测量取平均值

问题3：HC-SR04 盲区问题

盲区 < 2cm 是物理限制，无法消除。
解决方案：
- 如果需要测 < 2cm，使用红外测距传感器（如SHARP GP2Y0A21）
- HC-SR04数据手册标注最小距离2cm，实际可能在3~5cm才能可靠测量
- 代码中对 < US_MIN_DIST 的值过滤掉或标记为"太近"

问题4：串口只打印一次数据

检查是否只触发了一次US_Trigger()。应该周期性调用，参考代码中的100ms间隔。
另外检查g_us_ready标志是否正确清零了。

问题5：Echo高电平时间计算错误

确认定时器PSC=72，时钟=72MHz：
- 计数频率 = 72000000 / 72 = 1000000 Hz = 1MHz
- 每个计数 = 1μs
- 如果PSC设置成其他值，公式需要相应调整：
// 通用公式
uint32_t count_freq = 72000000 / (PSC + 1);
uint32_t high_time_us = (g_us_end_tick - g_us_start_tick);
uint32_t distance_cm = (high_time_us * 34) / 200; // 除以2再×100000转cm

❓ 常见问题 FAQ

Q: 测距不准？

A: 1)声速随温度变化(331.4+0.6T m/s)需温度补偿 2)超过4m回波太弱 3)盲区<2cm无法测

Q: 多传感器互相干扰？

A: 时分复用：依次触发每个传感器，等回波结束后再触发下一个。
