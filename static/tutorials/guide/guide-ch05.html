第5章：烧录与调试

📖 小白完全指南 · 5/14 | 阅读时间约3分钟

编译生成.bin/.hex文件后，需要用烧录器把它写进单片机的Flash里。调试则是让程序在芯片里暂停、单步执行、查看变量，像"慢放"一样找Bug。

💡 类比：烧录=把程序刻到芯片的DNA里（写死在Flash中），调试=给程序做体检（停住→检查→继续→再检查）。

OpenOCD 烧录

⚠️ 前提条件：需要先安装OpenOCD。由于GitHub下载被墙，请尝试以下方式：
① 搜索"OpenOCD xpack 国内镜像"下载
② 从淘宝购买开发板卖家提供的资料包（通常包含OpenOCD）
③ 让有VPN的朋友帮忙下载
下载后解压到 D:\EmbeddedTools\OpenOCD，把bin目录加PATH即可。
# 烧录命令
openocd -f interface/stlink.cfg -f target/stm32f1x.cfg     -c "program build/blink.elf verify reset exit"

# 只擦除
openocd -f interface/stlink.cfg -f target/stm32f1x.cfg     -c "init; reset halt; flash erase_sector 0 0 last; reset run; shutdown"

ST-Link 烧录（st-flash）

⚠️ 前提条件：需要安装stlink工具。下载地址：github.com/stlink-org/stlink/releases（同GitHub，可能被墙）。替代方案：使用STM32CubeIDE自带的st-link驱动。
# 安装stlink工具（需编译或下载预编译版）
# github.com/stlink-org/stlink/releases
# 下载后解压，把bin目录加PATH

# 烧录BIN文件
st-flash write blink.bin 0x08000000

# 烧录HEX文件
st-flash --format ihex write blink.hex

# 擦除整片Flash
st-flash erase

# 读取Flash内容
st-flash read flash_dump.bin 0x08000000 0x10000

串口ISP烧录（零硬件成本！）

💡 没有ST-Link？用USB转TTL也能烧！STM32内置Bootloader支持串口下载，只需要CH340模块（¥5）。速度比ST-Link慢，但零额外成本。
# 方式1：FlyMcu（最简单的Windows串口烧录工具）
# 下载：www.flymcu.com（免费，中文界面）
# 使用：
# 1. BOOT0跳线接1（或按住BOOT0键上电）
# 2. 打开FlyMcu → 选择COM口和hex文件
# 3. 点击"开始编程"
# 4. 烧录完成后BOOT0接回0，按复位键运行

# 方式2：stm32flash（命令行工具）
# 下载：github.com/ARMin stm32flash
# 使用：
stm32flash -w blink.hex -v -g 0x0 COM3
# -w 写入  -v 校验  -g 0x0 写完从0地址运行  COM3 你的串口
⚠️ 串口烧录要点：
• BOOT0必须接高电平(1)才能进入下载模式
• 烧录完成后BOOT0接回低电平(0)，按复位键才能运行程序
• 有些最小系统板有BOOT0跳线帽，方便切换
• 每次烧录都要切换BOOT0，不如ST-Link方便，但学习阶段完全够用

GDB调试

⚠️ 说明：GDB调试需要硬件调试器（ST-Link/DAP-Link/J-Link）。如果你暂时没有调试器，可以先跳过GDB部分，用printf调试（第11章详解）。下面提供两种调试方案：方案A用OpenOCD（免费开源，DAP-Link/ST-Link通用），方案B用st-util（ST-Link专用，更简单）。

方案A：OpenOCD + GDB（推荐，通用性更好）

# 终端1：启动OpenOCD作为GDB Server
openocd -f interface/stlink.cfg -f target/stm32f1x.cfg
# 看到 "Info : stm32f1x.cpu: hardware has 6 breakpoints" 就说明连接成功

# 终端2：启动GDB客户端
arm-none-eabi-gdb build/blink.elf

# GDB常用命令（两方案通用）
(gdb) target remote :3333         # 连接OpenOCD（注意端口3333）
(gdb) monitor reset halt          # 复位并停住
(gdb) load                         # 烧录程序到芯片
(gdb) break main                   # 在main函数设断点
(gdb) continue                     # 继续运行
(gdb) step                         # 单步（进入函数）
(gdb) next                         # 单步（跳过函数）
(gdb) print variable               # 打印变量值
(gdb) watch variable               # 监视变量变化
(gdb) backtrace                    # 查看调用栈
(gdb) info registers               # 查看CPU寄存器
(gdb) x/10x 0x20000000            # 查看内存（16进制）
(gdb) delete                       # 删除所有断点
(gdb) quit                         # 退出

方案B：st-util + GDB（ST-Link专用，简单）

⚠️ 前提：需安装stlink工具：github.com/stlink-org/stlink/releases（GitHub可能被墙，替代：STM32CubeIDE自带st-link驱动）
# 终端1：启动GDB Server
st-util -p 4242

# 终端2：启动GDB客户端
arm-none-eabi-gdb build/blink.elf

# GDB连接st-util
(gdb) target remote localhost:4242
(gdb) load
(gdb) break main
(gdb) continue
# 其余命令与方案A相同
💡 没有调试器怎么办？用串口printf调试！第11章详细介绍了printf重定向方法，足够解决80%的问题。买ST-Link只需¥10，建议入门后尽快入手。

VS Code Cortex-Debug 配置

⚠️ 前提：① 已安装Cortex-Debug插件 ② 已安装OpenOCD或stlink工具 ③ ST-Link调试器已连接

方案A配置：OpenOCD（推荐，与GDB方案A配套）

// .vscode/launch.json - OpenOCD方案
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Cortex Debug (OpenOCD)",
            "cwd": "${workspaceFolder}",
            "executable": "build/blink.elf",
            "request": "launch",
            "type": "cortex-debug",
            "runToEntryPoint": "main",
            "serverType": "openocd",
            "serverArgs": ["-f", "interface/stlink.cfg", "-f", "target/stm32f1x.cfg"],
            "device": "STM32F103C8",
            "svdFile": "STM32F103.svd"
        }
    ]
}

方案B配置：st-util（与GDB方案B配套）

// .vscode/launch.json - st-util方案
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Cortex Debug (st-util)",
            "cwd": "${workspaceFolder}",
            "executable": "build/blink.elf",
            "request": "launch",
            "type": "cortex-debug",
            "runToEntryPoint": "main",
            "serverType": "stutil",
            "serverArgs": ["-p", "4242"],
            "device": "STM32F103C8",
            "svdFile": "STM32F103.svd"
        }
    ]
}
🔍 常见问题：GDB连接不上
1. ST-Link驱动是否安装？st-info --probe 测试
2. 是否有其他程序占用了ST-Link？（关闭STM32CubeIDE等）
3. 接线是否正确？SWDIO/SWCLK/GND三根线必须接
