第10章：PCB与硬件基础

📖 小白完全指南 · 10/14 | 阅读时间约12分钟

嵌入式工程师不能只会写代码——硬件是软件的肉体。不懂硬件的程序员，遇到问题只会"可能是硬件问题"，然后束手无策。

💡 类比：原理图=建筑图纸（标注了每面墙、每根管线的位置），PCB=盖好的房子（电路板的物理实现），数据手册=建材规格表（每个元件的详细参数）。

从原理图到PCB的设计流程

一个嵌入式项目的硬件设计通常遵循以下流程：需求分析 → 芯片选型 → 原理图设计 → PCB Layout → 打样 → 焊接调试。每一步都很重要。

Step 1：需求分析 —— 确定输入/输出信号类型、数量、电源需求、尺寸限制
Step 2：芯片选型 —— 选择MCU型号、外围IC（传感器、电源芯片、接口芯片）
Step 3：原理图设计 —— 画出所有元件的电气连接关系
Step 4：PCB Layout —— 将原理图转化为实际的电路板走线
Step 5：打样 —— 发给PCB厂制造样板（嘉立创5元5片是入门神器）
Step 6：焊接调试 —— 手工焊接或SMT贴片，上电测试

原理图阅读方法

步骤1：找电源——VCC/GND/VDD/3V3/5V，先搞清楚供电网络
步骤2：找MCU——大方框就是主芯片，周围引脚标了功能
步骤3：看信号流向——从左到右：输入→处理→输出
步骤4：看网络标号——相同标号=电气连接（即使没画线）

原理图常见符号识别

符号含义常见场景
VCC / VDD / 3V3 / +5V正电源芯片供电引脚
GND / VSS地（0V参考）所有芯片都需要接地
AGND / DGND模拟地 / 数字地混合信号电路（ADC等）
NCNo Connect（不连接）芯片未使用的引脚
RESET / RST / NRST复位引脚低电平有效，上拉到VCC
BOOT0 / BOOT1启动模式选择STM32特有，决定从哪启动
SWCLK / SWDIO调试接口ST-Link/J-Link烧录口
VBAT备份电池供电RTC时钟保持
⚠️ 原理图不是PCB！原理图上靠近的元件，在PCB上可能离得很远。原理图讲究逻辑清晰，PCB讲究电气性能。很多新手以为原理图连线短=PCB连线短，这是大误区。

PCB Layout基础

Layout是把原理图变成实际电路板的过程。好的Layout让电路稳定工作，差的Layout会让电路异常、干扰严重甚至根本不工作。

PCB层数选择

层数结构价格(10x10cm)适用场景
2层顶层走线+底层走线¥5/5片低速数字电路、简单模拟
4层信号-GND-VCC-信号¥30-50/5片高速数字、混合信号
6层信号-GND-信号-信号-VCC-信号¥80-150/5片DDR内存、FPGA、RF
⚠️ 2层板最容易出错：2层板没有完整的参考地层，走线时需要注意信号回路。4层板多几十块钱但信号质量好一个数量级。入门做2层，有经验后尽量用4层。

PCB走线规则

// PCB Layout黄金法则（从简单到进阶）
//
// 1. 线宽与电流关系 (1oz铜厚, 温升10°C)
//    0.25mm (10mil):  ~1A
//    0.5mm  (20mil):  ~2A
//    1.0mm  (40mil):  ~3.5A
//    2.0mm  (80mil):  ~6A
//    → 电源线至少0.5mm！大电流加宽或用多边形铺铜
//
// 2. 线间距 (信号完整性)
//    普通信号: 0.15mm (6mil) 就够
//    高压信号: 1mm/100V 安全间距
//    差分信号: 紧密耦合，等长等间距
//
// 3. 过孔 (Via)
//    信号过孔: 内径0.3mm/外径0.6mm (大多数PCB厂标准)
//    电源过孔: 内径0.5mm/外径1.0mm (载流能力更强)
//    尽量避免在高速信号上打过孔
//
// 4. 去耦电容布局
//    每个VDD引脚附近放一个100nF电容
//    电容离引脚越近越好 (<5mm)
//    走线：VDD → 电容 → 芯片引脚 (不要芯片→电容→VDD)

去耦电容深入解析

去耦电容(Decoupling Capacitor)是PCB设计中最重要的单个元件。没有它，数字电路基本无法稳定工作。

💡 去耦电容=芯片旁边的"小水库"：
芯片内部成千上万个晶体管同时翻转时，需要瞬间大电流（di/dt可达A/ns级）。PCB走线有电感，电流跟不上，电压就会跌落。去耦电容就像芯片旁的小水库，随时提供瞬时电流。100nF是最常用的值，因为对于0~100MHz范围的噪声有良好的去耦效果。
// 去耦电容的选择与布局
// 
// 容量选择:
//   100nF (0.1μF) X7R陶瓷电容 → 每个VDD引脚放一个（标准配置）
//   10μF  钽电容/陶瓷电容 → 每个芯片放1~2个（大水库）
//   1μF   陶瓷电容 → 中等频率去耦
//
// 电容类型:
//   X7R: 温度稳定性好(-55~+125°C), 常用
//   X5R: 温度范围稍窄(-55~+85°C), 便宜
//   NP0/C0G: 精度高、温度特性好, 但容量小, 高频用
//   Y5V: 精度差，不推荐
//
// 错误的布局（电容离引脚5cm远）：
//   [芯片] ======5cm走线====== [100nF] ----- [VCC]
//   ↑ 走线电感很大，电容等于没接！
//
// 正确的布局（电容紧挨引脚）：
//   [VCC] -- [100nF] -- [芯片VDD引脚]
//                    |
//   过孔到GND层 ←---┘
//
// 更优的做法（多层板）：
//   顶层: VDD走线 → 电容焊盘 → 芯片VDD焊盘
//   电容另一端 → 过孔 → 第二层GND平面
//   ↓ 回路面积最小，电感最低

电源与地的处理

// 电源设计要点
//
// 1. 星形电源拓扑 vs 总线拓扑
//    ★ 星形（推荐）: 从电源入口分别走线到各模块
//      [电源入口]→[MCU电源]
//               →[传感器电源]
//               →[通信模块电源]
//    优点：各模块干扰不互相耦合
//
//    × 总线形（不推荐）:
//      [电源入口]→[MCU]→[传感器]→[通信模块]
//    缺点：后级模块受前级干扰影响
//
// 2. 地平面设计
//    - 尽量用完整的地平面（不要切碎！）
//    - 数字地和模拟地分开，在电源入口处单点连接（0Ω电阻或磁珠）
//    - 高频信号下方必须有连续的地参考面
//
// 3. 电源滤波
//    [电源输入] → [TVS管] → [π型滤波] → [LDO/DCDC] → [去耦电容] → [负载]
//                          ↓
//                  10μF→磁珠→100nF
//    TVS: 防浪涌（TVS管击穿后短路，保护后级）

//    VDD(数字3.3V) → 磁珠/0Ω → VDDA(模拟3.3V)

电磁兼容(EMC)基础

EMC（电磁兼容性）分为两方面：EMI（不干扰别人）和EMS（不被别人干扰）。对于嵌入式系统，最常遇到的问题是"板子一接电机就不正常"或"附近一打电话就复位"。

// EMC设计的"三板斧"
//
// 1. 去耦电容（最重要的EMC措施）
//    每个IC的VCC-GND之间放0.1μF
//    多个IC的板子再放1个10μF钽电容
//
// 2. 地平面/接地
//    多层板用完整的GND平面层
//    双面板底层尽量铺铜做GND
//    信号线不要跨分割的地平面
//
// 3. 信号完整性
//    高速信号串接22~33Ω电阻（抑制反射）
//    长走线避免直角转弯（用45°或圆弧）
//    差分信号等长走线
//    I2C/SPI总线串接小电阻(10~100Ω)抑制振铃
🔍 常见EMC问题与解决方法

问题1：附近手机来电时板子复位
原因：GSM信号（900MHz/1800MHz）通过IO线耦合进来，干扰RESET引脚
解决：① RESET引脚对地加100nF电容 ② 敏感信号线两端对地加22pF电容 ③ 检查是否有长导线当天线

问题2：控制电机时单片机跑飞
原因：电机电刷火花产生宽频谱电磁干扰
解决：① 电机两端并联104电容(100nF) ② 电机外壳接地 ③ MCU与电机驱动之间用光耦隔离 ④ 驱动电路和MCU电路分开供电

问题3：ADC读数抖动厉害
原因：数字噪声窜入模拟电路
解决：① 模拟地与数字地分开，单点连接 ② VDDA加LC滤波 ③ ADC输入加RC低通滤波(R=1K, C=100nF)

万用表使用

# 测电压
红笔接VDD，黑笔接GND → 读数应为3.3V或5V

# 测通断（蜂鸣档）
两根表笔分别接两端 → 蜂鸣=导通（短路），无声=断路

# 测电阻
断电！量程选对 → 读数应与色环/标称值一致

# 测电流
串联进电路 → 注意量程（mA档 or A档）

//

//     - 测AMS1117输出：应为3.3V（LDO坏？短路？）

//
// 第4步：测BOOT引脚
//   BOOT0: 应为0V（从Flash启动）
//   BOOT1: 应为0V

//
// 第5步：用二极管档测IO口
//   红笔接GND，黑笔接IO → 应显示0.5~0.7V（内部ESD二极管压降）
//   如果显示0V → 该IO口可能已击穿短路

LDO功耗计算：为什么AMS1117会烫手踩坑经验
新手最常犯的错误：用AMS1117从12V降压到3.3V给STM32供电，然后发现"芯片烫得手都摸不了"。
// === LDO功耗计算 ===
// P_dissipation = (Vin - Vout) × I_load
//
// 案例1：常见错误配置
//   Vin=12V, Vout=3.3V, I=100mA (STM32+几个LED)
//   P = (12 - 3.3) × 0.1 = 0.87W
//   → AMS1117 SOT-223封装热阻JA≈90°C/W
//   → 温升 = 0.87W × 90°C/W ≈ 78°C
//   → 环境25°C + 78°C = 103°C ← 手摸上去立刻起泡！
//
// 案例2：正确配置
//   Vin=5V (USB), Vout=3.3V, I=100mA
//   P = (5 - 3.3) × 0.1 = 0.17W
//   → 温升 ≈ 15°C → 40°C ← 微温，正常
//
// 案例3：大电流场景
//   Vin=5V, Vout=3.3V, I=500mA (STM32+ESP32+LCD)
//   P = (5 - 3.3) × 0.5 = 0.85W
//   → 温升 ≈ 77°C
//   → 此时应该用DC-DC（效率85%+），而非LDO

// === LDO选型建议 ===
// 电流<100mA, 压差<2V   → AMS1117 (经济)
// 电流<300mA, 压差<1V   → ME6211/XC6206 (低压差)
// 电流>500mA            → MP1584/MP2307 等DC-DC
// 对噪声敏感的模拟电路 → LDO（纹波极低）
// 电池供电设备         → DC-DC（效率优先）

电平转换电路：3.3V与5V的桥梁

电平转换方案对比必学踩坑经验
STM32是3.3V器件，但很多外设（LCD1602、超声波模块HC-SR04、Arduino传感器）是5V的。直接连接可能烧毁IO口。以下是几种实用的电平转换方案。
// === 方案1：MOSFET电平转换（双向，最常用）===
// 适用：I2C、单总线、任何双向信号
// 芯片：BSS138 + 2个10K上拉电阻（或直接用4通道模块TXS0104E）
//
//   3.3V侧           5V侧
//    │                │
//    R1(10K)         R2(10K)
//    │                │
// SDA_3V3 ──┬─ S ─── D ──┬── SDA_5V  (BSS138 N-MOSFET)
//            │     G      │
//            └────────────┘
//
// 原理：当3.3V侧拉低→MOSFET导通→5V侧也被拉低
//      当5V侧拉低→体二极管导通→源极拉低→MOSFET导通→3.3V侧拉低
//      双方都释放→上拉电阻把两边拉到各自电源电压
// 优点：双向、便宜(¥0.2/路)
// 缺点：速度受上拉电阻限制（I2C 400KHz OK, SPI高速不行）

// === 方案2：专用电平转换芯片（简单可靠）===
// TXS0108E: 8通道双向电平转换，自动方向检测
//   1.2V-3.6V (A侧) ↔ 1.65V-5.5V (B侧)
//   最大速率: 60Mbps (推挽) / 2Mbps (开漏)
//   价格: ¥2-5
//
// 接线极简：
//   A1-A8 接3.3V侧信号
//   B1-B8 接5V侧信号
//   OE拉高使能
//   VCCA接3.3V, VCCB接5V
//   完成！不需要任何方向控制

// === 方案3：电阻分压（单向，仅3.3V→5V输出或5V→3.3V输入）===
// 场景A：5V信号→3.3V MCU输入
//   5V_信号 ──[R1=1.8K]──┬──[R2=3.3K]── GND
//                        └──→ MCU引脚 (~3.3V)
//   Vout = 5 × 3.3/(1.8+3.3) ≈ 3.23V ✓
//
// 场景B：3.3V MCU→5V设备输入（检查5V设备VIH）
//   多数5V CMOS器件VIH=0.7×VDD=3.5V → 3.3V不够！
//   此时不能直接连，需用MOSFET方案或74HCT系列（TTL输入兼容）

// === 方案4：74HCT系列（3.3V→5V单向输出专用）===
// 74HCT245: 8位总线收发器
//   输入阈值: VIH=2.0V (TTL兼容！3.3V刚好满足)
//   VCC=5V供电，输出=5V
//   优点：驱动能力强(±6mA)、简单可靠
//   缺点：单向
⚠️ 关键判断：3.3V输出能否驱动5V输入？查5V器件数据手册的VIH(min)。如果VIH(min)≤3.3V×0.7≈2.3V，则可以直连。如果VIH(min)>3.3V（大多数CMOS器件），则需要电平转换。

晶振电路设计：从不起振到稳定时钟

晶振负载电容计算必学不起振第一坑
"程序烧录进去了，但板子不跑"——90%是因为晶振问题。晶振电路虽然简单（晶振+2个电容），但电容值选错就会不起振。
// === 晶振负载电容计算 ===
// CL = (C1 × C2) / (C1 + C2) + C_stray
// 其中：
//   CL = 晶振标称负载电容（如12.5pF, 20pF）
//   C1, C2 = PCB上的负载电容
//   C_stray = PCB走线杂散电容（通常3-5pF）
//
// 示例：STM32常用8MHz晶振，CL=20pF
//   设C1=C2=C, C_stray=5pF
//   CL = C/2 + 5 = 20pF
//   → C = 30pF
//   → 选27pF或33pF都可以（有一定容差）
//
// 如果CL=12.5pF（低负载晶振）:
//   12.5 = C/2 + 5
//   → C = 15pF
//   → 选15pF或18pF
//
// === 常见晶振不起振的原因 ===
// 1. 负载电容选错（最常见！）
//    买的是20pF晶振，配了22pF电容 ← 可能行
//    买的是12.5pF晶振，配了22pF电容 ← 偏差大，可能不起振
//
// 2. 晶振离MCU太远
//    晶振和两个电容应该紧挨MCU的OSC_IN/OSC_OUT引脚
//    走线超过10mm就可能出问题
//    PCB走线越短越好，下面不要走其他信号线
//
// 3. 晶振下方铺了太多铜皮
//    晶振下方的GND铺铜会增加C_stray → 等效负载电容偏大
//    在晶振下方挖空地平面（如2mm×5mm区域）
//
// 4. 用了面包板
//    面包板杂散电容大（~10pF），高频晶振几乎必然不起振
//    8MHz在面包板上算勉强，16MHz+别想在面包板上跑
//
// 5. 32.768KHz晶振负载电容特殊
//    RTC晶振负载电容通常6pF或12.5pF
//    常见错误：配了22pF电容 ← CL=12.5pF→C≈15pF，22pF偏大！

// === 晶振选型建议 ===
// 频率      | 常用负载电容 | 推荐电容  | 用途
// 8MHz     | 20pF         | 27-33pF  | STM32 HSE
// 12MHz    | 20pF         | 27-33pF  | STM32 HSE
// 16MHz    | 16-20pF      | 22-30pF  | STM32 HSE
// 32.768KHz| 6-12.5pF     | 10-18pF  | RTC
// 25MHz    | 18-20pF      | 27-30pF  | STM32 HSE / Ethernet
💡 检测晶振是否起振：用万用表DC档测OSC_IN电压，起振后约VDD/2（~1.6V）；不起振则为0V或VDD。或用示波器看是否有正弦波。更精确：用手指触碰晶振外壳，起振的晶振被触碰时波形会抖动（频率微变）。

ESD保护电路设计

ESD保护：防止静电打死芯片重要生产必备
冬天脱毛衣的瞬间，人体可带几千到上万伏的静电。如果没有ESD保护，一个触摸就能让MCU的IO口永久损坏。所有暴露给用户的接口（USB、按键、插针、SD卡座）都需要ESD保护。
// === ESD保护器件选择 ===
//
// 1. TVS管（瞬态电压抑制二极管）
//   原理：超过击穿电压时，TVS管在皮秒级导通，把浪涌导入地
//   选型要点：
//   - 工作电压VRWM > 信号电压（3.3V供电→选VRWM≥5V的）
//   - 钳位电压VC越低越好（保护效果越好）
//   - 功率: USB选400W+, 普通IO选100W+
//
// 推荐型号：
//   3.3V IO: PESD3V3L1BA (VRWM=3.3V, VC=10V, 电容0.5pF)
//   USB D+/D-: PESD5V0L1BA (VRWM=5V, VC=10V)
//   USB VBUS: SMAJ5.0A (VRWM=5V, 功率400W)
//   12V电源: SMBJ12A (VRWM=12V, 功率600W)
//
// === ESD保护布局 ===
// USB接口 ESD保护电路：
//
//   USB插座 → [共模扼流圈] → [TVS阵列] → MCU
//   VBUS ─┬─ 100nF ─ GND ── TVS(SMAJ5.0A)
//         │
//   D+  ──┬── R(22Ω) ── TVS(PESD5V0L1BA) ── GND
//         │
//   D-  ──┬── R(22Ω) ── TVS(PESD5V0L1BA) ── GND
//         │
//   GND ──┴───────────────┬─ 1MΩ ─ GND(板内地)
//                         └─ 1nF/2KV ─ 外壳地
//
// 关键：TVS管必须靠近接口放置（<5mm），走线先经TVS再到芯片
//        串接的小电阻限制浪涌电流
//        GND通过RC网络隔离（防ESD电流走板内地）
⚠️ ESD设计口诀："先保护后功能"——信号进来第一步经过保护器件，然后再到功能电路。如果顺序反了（先到芯片再到TVS），TVS形同虚设。

运放在嵌入式中的应用基础

运放基础电路进阶模拟电路
嵌入式系统经常需要处理微弱的模拟信号（传感器mV级输出），直接进ADC信噪比太差。运放（Operational Amplifier）是最常用的信号调理器件。
// === 电路1：电压跟随器（缓冲器）===
// 用途：高阻抗输入→低阻抗输出，隔离信号源
//       Vin ─── +\\
//                ─┴─ Vout (Vout = Vin)
//       ┌──── -/
//       │
//       └──────┘ (反馈直接连接)
// 增益 = 1，输入阻抗 ≈ ∞，输出阻抗 ≈ 0
// 典型应用：分压电阻后加跟随器再进ADC
//   （避免ADC输入阻抗影响分压精度）

// === 电路2：同相放大器 ===
// 用途：放大微弱信号
//       Vin ─── +\\     Rf
//                ─┴──┬─[====]─┐
//       GND ─[Rg]─┘   │        │
//                     └────────┘ Vout
//       其中Rg接在-与GND之间，Rf接在Vout与-之间
//       Vout = Vin × (1 + Rf/Rg)
//
// 示例：电流检测（检流电阻上的mV信号放大到ADC范围）
//   0.1Ω检流电阻，1A时压降=0.1V=100mV
//   需要放大到0-3.3V量程
//   增益 = 3.3/0.1 = 33
//   Rf=100K, Rg=3.09K → 增益=1+100K/3.09K≈33.3

// === 电路3：差分放大器 ===
// 用途：测量两个电压的差值（如检流电阻两端）
//   Vout = (V2 - V1) × Rf/Rg
//   优点：共模干扰被抵消（常见于电流检测、桥式传感器）

// === 常用运放芯片 ===
// LM358: 双运放, 单电源3-32V, 带宽1MHz, ¥0.3
//   → 最便宜最常用的通用运放，速度慢但够用
// LMV358: 双运放, 轨到轨输出, 带宽1MHz, ¥0.5
//   → LM358的升级版，输出能到电源轨
// MCP6002: 双运放, 轨到轨输入输出, 带宽1MHz, ¥1
//   → 低电压(1.8-6V)，适合3.3V系统
// OPA2333: 双运放, 零漂移, 带宽350KHz, ¥15
//   → 精密运放，热电偶/桥式传感器专用

// === ADC输入保护与滤波 ===
// 信号→[RC低通]→[运放跟随]→[限幅保护]→[ADC]
//                         │
//   R=1K, C=100nF   3.3V─├─ BAT54S ─ GND
//   (fc=1.6KHz)          (肖特基二极管钳位)
//
// 保护原理：输入超过3.3V+0.3V时上二极管导通→钳位到3.3V+0.3V
//          输入低于-0.3V时下二极管导通→钳位到-0.3V
//          串联1K电阻限制钳位电流

示波器使用技巧

示波器快速上手调试工具
万用表只能看静态电压，示波器能看波形随时间的变化。对于串口/I2C/SPI通信故障、电源纹波、晶振是否起振等问题，示波器是必需的。
// === 示波器基本操作 ===
// 1. 垂直设置（Volts/Div）：每个格子的电压
//    看3.3V信号 → 设1V/div → 信号占3.3格
//
// 2. 水平设置（Time/Div）：每个格子的时间
//    看115200波特率串口（每bit≈8.68μs）→ 设10μs/div
//    看1KHz PWM → 设200μs/div
//
// 3. 触发（Trigger）：决定何时"冻结"波形
//    触发源：选CH1或CH2（看哪个信号）
//    触发模式：Auto(自动)/Normal(正常)/Single(单次捕获)
//    触发电平：设到信号幅度的50%（如1.65V for 3.3V信号）
//    边沿：上升沿/下降沿
//
// 4. 耦合（Coupling）：
//    DC耦合：显示信号的真实电压（含直流偏置）
//    AC耦合：滤除直流分量，只显示交流变化
//    测电源纹波 → AC耦合！否则DC 3.3V把纹波淹没了

// === 常用测量场景 ===
// 1. 测电源纹波
//    - CH1设为AC耦合，20mV/div
//    - 探头用×1档（不要用×10衰减）
//    - 探头尖端和地线夹紧贴VDD和GND（缩短地线环路！）
//    - 正常纹波：LDO<5mVpp, DC-DC<50mVpp
//
// 2. 测晶振起振
//    - 探头×10档（减小负载效应）
//    - 测OSC_OUT引脚
//    - 应看到正弦波，幅度约0.5-1.5Vpp
//
// 3. 测串口波形
//    - 用2个通道同时接TX和RX
//    - 触发设在上位机TX通道的下降沿（起始位）
//    - 手动解码：起始位(1bit低)+8bit数据(LSB)+停止位(1bit高)
//    - 或使用示波器的串口解码功能（如果有）
//
// 4. 测PWM波形
//    - 测量：频率、占空比、上升/下降时间
//    - 关注上升沿是否干净（振铃/过冲？）
💡 入门建议：¥200的DSO138迷你示波器只能看音频级信号。嵌入式开发至少需要50MHz带宽+1GSa/s采样率。推荐入门选择：Rigol DS1054Z（¥2000-3000）或Siglent SDS1104X-E（¥2000+）。如果资金紧张，先用逻辑分析仪（¥30的Saleae克隆）+万用表凑合。

// === PCB投板前检查清单 ===
//
// □ 1. DRC检查（Design Rule Check）
//    - 运行完整的DRC，确保0错误
//    - 重点：线间距、过孔大小、焊盘间距
//    - 嘉立创工艺能力：最小线宽/间距4mil(0.1mm), 最小过孔0.3mm/0.6mm
//
// □ 2. 丝印检查
//    - 元件标号是否被焊盘遮挡
//    - 正负极标记是否清晰（二极管/LED/电容极性）
//    - 文字方向一致（都朝上或都朝左）
//    - 1号引脚标记清晰（IC的丝印小圆点/凹槽）
//
// □ 3. 封装检查
//    - 每个元件封装与实物匹配吗？
//    - 特别检查：
//      QFP/LQFP引脚间距（0.5mm vs 0.8mm？）
//      SOP封装体的宽度（SOIC-8 vs SOP-8 Wide？）
//      排针排母间距（2.54mm标准，确认）
//      USB座/DC电源座方向（别焊反面！）
//
// □ 4. 供电检查
//    - VCC和GND是否短路（用EDA的连通性检查）
//    - 去耦电容是否每个VDD都有（100nF）
//    - 电源输入端的极性保护（防反接二极管）加了吗？
//    - 大电流走线宽度够吗？（≥0.5mm/1A）
//
// □ 5. 关键信号检查
//    - SWD接口：SWDIO/SWCLK/GND/VCC四根线都引出了吗？
//    - 串口：TX/RX是否交叉连接（TX→RX, RX→TX）

元器件采购避坑指南

🔍 元器件采购常见坑

1. 买到翻新/假货芯片
• 现象：芯片外观有打磨痕迹、引脚氧化发黑、印字模糊
• 重灾区：STM32F103C8T6、AMS1117、CH340、MAX485
• 对策：
- 尽量从立创商城/得捷/Mouser等正规渠道购买
- 淘宝买的话优先选企业店铺+高销量+好评率>99%
- 收到后用放大镜检查芯片丝印和引脚
- 买几片样品先测试，确认无误再批量采购

2. 钽电容焊反爆炸
• 钽电容正负极接反会短路+剧烈发热+冒烟+可能起火
• 钽电容上有标记的一端是正极（与铝电解电容相反！）
• 新手常见错误：按铝电解电容的惯性把标记端接GND
• 对策：
- 确认钽电容正极标记（通常是横杠/+号）
- 降额使用：5V供电至少选10V耐压的钽电容
- 能用陶瓷电容替代就用陶瓷电容（不会炸）

3. 晶振买错负载电容
• 晶振有12.5pF/20pF等不同负载电容版本
• 买了12.5pF的晶振但电路配了22pF电容→可能不起振
• 对策：购买时向卖家确认"负载电容多少"

4. 接插件/排针间距不对
• 最常见：2.54mm和2.0mm间距搞混
• 面包板用2.54mm，别买成2.0mm的
• 对策：买排针排母时用卡尺量一下确认间距

焊接常见缺陷识别与修复

// === 焊接缺陷图解说明 ===
//
// 1. 理想焊点：
//    呈凹面坡形（像富士山）
//    表面光亮平滑
//    焊锡充分润湿焊盘和引脚
//
// 2. 冷焊（Cold Joint）：最常见缺陷！
//    外观：表面粗糙、灰暗无光泽、呈颗粒状
//    原因：烙铁温度不够/加热时间太短/移动了元件
//    后果：接触不良，时通时断（最难排查的Bug）
//    修复：加助焊剂重新加热
//
// 3. 虚焊（Dry Joint）：
//    外观：焊锡只在引脚上成球，没润湿焊盘
//    原因：焊盘氧化/没加助焊剂/烙铁没同时接触焊盘和引脚
//    后果：完全不通
//    修复：加助焊剂，烙铁同时接触焊盘和引脚重新加热
//
// 4. 桥接/连锡（Bridge）：
//    外观：相邻两个引脚被焊锡连在一起
//    原因：焊锡太多/拖焊速度太慢/没加助焊剂
//    修复：加助焊剂+用吸锡带吸走多余焊锡
//
// 5. 焊锡不足：
//    外观：焊盘上的焊锡太少，引脚露出
//    原因：送锡不够
//    修复：补充焊锡
//
// 6. 过热/焊盘脱落：
//    外观：PCB上的铜焊盘翘起或脱落
//    原因：烙铁温度太高/加热太久
//    修复：飞线修补（难）或报废
//    预防：烙铁温度≤350°C，每个焊点≤3秒

// === 放大镜检查要点 ===
// 必备工具：¥15的珠宝放大镜（30X/60X）
// 逐引脚检查：
//   1. 是否有连锡（相邻引脚间）
//   2. 焊点是否光亮（灰暗=冷焊）
//   3. 焊锡是否润湿焊盘（只在引脚上成球=虚焊）
//   4. 是否有锡珠飞溅（可能造成短路）
🔍 常见问题：上电没反应
1. 万用表测VDD/GND之间电压 → 应为3.3V
2. 测晶振引脚 → 应有正弦波（示波器）或约1.6V DC（万用表）
3. 测RESET引脚 → 应为高电平(3.3V)，低电平=一直复位
4. 检查BOOT0/BOOT1引脚状态 → 都为低才从Flash启动
🔍 常见问题：板子短路冒烟
1. 立即断电！ 拔掉USB/电源适配器
2. 用万用表蜂鸣档测VDD和GND → 蜂鸣=短路
3. 嗅探大法：闻哪里焦糊味（LDO和钽电容是重灾区）
4. 手指感觉大法：断电后摸芯片，烫手的在短路（电流全流过它了）
5. 割线法：切断电源走线，分段排查短路区域
6. 极限法：用可调电源限制电流（如0.5A），短路处会发烫
7. 常见短路原因：焊盘连锡、钽电容焊反、芯片焊歪
🔍 常见问题：ST-Link/J-Link连接不上芯片
1. SWDIO/SWCLK接线正确？（SWDIO→SWDIO, SWCLK→SWCLK, GND→GND, 3.3V→VCC）
2. 杜邦线是否松动？（SWD频率高，接触不良就失败）
3. BOOT0是否为低电平？（高电平时从系统存储器启动，SWD可能无法连接）
4. NRST是否为高电平？（如果一直被拉低，芯片处于复位状态）
5. 芯片是否正常供电（3.3V）？
6. SWD引脚是否有其他电路冲突？（某些项目把SWDIO用作普通IO）
7. 连接时按住RESET再释放（"Connect Under Reset"）

本章总结

📋 知识点回顾：

设计流程：需求分析 → 芯片选型 → 原理图 → PCB Layout → 打样 → 焊接调试。先想清楚再画图。

电源设计（新增重点）：LDO简单安静但效率低（Vin-Vout全变热），DC-DC高效但有纹波。AMS1117从12V降3.3V→100mA就烫手。大电流/高压差→必须DC-DC。

原理图阅读：按电源→MCU→信号流向→网络标号的顺序看。注意VCC/GND/NC/RESET/BOOT等关键符号。

数据手册：先看Pinout和Typical Application。关键参数：VDD范围、IO电流、工作温度、封装。Absolute Maximum Ratings是"超过必烧"的底线。

PCB Layout核心：去耦电容(100nF紧贴VDD引脚)、线宽与电流匹配、完整地平面、电源星形拓扑。2层板够入门，4层板信号质量好得多。

去耦电容：每个VDD引脚一颗100nF X7R电容，布局离引脚≤5mm。它是数字电路稳定工作的"定海神针"。

晶振电路（新增）：负载电容C=(CL-C_stray)×2，这是晶振不起振的最常见原因。晶振远离MCU >10mm=可能不起振。面包板上别跑高频晶振。

电平转换（新增）：3.3V↔5V：I2C用MOSFET(BSS138)，多通道用TXS0108，单向用74HCT245。关键是查VIH——3.3V输出能否被5V设备识别为"高"。

ESD保护（新增）：暴露接口必须有TVS管保护。USB D+/D-用PESD5V0L1BA。顺序："先保护后功能"。

运放应用（新增）：电压跟随器→缓冲传感器进ADC；同相放大器→放大微弱信号；差分放大器→电流检测。LM358是最便宜的入门运放。

EMC设计：去耦+地平面+信号端接+模拟数字地隔离。电机干扰要光耦隔离。

检测工具（扩展）：万用表(电压/通断/电阻)+逻辑分析仪(¥30, 协议解码神器)+示波器(50MHz+)。逻辑分析仪看协议数据，示波器看信号质量。

投板检查（新增）：DRC零错误+封装匹配+供电检查+SWD引出+Gerber逐层预览。10分钟检查避免5天等待。

焊接（扩展）：拖焊法焊QFP，热风枪焊QFN。识别冷焊(灰暗)/虚焊(不润湿)/连锡(桥接)。助焊剂是新手最好的朋友。

采购避坑（新增）：假货重灾区→STM32/AMS1117/CH340。钽电容标记端是正极（别接GND！）。晶振确认负载电容再买。

排错顺序：电源 → 晶振 → 复位 → BOOT → IO口。按这个顺序检查，80%的硬件问题都能找到。
