第9章：通信协议详解

📖 小白完全指南 · 9/14 | 阅读时间约3分钟

单片机要与传感器、显示屏、存储器等外设通信，离不开通信协议。最常用的三种：UART、I2C、SPI，各有各的场景。

💡 类比：
• UART=对讲机（两点之间，全双工，各说各的）
• I2C=课堂点名（一条总线，老师(Master)点名，学生(Slave)应答，7位学号=地址）
• SPI=专线电话（四根线，一对一高速通话，片选=拨号选人）

三种协议对比

特性UARTI2CSPI
线数2 (TX/RX)2 (SDA/SCL)4 (MOSI/MISO/SCK/CS)
拓扑点对点多从机（地址寻址）多从机（片选线）
速度≤1Mbps≤3.4Mbps≤80Mbps
双工全双工半双工全双工
距离远（15m+）近（1m）极近（0.3m）
典型用途串口调试/GPS传感器/EEPROMFlash/显示屏/ADC

I2C协议详解

// I2C读写示例（HAL库）
// 写：主机→从机
HAL_I2C_Mem_Write(&hi2c1, 0x68<<1, 0x75, 1, &data, 1, 1000);
//           I2C句柄  从机地址(左移1位) 寄存器地址  长度  数据  长度  超时

// 读：从机→主机
uint8_t reg = 0x75;
HAL_I2C_Mem_Read(&hi2c1, 0x68<<1, reg, 1, &who_am_i, 1, 1000);

// I2C地址扫描（扫描0x03-0x77）
void i2c_scan(void) {
    printf("I2C Scan:\n");
    for (uint8_t addr = 0x03; addr < 0x78; addr++) {
        if (HAL_I2C_IsDeviceReady(&hi2c1, addr<<1, 3, 10) == HAL_OK) {
            printf("  Found device at 0x%02X
", addr);
        }
    }
}

CAN总线简介

CAN(Controller Area Network)是汽车和工业控制的标准总线。差分信号抗干扰，最高1Mbps，支持多节点，有优先级仲裁。

⚠️ 前提：STM32的CAN外设需要配置波特率、过滤器、中断。以下代码基于STM32F103 + HAL库，需先用CubeMX启用CAN外设。
// STM32 HAL CAN初始化（CubeMX生成后补充）
CAN_HandleTypeDef hcan;

void MX_CAN_Init(void) {
    hcan.Instance = CAN1;
    hcan.Init.Prescaler = 4;           // 72MHz/4=18MHz
    hcan.Init.Mode = CAN_MODE_NORMAL;
    hcan.Init.SyncJumpWidth = CAN_SJW_1TQ;
    hcan.Init.TimeSeg1 = CAN_BS1_13TQ; // 时间段1
    hcan.Init.TimeSeg2 = CAN_BS2_2TQ;  // 时间段2
    // 波特率 = 18MHz / (1+13+2) = 1MHz... 实际500Kbps常用
    hcan.Init.TimeTriggeredMode = DISABLE;
    hcan.Init.AutoBusOff = ENABLE;
    hcan.Init.AutoWakeUp = DISABLE;
    hcan.Init.AutoRetransmission = ENABLE;
    HAL_CAN_Init(&hcan);
}

// 配置过滤器（必须！否则收不到数据）
CAN_FilterTypeDef filter;
filter.FilterBank = 0;
filter.FilterMode = CAN_FILTERMODE_IDMASK;
filter.FilterScale = CAN_FILTERSCALE_32BIT;
filter.FilterIdHigh = 0x0000;
filter.FilterIdLow = 0x0000;
filter.FilterMaskIdHigh = 0x0000;
filter.FilterMaskIdLow = 0x0000;
filter.FilterFIFOAssignment = CAN_RX_FIFO0;
filter.FilterActivation = ENABLE;
HAL_CAN_ConfigFilter(&hcan, &filter);
HAL_CAN_Start(&hcan);
HAL_CAN_ActivateNotification(&hcan, CAN_IT_RX_FIFO0_MSG_PENDING);
// STM32 HAL CAN发送
CAN_TxHeaderTypeDef tx_header;
tx_header.StdId = 0x123;
tx_header.IDE   = CAN_ID_STD;
tx_header.RTR   = CAN_RTR_DATA;
tx_header.DLC   = 8;

uint8_t tx_data[8] = {0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07, 0x08};
uint32_t tx_mailbox;
HAL_CAN_AddTxMessage(&hcan, &tx_header, tx_data, &tx_mailbox);
// CAN接收中断回调
void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan) {
    CAN_RxHeaderTypeDef rx_header;
    uint8_t rx_data[8];
    if (HAL_CAN_GetRxMessage(hcan, CAN_RX_FIFO0, &rx_header, rx_data) == HAL_OK) {
        printf("CAN ID:0x%lx DLC:%lu Data:", rx_header.StdId, rx_header.DLC);
        for (int i = 0; i < rx_header.DLC; i++) {
            printf("%02X ", rx_data[i]);
        }
        printf("\n");
    }
}

逻辑分析仪

逻辑分析仪是调试通信协议的必备神器——它同时抓取多路数字信号，用软件解码I2C/SPI/UART等协议的数据。

逻辑分析仪协议调试¥20-50
8通道/24MHz采样率就够用。配合 sigrok/PulseView 开源软件，能自动解码I2C/SPI/UART/CAN等协议。
# PulseView（开源逻辑分析仪软件）
# 1. 下载：sigrok.org
# 2. 连接逻辑分析仪到USB
# 3. 选择通道和采样率（推荐8MHz以上）
# 4. 添加协议解码器（I2C/SPI/UART等）
# 5. 捕获→查看解码数据
🔍 常见问题：I2C设备无响应
1. 上拉电阻！I2C的SDA/SCL必须各接4.7K上拉电阻到VCC
2. 地址是否正确？很多传感器有AD0引脚，接地/接VCC地址不同
3. 用 i2c_scan() 扫描确认设备是否存在
4. 逻辑分析仪抓波形，看是否有ACK
