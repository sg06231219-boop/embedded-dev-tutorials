第3章：Makefile实战

📖 小白完全指南 · 3/14 | 阅读时间约3分钟

Makefile是嵌入式项目最经典的构建方式——虽然CMake更现代，但大量开源项目仍在用Makefile，必须会读会写。

💡 类比：Makefile就像菜谱——target=菜名（"红烧肉"），prerequisites=食材（五花肉、酱油），recipe=做法（切肉→焯水→炖煮）。make就是你雇的厨师，按菜谱一步步做。

Makefile基本规则

# 格式（recipe必须用TAB缩进！）
target: prerequisites
	recipe

# 例子
main.o: main.c main.h
	arm-none-eabi-gcc -c -mcpu=cortex-m3 -mthumb main.c -o main.o

变量与自动变量

CC = arm-none-eabi-gcc
CFLAGS = -mcpu=cortex-m3 -mthumb -Wall -O2 -g
TARGET = myproject

# 自动变量
# $@ = 目标名    $< = 第一个依赖    $^ = 所有依赖
main.o: main.c main.h
	$(CC) $(CFLAGS) -c $< -o $@

模式规则

# 通用规则：所有.c → .o
%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

# 链接：所有.o → .elf
$(TARGET).elf: $(OBJS)
	$(CC) $(LDFLAGS) $^ -o $@

完整STM32 Makefile模板

# ====== STM32F103 Makefile ======
CC = arm-none-eabi-gcc
OBJCOPY = arm-none-eabi-objcopy
SIZE = arm-none-eabi-size
TARGET = blink
MCU = cortex-m3

C_SOURCES = $(wildcard src/*.c)
INCLUDES = -Iinc -Idrivers/CMSIS

CFLAGS  = -mcpu=$(MCU) -mthumb -Wall -O2 -g3
CFLAGS += -ffunction-sections -fdata-sections $(INCLUDES)
CFLAGS += -DSTM32F103C8T6

LDFLAGS  = -mcpu=$(MCU) -mthumb -Wl,--gc-sections
LDFLAGS += -TSTM32F103C8Tx.ld -Wl,-Map=$(TARGET).map
LDFLAGS += --specs=nano.specs --specs=nosys.specs

OBJS = $(C_SOURCES:.c=.o)

.PHONY: all clean flash

all: $(TARGET).bin $(TARGET).hex
	@echo "Build complete!"
	$(SIZE) $(TARGET).elf

$(TARGET).elf: $(OBJS)
	$(CC) $(LDFLAGS) $^ -o $@

$(TARGET).bin: $(TARGET).elf
	$(OBJCOPY) -O binary $< $@

$(TARGET).hex: $(TARGET).elf
	$(OBJCOPY) -O ihex $< $@

%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

clean:
	powershell -Command "Remove-Item src/*.o $(TARGET).elf $(TARGET).bin $(TARGET).hex $(TARGET).map -ErrorAction SilentlyContinue"
# 注：嵌入式Makefile通常在Windows PowerShell环境运行
# 如果你在Linux/WSL下，改用：
# 	rm -f $(OBJS) $(TARGET).elf $(TARGET).bin $(TARGET).hex $(TARGET).map

flash: $(TARGET).bin
	st-flash write $(TARGET).bin 0x08000000
⚠️ 重要：Makefile中recipe行必须用TAB缩进，不能用空格！这是最常见的坑。复制粘贴后如果报"missing separator"，把空格替换成TAB。
🔍 常见问题：make报"missing separator"
recipe行用了空格而非TAB。解决：VS Code中打开Makefile，确保右下角显示"Tabs"而非"Spaces"。
💡 小技巧：
• make -n 干跑（只打印不执行）
• make -j4 4线程并行编译
• make -B 强制重新编译全部
• make V=1 显示完整编译命令
📝 完整示例：用Makefile构建STM32项目
项目目录：
stm32-blink/
├── Makefile
├── STM32F103C8Tx.ld
├── inc/
│   └── main.h
└── src/
    ├── main.c
    ├── stm32f1xx_it.c
    └── system_stm32f1xx.c
运行命令：
cd stm32-blink
make clean && make -j4
输出结果：
arm-none-eabi-gcc -c -mcpu=cortex-m3 -mthumb -Wall -O2 -g3 ... -o src/main.o
arm-none-eabi-gcc -c -mcpu=cortex-m3 -mthumb -Wall -O2 -g3 ... -o src/stm32f1xx_it.o
arm-none-eabi-gcc -c -mcpu=cortex-m3 -mthumb -Wall -O2 -g3 ... -o src/system_stm32f1xx.o
arm-none-eabi-gcc ... -o blink.elf
arm-none-eabi-objcopy -O binary blink.elf blink.bin
arm-none-eabi-objcopy -O ihex blink.elf blink.hex

   1580      20     256    1856     740 blink.elf

Build complete!
