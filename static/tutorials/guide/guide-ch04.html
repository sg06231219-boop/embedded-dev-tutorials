第4章：串口通信 — 单片机与电脑对话

📖 小白完全指南 · 4/14 | 阅读时间约3分钟

串口(UART)是嵌入式开发最基础、最常用的通信方式。单片机的printf输出、调试信息、AT指令控制，全靠串口。可以说：不会串口，嵌入式寸步难行。

💡 类比：串口就像两个人用对讲机通话——波特率=说话速度（必须一致才能听懂），数据位=每个词的长度，停止位=说完了喘口气，校验位=确认对方听清了。速度不一样就全是乱码，就像一个人说中文、一个说英文。

串口参数

参数常见值说明
波特率(Baud Rate)9600 / 115200通信速度，双方必须一致
数据位(Data Bits)8每个字节的有效位数
停止位(Stop Bits)1帧结束标志
校验位(Parity)None简单错误检测，通常不用
流控(Flow Control)None硬件/软件流控，通常不用

日常最常用的配置：115200 8N1（115200波特率，8数据位，无校验，1停止位）

Python串口编程脚本化串口通信
import serial
import time

# 打开串口
ser = serial.Serial('COM3', 115200, timeout=1)

# 发送数据
ser.write(b'Hello STM32!\n')

# 接收数据（按行读取）
line = ser.readline().decode('utf-8').strip()
print(f"Received: {line}")

# 接收指定字节数
data = ser.read(10)  # 读10字节
print(f"Hex: {data.hex()}")

# 关闭
ser.close()

PuTTY串口配置

步骤1：打开PuTTY → 选择"Serial"连接类型
步骤2：Serial line填 COM3，Speed填 115200
步骤3：左侧菜单 → Connection → Serial → 确认 8 data bits, 1 stop bits, None parity
步骤4：点Open连接
💡 小技巧：PuTTY的串口窗口看不到输入的字符？正常——STM32通常只发不收。如果想看到本地回显：Terminal → Features → 勾选"Local echo"。

🔍 常见问题：设备管理器看不到COM口
1. 检查USB线是否插好（有的线只能充电不能传数据！）
2. 检查驱动是否安装（设备管理器里黄色感叹号=驱动未装）
3. CH340在Win10/11有时不需要额外驱动，直接识别
4. 查看COM口号：python -m serial.tools.list_ports（如果python命令不存在，试试python3.14）

安装并检测串口：
pip install pyserial
python -m serial.tools.list_ports
# 如果python命令不存在，试试：python3.14 -m serial.tools.list_ports

打开串口终端（STM32已运行，每秒发送温度数据）：

# 如果python命令不存在：python3.14 -m serial.tools.miniterm COM3 115200
串口输出：

--- Quit: Ctrl+] | Menu: Ctrl+T ---
[1234] Temperature: 25.3 C
[2234] Temperature: 25.5 C
[3234] Temperature: 25.4 C
^C--- exit ---
