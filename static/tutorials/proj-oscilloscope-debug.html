📈 教程20: 示波器调试实战

📋 项目概述

示波器是嵌入式开发者的"第三只眼"。本教程将教你用示波器测量GPIO翻转、I2C/SPI/UART时序、PWM信号，并利用示波器的触发功能捕捉偶发BUG，让你的调试效率提升10倍。

示波器 vs 逻辑分析仪：
- 示波器：测量模拟信号(波形形状、幅度、噪声)、高频信号、协议模拟部分(UART/SPI/I2C)
- 逻辑分析仪：捕获数字信号、解码协议数据(更擅长协议层分析)
两者配合使用最佳！

⚠️ 示波器安全使用：
1. 探头衰减比要设置正确(1×=不衰减，10×=衰减10倍)
2. 测量高压信号时，确认示波器探头耐压范围
3. 共地：示波器所有通道的地是相通的，不要用示波器测浮地设备
4. 接地夹要尽量短，减少环路面积降低噪声

触发模式详解

模式说明适用场景
Auto有信号自动触发，无信号自动扫描快速观察，不知道信号何时出现
Normal有信号才触发，无信号保持显示偶发信号、噪声环境
Single满足触发条件一次后停止一次性事件(掉电、异常)
Roll滚动模式，右向左滚动显示超低频信号、长时间监测
触发耦合:
- DC: 显示直流+交流成分 (最常用)
- AC: 阻断直流，只显示交流 (测AC信号纹波)
- HF: 抑制高频 (滤除噪声尖峰)
- LF: 抑制低频 (滤除工频干扰)

触发类型:
- Edge: 边沿触发(上升/下降沿)
- Pulse: 脉宽触发(大于/小于/等于某脉宽)
- Nth Edge: N次边沿触发
- I2C/SPI/UART: 协议触发(高端示波器)

探头衰减比(1× vs 10×)

┌─────────┬────────────────┬─────────────────────────┐
│  档位   │  特点           │  使用场景                │
├─────────┼────────────────┼─────────────────────────┤
│  1×    │  不衰减          │  低频、低压信号          │
│         │  带宽受限(~10MHz)│  观察信号整体形状        │
│         │  负载大(1MΩ)     │                         │
├─────────┼────────────────┼─────────────────────────┤
│  10×   │  衰减10倍        │  高频信号(>10MHz)       │
│         │  带宽大(~100MHz) │  高压信号               │
│         │  负载小(10MΩ)    │  精确测量               │
└─────────┴────────────────┴─────────────────────────┘

⚠️ 必须在示波器上设置正确的探头衰减比！
   否则读数和触发电平都会错误

GPIO翻转测试代码

/* GPIO翻转测试 - 用于测量GPIO响应速度 */
#include "main.h"

void Test_GPIO_Toggle(void)
{
    /* PB0输出，50%占空比翻转 */
    HAL_GPIO_TogglePin(GPIOB, GPIO_PIN_0);

    /* 用示波器测量PB0引脚
     * 设置时基: 1us/div
     * 观察: 上升/下降时间、周期
     *
     * STM32F103 GPIO翻转频率理论:
     * - 72MHz / 4 cycles ≈ 18MHz (汇编优化)
     * - 72MHz / 12 cycles ≈ 6MHz (HAL库调用)
     */
}

/* 中断延迟测量 */
volatile uint8_t g_int_triggered = 0;

void EXTI0_IRQHandler(void)
{
    /* 在这里拉高一个GPIO(PB1)作为测量点 */
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_1, GPIO_PIN_SET);
    /* 中断处理代码 */
    g_int_triggered = 1;
    HAL_GPIO_EXTI_IRQHandler(GPIO_PIN_0);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_1, GPIO_PIN_RESET);
}

/* 测试方法:
 * CH1: PA0(外部中断触发信号)
 * CH2: PB1(中断处理标志)
 * 测量: 从CH1下降沿到CH2上升沿的延迟
 *
 * 中断延迟 = 中断响应时间 + 用户代码执行时间
 * Cortex-M3/M4 中断响应 ≈ 12个时钟周期 ≈ 0.17us @ 72MHz
 */

I2C时序测试代码

/* I2C时序测试 - 用于测量I2C波形 */
#include "main.h"

I2C_HandleTypeDef hi2c1;

void Test_I2C_Scanner(void)
{
    /* 扫描I2C总线上的设备地址 */
    uint8_t addr;
    HAL_StatusTypeDef status;

    for (addr = 1; addr < 128; addr++)
    {
        status = HAL_I2C_IsDeviceReady(&hi2c1, (uint16_t)(addr << 1), 3, 100);

        if (status == HAL_OK)
        {
            printf("Found device at 0x%02X\r\n", addr);
        }
    }
}

/* I2C时序测量要点:
 *
 * ┌────────────────┐    ┌────────────────────────┐
 * │ START condition │    │ SCL (时钟)              │
 * │ S ──────────────┘    │  ___ ___ ___ ___ ___    │
 * │                     │    ┌───┐   ┌───┐        │
 * │ SDA (数据)           │ ───┘   └───┘   └────    │
 * │ ═══╗    ╔═══════════│ ↑    ↑    ↑    ↑        │
 * │    ║    ║ SCL低电平 │ SDA在SCL高时变化         │
 * │    ╚════╝           │ (START/STOP条件)         │
 * └────────────────┘                            │
 *                                                │
 * I2C标准速率:                                   │
 * - Standard: 100KHz, t_LOW=4.7us, t_HIGH=4.7us│
 * - Fast: 400KHz, t_LOW=1.3us, t_HIGH=0.6us    │
 * - Fast Mode+: 1MHz (需要IO开漏+上拉匹配)      │
 */

SPI时序测试代码

/* SPI时序测试 */
#include "main.h"

SPI_HandleTypeDef hspi1;

void Test_SPI_Transfer(void)
{
    uint8_t tx_data[8] = {0x55, 0xAA, 0x0F, 0xF0, 0x33, 0xCC, 0x12, 0x21};
    uint8_t rx_data[8];

    /* SPI时序测量:
     * ┌───────────────────────────────────────┐
     * │ SCK (时钟)                             │
     * │  ────┐   ────┐   ────┐   ────┐       │
     * │       └───┘   └───┘   └───┘   └─     │
     * │                                        │
     * │ MOSI: ──── 1 ──── 0 ──── 1 ────      │
     * │ ──────────────────────────────        │
     * │                                        │
     * │ CPOL=0, CPHA=0 时序:                  │
     * │  - SCLK空闲=0, 第一个边沿采样          │
     * │                                        │
     * │ CPOL=1, CPHA=1 时序:                  │
     * │  - SCLK空闲=1, 第二个边沿采样          │
     * └───────────────────────────────────────┘
     */

    HAL_SPI_TransmitReceive(&hspi1, tx_data, rx_data, 8, 1000);
}

/* SPI时序参数说明:
 * CPOL: 时钟极性
 *   CPOL=0: 空闲SCLK=0, 第一个边沿上升
 *   CPOL=1: 空闲SCLK=1, 第一个边沿下降
 *
 * CPHA: 时钟相位
 *   CPHA=0: 第一个边沿采样
 *   CPHA=1: 第二个边沿采样
 *
 * 组合:
 *   Mode 0: CPOL=0, CPHA=0 ← 最常用
 *   Mode 1: CPOL=0, CPHA=1
 *   Mode 2: CPOL=1, CPHA=0
 *   Mode 3: CPOL=1, CPHA=1
 */

UART波形测试代码

/* UART时序测试 */
#include "main.h"

UART_HandleTypeDef huart1;

void Test_UART_Send(void)
{
    uint8_t data = 0x55; /* 二进制: 01010101 */

    /* UART时序 (9600bps, 8N1):
     * ┌────────────────────────────────────────┐
     * │                                          │
     * │  起始位(0)  D0  D1  D2  D3  D4  D5  D6  D7  停止位(1)
     * │  ──────────╱───┘───┘───┘───┘───┘───┘───┘───┘───────
     * │           ────────────────────────────────────────────
     * │                                               ↑
     * │                                           空闲=1
     * │
     * │ 1位时间 = 1/9600 ≈ 104.17us
     * │ 1帧 = 10位 = 1.042ms
     * │ 波特率误差应 < 2%
     * └────────────────────────────────────────┘
     */

    HAL_UART_Transmit(&huart1, &data, 1, 1000);
}

/* 常用波特率位时间对照:
 *  9600bps: 104.17us
 *  19200bps: 52.08us
 *  38400bps: 26.04us
 *  57600bps: 17.36us
 * 115200bps: 8.68us
 *
 * 示波器设置:
 * - 时基: 根据波特率选择
 *   9600 → 500us/div (能看到1位)
 *   115200 → 10us/div (能看到1位)
 * - 触发: 下降沿触发(从空闲到起始位0)
 */

PWM测量测试

/* PWM信号测试 */
#include "main.h"

TIM_HandleTypeDef htim3;

void Test_PWM_Measure(void)
{
    /* 配置TIM3 CH2 (PA7) 输出PWM */
    /* 频率 = 72MHz / (PSC+1) / (ARR+1)
     * 例: PSC=7199, ARR=999 → 72M/7200/1000 = 10Hz
     *      PSC=71, ARR=999 → 72M/72/1000 = 1000Hz */

    HAL_TIM_PWM_Start(&htim3, TIM_CHANNEL_2);

    /* PWM参数:
     * ┌────────────────────────────────────────┐
     * │                                          │
     * │ ══════════════╗          ╔════════════  │
     * │               ║          ║              │
     * │               ╚══════════╝              │
     * │                                          │
     * │ ←──── T ──────→│                        │
     * │   ┌─── Duty ───┐│                       │
     * │                                          │
     * │  占空比 = D/T × 100%                    │
     * │  周期的倒数 = 频率                      │
     * └────────────────────────────────────────┘
     */

    /* 改变占空比 */
    __HAL_TIM_SET_COMPARE(&htim3, TIM_CHANNEL_2, 500); /* 50% */
}

/* PWM测量方法:
 * 示波器测量:
 * 1. 频率: 使用示波器频率计或光标测量周期
 * 2. 占空比: 示波器的"测量"功能 → 占空比
 * 3. 幅值: 高电平 - 低电平
 *
 * 示波器Cursor(光标)测量:
 * 手动移动光标A/B到信号边沿，示波器计算:
 * - ΔT: 时间差
 * - ΔV: 电压差
 */

📝 分步操作指引

第1步：示波器基础设置
1. 开机后按[Auto]键，让示波器自动识别信号
2. 确认探头衰减比设置：探头菜单 → 10×，示波器也要对应设置CH1 → 10×
3. 接地夹夹到开发板GND，探头尖接触测试点
4. 调节时基(Time/Div)和档位(Volt/Div)，让波形清晰
第2步：GPIO翻转测量
1. 烧录GPIO翻转测试代码
2. 示波器探头接PB0，地线夹到板子GND
3. 设置时基: 1μs/div
4. 观察周期，计算翻转频率
5. 对比HAL库vs直接操作寄存器的速度差异
第3步：I2C/SPI时序分析
1. CH1接SCL，CH2接SDA
2. 触发方式: 边沿触发，电平设在SCL中点
3. 时基: 根据速率选择
I2C 100KHz → 50μs/div
SPI 1MHz → 1μs/div
4. 观察: 启动条件、地址发送、数据ACK、停止条件
第4步：UART波形调试
1. 示波器接TX引脚，地线接GND
2. 设置触发: 下降沿(因为UART空闲=1，起始位=0)
3. 时基: 根据波特率选择(9600→200μs/div, 115200→20μs/div)
4. 观察: 起始位、数据位、停止位是否正确
5. 用光标测量位时间，验证波特率精度
第5步：中断延迟测量
1. CH1接外部中断源(按键/PWM)
2. CH2接GPIO标志(中断入口处拉高)
3. 触发: 单次触发，边沿触发
4. 测量CH1边沿到CH2边沿的时间差
5. 这个延迟就是中断响应时间+用户代码执行时间
第6步：捕捉偶发BUG
1. 设置触发为单次模式
2. 触发条件设为异常信号特征
- I2C总线卡死 → SCL低电平脉宽触发(>5ms)
- 复位异常 → 下降沿窄脉冲触发
- 看门狗复位 → 复位引脚下降沿
3. 降低时基，确保捕捉到事件前后的波形
4. 反复触发直到捕获到异常

❓ 常见问题排查

问题1：探头衰减比设置错误

⚠️ 这是最常见的测量错误！
如果探头在10×档，但示波器设置在1×档：
- 示波器显示的电压值是实际值的1/10
- 触发电平也会错误
- 测量结果完全不对

解决：每次换探头或换档位，都要确认示波器的CH设置与探头档位一致。

问题2：触发设置问题

现象原因解决
波形不稳定，左右滑动触发不稳定，触发电平不在信号范围内调节触发电平旋钮，使其在波形中间
一直等待TriggerNormal模式但没有满足触发条件的信号按Auto或检查探头连接
只有一条水平线触发电平太高/太低，或耦合设置错误检查GND耦合，或重新Auto
偶发信号看不到用的是Auto模式而不是Normal改用Normal或Single模式

问题3：带宽选择

示波器带宽选择原则：
示波器带宽 ≥ 信号最高频率成分 × 3~5

常见场景带宽需求：
- GPIO翻转(1MHz): 20MHz示波器足够
- UART(115KHz): 1MHz示波器即可
- I2C(400KHz): 10MHz示波器
- SPI(10MHz): 50MHz示波器
- USB全速(12MHz): 100MHz以上示波器
- DDR内存(100MHz+): 200MHz+示波器

问题4：接地问题

接地环路导致的噪声:
┌─────────────────────────────────┐
│                                 │
│  接地夹线太长 ──────────────────│ ← 形成大环路
│  ┌───────────────────────────┐ │    收集空间噪声
│  │                           │ │
│  │   探头尖              │ │
│  │                           │ │
│  └───────────────────────────┘ │
│                                 │
└─────────────────────────────────┘

正确做法:
1. 接地夹线尽量短
2. 使用探头自带的接地弹簧(附件)
3. 如果信号质量差，用"保护地"(Guard)而非信号地
4. 对于高频信号，考虑差分探头

问题5：带宽限制

示波器带宽限制功能：
大多数示波器有20MHz带宽限制，用来滤除高频噪声。

使用场景：
- 当信号包含高频毛刺但我们只关心低频成分时
- 测量UART/低频PWM时，开启20MHz限制可以减少噪声
- 测高速信号时记得关闭带宽限制

操作：按下示波器的[BW Limit]键或通道菜单。
💡 示波器高级功能：
1. FFT频谱分析：分析信号频率成分，查找干扰源
2. 协议解码：高端示波器内置I2C/SPI/UART解码，直接显示数值
3. 模板触发：设定合格波形模板，超出则触发(生产测试)
4. 分段存储：将内存分成多段，记录多次事件
5. 外部触发输入：用外部信号作为触发源，同步测量
6. XY模式：用于李萨如图形，观察两个信号的相位关系

❓ 常见问题 FAQ

Q: 没有示波器怎么办？

A: 1)逻辑分析仪(¥60)可看数字信号 2)Saleae软件很强大 3)STM32的GPIO翻转+逻辑分析仪替代 4)SoundCard示波器(用声卡)看低频信号

Q: 如何测量中断延迟？

A: 在中断入口翻转GPIO(拉高)，处理完再拉低。示波器测量GPIO高电平宽度=中断处理时间。触发信号用外部中断源。
