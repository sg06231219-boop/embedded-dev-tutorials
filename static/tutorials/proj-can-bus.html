教程12: CAN总线通信

📋 项目概述

本项目使用两块 STM32F103C8T6 通过 TJA1050 CAN收发器芯片建立 CAN总线通信。深入讲解 CAN协议原理（帧格式、仲裁机制、错误处理），完成 HAL_CAN初始化、过滤器配置、双向发送接收，实现可靠的多节点工业级通信。

📝 接线文字描述：
两块STM32F103C8T6板和两个TJA1050模块各自独立供电，但所有GND必须连接在一起（CAN总线的共地非常重要！）。TJA1050模块放在总线两端，每个模块的VCC接3.3V，GND接公共地线。节点A的PB9连TJA1050-A的TXD，PB8连RXD；节点B同理。CANH和CANL通过双绞线连接：CANH→CANH，CANL→CANL。两端各接一个120Ω终端电阻（短接在TJA1050的CANH/CANL引脚之间，或使用插针式终端电阻模块）。两根双绞线建议使用不同颜色区分，避免接反。

📚 CAN协议原理

CAN帧格式

字段长度说明
SOF1bitStart of Frame，显性0
Arbitration Field12/32bit11位ID + RTR/IDE
Control Field6bitDLC(4bit) + 保留位
Data Field0~64bit实际数据，DLC指定长度
CRC Field15+2bit校验码
ACK Field2bit确认槽
EOF7bit帧结束

标准帧 vs 扩展帧

  • 标准帧：11位ID，用于普通优先级通信
  • 扩展帧：29位ID，用于需要长ID的复杂系统

仲裁机制（多主总线）

当多个节点同时发送时，通过ID的逐位比较决定优先级。ID越小优先级越高（显性0优先）。低优先级节点在仲裁失败后自动转为接收模式，等待总线空闲后重发。

错误处理

错误类型说明
Bit Stuffing连续5个相同电平后必须插入反向位，违规则报错
CRC错误接收方计算CRC与发送方不匹配
ACK错误发送方未收到ACK响应
Form错误固定格式字段出现非法值
Bit Error发送时监控总线电平与自己不符

波特率计算

CAN波特率 = APB1时钟 / (BS1 + BS2) / Prescaler

示例: APB1=36MHz, 波特率=500kbps
Prescaler = 9-1 = 8
BS1 = 6-1 = 5  (Tq)
BS2 = 8-1 = 7  (Tq)
Time Quantum (Tq) = 36MHz / 8 / (5+7+1) = 500kHz
实际波特率 = 36MHz / 8 / 12 = 375kHz (需微调)
→ 调整为: Prescaler=4, BS1=9, BS2=8
实际波特率 = 36MHz / 4 / 18 = 500kHz ✓

Time quantum (Tq) 分配:
[ SYNC ][  BS1  ][  BS2  ]
1Tq      6-16Tq  5-8Tq

🛠️ 分步骤操作指引

步骤1：硬件检查
1. 确认TJA1050模块为3.3V兼容版本（不能用5V版本）
2. 检查终端电阻是否为120Ω
3. 确认双绞线已正确连接CANH/CANL
4. 用万用表测量总线两端电阻应为60Ω（两个120Ω并联）
步骤2：CubeMX配置（两块板相同）
1. SYS → Debug: Serial Wire
2. RCC → HSE: Crystal
3. Clock: APB1=36MHz（CAN挂载在APB1）
4. CAN1 → Mode: bxCAN (Beta CAN)
5. Parameter Settings:
  Prescaler: 9（36MHz/9=4MHz）
  Time Quantum: 4MHz
  BS1: 6 Tq
  BS2: 7 Tq
  SAM: Single Sample（一般选这个）
  NVIC → CAN1 TX/RX interrupts: Enabled
6. PB8 → CAN_RX, PB9 → CAN_TX
7. USART1 → Asynchronous, 115200（调试用）
步骤3：配置过滤器
1. 过滤器决定接收哪些CAN帧
2. 每块板设置不同ID过滤或广播过滤
3. Filter Bank 0: 接收所有帧（ID=0x000）
4. 或 Filter Bank 0: 只接收指定ID范围
步骤4：节点A发送，节点B接收测试
1. 节点A烧录发送代码
2. 节点B烧录接收代码
3. 打开两个串口助手，观察数据
4. 成功后交换测试
步骤5：双向通信测试
1. 两块板都烧录完整代码
2. 测试双向收发
3. 测试多帧数据
4. 添加超时和错误处理

💻 完整代码

can_node_a.c — 发送节点

#include "main.h"
#include "can.h"
#include "usart.h"
#include <stdio.h>

CAN_TxHeaderTypeDef tx_header;
CAN_RxHeaderTypeDef rx_header;
uint8_t tx_data[8];
uint8_t rx_data[8];
uint32_t tx_mailbox;

/* printf重定向 */
int fputc(int ch, FILE *f)
{
    HAL_UART_Transmit(&huart1, (uint8_t *)&ch, 1, HAL_MAX_DELAY);
    return ch;
}

void CAN_SendMessage(uint32_t id, uint8_t *data, uint8_t len)
{
    tx_header.StdId = id;
    tx_header.ExtId = 0;
    tx_header.IDE = CAN_ID_STD;
    tx_header.RTR = CAN_RTR_DATA;
    tx_header.DLC = len;

    if (HAL_CAN_AddTxMessage(&hcan1, &tx_header, data, &tx_mailbox) != HAL_OK) {
        printf("CAN TX Error!\r\n");
    } else {
        printf("TX[0x%03lX]: ", id);
        for (int i = 0; i < len; i++) printf("%02X ", data[i]);
        printf("\r\n");
    }
}

void CAN_FilterConfig(void)
{
    CAN_FilterTypeDef filter;

    /* 过滤器初始化：在HAL_CAN_Init之后调用 */
    filter.FilterBank = 0;
    filter.FilterMode = CAN_FILTERMODE_IDMASK;   // 掩码模式
    filter.FilterScale = CAN_FILTERSCALE_32BIT; // 32位尺度
    filter.FilterIdHigh = 0x0000;               // ID高位
    filter.FilterIdLow  = 0x0000;                // ID低位
    filter.FilterMaskIdHigh = 0x0000;            // 掩码高位(0=接收所有)
    filter.FilterMaskIdLow  = 0x0000;            // 掩码低位
    filter.FilterFIFOAssignment = CAN_FILTER_FIFO0;
    filter.FilterActivation = CAN_FILTER_ENABLE;

    if (HAL_CAN_ConfigFilter(&hcan1, &filter) != HAL_OK) {
        Error_Handler();
    }
}

void CAN_Start(void)
{
    /* 启动CAN */
    if (HAL_CAN_Start(&hcan1) != HAL_OK) {
        Error_Handler();
    }

    /* 激活通知 */
    if (HAL_CAN_ActivateNotification(&hcan1,
            CAN_IT_TX_MAILBOX_EMPTY | CAN_IT_RX_FIFO0_MSG_PENDING |
            CAN_IT_RX_FIFO1_MSG_PENDING) != HAL_OK) {
        Error_Handler();
    }
}

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_USART1_UART_Init();
    MX_CAN1_Init();

    printf("CAN Node A - Sender\r\n");

    CAN_FilterConfig();
    CAN_Start();

    /* 初始化发送头 */
    tx_header.StdId = 0x123;
    tx_header.IDE = CAN_ID_STD;
    tx_header.RTR = CAN_RTR_DATA;

    uint8_t counter = 0;

    while (1) {
        /* 构造8字节数据 */
        tx_data[0] = 0x01;              // 节点A标识
        tx_data[1] = counter;           // 计数器
        tx_data[2] = 0xAA;             // 测试数据
        tx_data[3] = 0x55;
        tx_data[4] = (HAL_GetTick() >> 8) & 0xFF;
        tx_data[5] = HAL_GetTick() & 0xFF;
        tx_data[6] = 0x00;
        tx_data[7] = 0x00;

        CAN_SendMessage(0x123, tx_data, 8);

        /* 每秒发送 */
        HAL_Delay(1000);
        counter++;
    }
}

/* 接收回调函数 */
void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan)
{
    if (HAL_CAN_GetRxMessage(hcan, CAN_RX_FIFO0, &rx_header, rx_data) == HAL_OK) {
        printf("RX[0x%03lX]: ", rx_header.StdId);
        for (int i = 0; i < rx_header.DLC; i++) {
            printf("%02X ", rx_data[i]);
        }
        printf("\r\n");
    }
}

void HAL_CAN_TxMailbox0CompleteCallback(CAN_HandleTypeDef *hcan)
{
    // 可在此记录发送成功
}

void HAL_CAN_ErrorCallback(CAN_HandleTypeDef *hcan)
{
    printf("CAN Error! ErrorCode: %lu\r\n", hcan->ErrorCode);
}

can_node_b.c — 接收节点

#include "main.h"
#include "can.h"
#include "usart.h"
#include <stdio.h>

CAN_RxHeaderTypeDef rx_header;
CAN_TxHeaderTypeDef tx_header;
uint8_t tx_data[8];
uint8_t rx_data[8];
uint32_t tx_mailbox;

int fputc(int ch, FILE *f)
{
    HAL_UART_Transmit(&huart1, (uint8_t *)&ch, 1, HAL_MAX_DELAY);
    return ch;
}

void CAN_Reply(uint32_t id, uint8_t *data, uint8_t len)
{
    tx_header.StdId = id + 0x100;  // 回复ID=原ID+0x100
    tx_header.IDE = CAN_ID_STD;
    tx_header.RTR = CAN_RTR_DATA;
    tx_header.DLC = len;

    if (HAL_CAN_AddTxMessage(&hcan1, &tx_header, data, &tx_mailbox) == HAL_OK) {
        printf("REPLY[0x%03lX]: ", tx_header.StdId);
        for (int i = 0; i < len; i++) printf("%02X ", data[i]);
        printf("\r\n");
    }
}

void CAN_FilterConfig(void)
{
    CAN_FilterTypeDef filter;

    /* 过滤器Bank 0: 接收所有标准帧 */
    filter.FilterBank = 0;
    filter.FilterMode = CAN_FILTERMODE_IDMASK;
    filter.FilterScale = CAN_FILTERSCALE_32BIT;
    filter.FilterIdHigh = 0x0000;
    filter.FilterIdLow  = 0x0000;
    filter.FilterMaskIdHigh = 0x0000;
    filter.FilterMaskIdLow  = 0x0000;
    filter.FilterFIFOAssignment = CAN_FILTER_FIFO0;
    filter.FilterActivation = CAN_FILTER_ENABLE;

    if (HAL_CAN_ConfigFilter(&hcan1, &filter) != HAL_OK) {
        Error_Handler();
    }

    /* 过滤器Bank 1: 接收0x200~0x20F（电机控制帧）*/
    filter.FilterBank = 1;
    filter.FilterIdHigh = 0x200 << 5;
    filter.FilterIdLow  = 0;
    filter.FilterMaskIdHigh = 0x7F0 << 5; // 掩码：只匹配高4位
    filter.FilterMaskIdLow  = 0;
    filter.FilterFIFOAssignment = CAN_FILTER_FIFO1;
    filter.FilterActivation = CAN_FILTER_ENABLE;

    if (HAL_CAN_ConfigFilter(&hcan1, &filter) != HAL_OK) {
        Error_Handler();
    }
}

void CAN_Start(void)
{
    if (HAL_CAN_Start(&hcan1) != HAL_OK) {
        Error_Handler();
    }

    if (HAL_CAN_ActivateNotification(&hcan1,
            CAN_IT_TX_MAILBOX_EMPTY | CAN_IT_RX_FIFO0_MSG_PENDING |
            CAN_IT_RX_FIFO1_MSG_PENDING) != HAL_OK) {
        Error_Handler();
    }
}

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_USART1_UART_Init();
    MX_CAN1_Init();

    printf("CAN Node B - Receiver\r\n");

    CAN_FilterConfig();
    CAN_Start();

    while (1) {
        /* 主循环可添加其他任务 */
        HAL_Delay(100);
    }
}

/* FIFO0接收回调（普通帧）*/
void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan)
{
    if (HAL_CAN_GetRxMessage(hcan, CAN_RX_FIFO0, &rx_header, rx_data) == HAL_OK) {
        printf("RX0[0x%03lX]: ", rx_header.StdId);
        for (int i = 0; i < rx_header.DLC; i++) {
            printf("%02X ", rx_data[i]);
        }
        printf("\r\n");

        /* 如果是来自Node A的0x123帧，回复确认 */
        if (rx_header.StdId == 0x123) {
            tx_data[0] = 0x02; // Node B标识
            tx_data[1] = rx_data[1]; // 回显计数器
            tx_data[2] = 'O';
            tx_data[3] = 'K';
            CAN_Reply(0x123, tx_data, 4);
        }
    }
}

/* FIFO1接收回调（电机帧）*/
void HAL_CAN_RxFifo1MsgPendingCallback(CAN_HandleTypeDef *hcan)
{
    if (HAL_CAN_GetRxMessage(hcan, CAN_RX_FIFO1, &rx_header, rx_data) == HAL_OK) {
        printf("RX1[0x%03lX]: ", rx_header.StdId);
        for (int i = 0; i < rx_header.DLC; i++) {
            printf("%02X ", rx_data[i]);
        }
        printf("\r\n");
    }
}

void HAL_CAN_ErrorCallback(CAN_HandleTypeDef *hcan)
{
    uint32_t err = HAL_CAN_GetError(hcan);
    printf("CAN Error! 0x%08lX\r\n", err);

    /* 尝试恢复总线 */
    if (err & (CAN_ERROR_BOFF | CAN_ERROR_EPVF)) {
        printf("Bus-off/Error Passive - Trying to recover...\r\n");
        HAL_CAN_Stop(hcan);
        HAL_Delay(1000);
        CAN_FilterConfig();
        CAN_Start();
    }
}

❓ 常见问题排查

⚠️ 问题1：两个节点不能通信
现象：发送端显示TX OK，但接收端无数据
原因：波特率不一致、未共地、终端电阻缺失
解决：
1. 首先确认两端CubeMX配置完全一致（Prescaler/BS1/BS2）
2. 用万用表测量总线两端电阻，应为约60Ω
3. 确认两个TJA1050的GND已连接在一起
4. 检查CANH/CANL没有接反
5. 发送端发送时用示波器/逻辑分析仪测量总线，确认有信号
⚠️ 问题2：过滤器配置复杂
现象：数据收不到或收到乱帧
原因：过滤器掩码配置不正确
解决：
1. 初学者先用掩码=0接收所有帧，确认硬件正常
2. IDLIST模式比MASK模式简单，需要多个ID时优先选IDLIST
3. ID左移5位：FilterId = ID << 5（低5位是RTR和IDE）
4. 检查是否与其他过滤器冲突
5. 两个FIFO的过滤器独立配置，可同时启用
⚠️ 问题3：缺少120Ω终端电阻
现象：通信距离超过0.5米后数据错误率高
原因：CAN总线需要特征阻抗匹配
解决：
1. 总线两端各接一个120Ω终端电阻
2. 短距离测试时也建议加上终端电阻
3. 可使用插针式终端电阻模块，便于拔插
4. 确认万用表测量CANH-CANL电阻约60Ω
⚠️ 问题4：波特率计算错误
现象：偶发性接收错误或总线错误计数增加
原因：APB1时钟或分频系数计算错误
解决：
1. 确认APB1实际时钟（CubeMX Clock Configuration查看）
2. 使用公式反推：实际波特率 = APB1时钟 / (Prescaler × (BS1+BS2+1))
3. STM32F103的CAN挂载在APB1（36MHz max），不是APB2
4. 建议初学者使用标准500kbps配置，不要随意修改
⚠️ 问题5：Bus-Off状态
现象：CAN突然停止工作，HAL_CAN_ErrorCallback触发CAN_ERROR_BOFF
原因：连续发送错误过多（如总线断开）解决：
1. Bus-Off是CAN控制器的保护机制，检测到严重错误后自动关闭
2. 错误处理回调中调用 HAL_CAN_Stop 后重新初始化
3. 检查物理层连接（断线、短路）
4. 确保两端共地
💡 调试技巧：
1. 使用CAN分析仪(PCAN/USBCAN)连接电脑，实时监控总线所有帧
2. 示波器测量TXD/RXD引脚，确认波形正确
3. 先测试标准帧，熟练后再用扩展帧
4. 波特率测试：发送0x555555...（产生足够位翻转），测量实际频率
5. 注意STM32的CAN1和CAN2是独立的，本项目只用CAN1

❓ 常见问题 FAQ

Q: CAN通信不上？

A: 1)两个节点波特率必须一致 2)CAN_H和CAN_L不要接反 3)需要120Ω终端电阻 4)至少2个节点才能正常通信

Q: CAN过滤器怎么配？

A: STM32 CAN过滤器有列表模式和掩码模式。初学用列表模式（精确匹配ID），高级用掩码模式（范围过滤）。
